# Package the case study

## Objective

Our big goal in this lesson is to organize your hard work and newfound skills into a clear, compelling story! You'll create a markdown file, `case-study-1.md`, that effectively showcases your problem-solving journey. This file will highlight the problem you tackled, your design choices, how you executed your solution, the evidence of your success, and the lessons you learned. Think of it as your professional portfolio piece, proving your skill and understanding.

## Walkthrough

Let's turn your capstone project into an impressive case study, step by step:

1.  **Create your case study file:**
    We'll start by creating an empty markdown file named `case-study-1.md`.
    ```bash
    touch case-study-1.md
    ```
    *Why this is safe:* The `touch` command is very gentle. It simply creates an empty file if one doesn't already exist with that name. If a file with `case-study-1.md` already existed, `touch` would only update its "last modified" timestamp, leaving its contents completely untouched. It will not delete or alter any of your important work.

2.  **Open the file for editing:**
    Now that the file exists, let's open it in a text editor to add the structure. We'll use `nano`, a friendly editor right in your terminal.
    ```bash
    nano case-study-1.md
    ```
    *Why this is safe:* Opening a file with `nano` is safe. `nano` simply displays the file's contents (or an empty file, in our case) and allows you to type. No changes are saved until you explicitly choose to save (by pressing `Ctrl+O`, then `Enter`, then `Ctrl+X` to exit).

3.  **Add the case study structure:**
    Inside `nano`, type (or copy and paste) the following structure. These headings will guide you in filling out your case study later.
    ```markdown
    # Case Study: [Your Project Title Here]

    ## Problem
    *Describe the challenge or problem you set out to solve.*

    ## Design
    *Explain how you planned to solve the problem. What tools did you choose? Why?*

    ## Run
    *Detail the steps you took to implement your design. What commands did you use?*

    ## Evidence
    *Show the results! This could be screenshots of successful output, configuration files, or other proofs.*

    ## Lessons
    *What did you learn from this project? What would you do differently next time?*
    ```
    Once you've added the structure, save the file by pressing `Ctrl+O`, then `Enter`, and finally `Ctrl+X` to exit `nano`.

You now have a perfectly structured `case-study-1.md` file, ready to be filled with the exciting details of your project!
