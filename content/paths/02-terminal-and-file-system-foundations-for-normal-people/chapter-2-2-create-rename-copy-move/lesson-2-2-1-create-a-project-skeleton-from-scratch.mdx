# Create a project skeleton from scratch

## What you will build
A tiny-agent-project/ folder containing src/main.py, docs/plan.md, tests/test_main.py, and README.md вЂ” all created from the terminal.

## Why this matters
The terminal is the lingua franca of every professional development environment. Navigating confidently from the command line is what separates a developer who can work anywhere from one who is helpless without a GUI. These skills transfer directly to remote servers, CI runners, and containerized environments.

## Before you start
Build a multi-level project folder from the terminal using `mkdir` and verify the result with `tree` or `Get-ChildItem -Recurse`. By the end you will have a real `tiny-agent-project/` with `src/`, `docs/`, and `tests/` subdirectories, plus starter files in the right places.


## Walkthrough

### Step 1 вЂ” Start from a known location

Make sure you are in your home directory (or in the `cli-academy/projects/` folder if you created one in Path 01).

**macOS / Linux**

```bash
cd ~
pwd
```

**Windows PowerShell**

```powershell
cd ~
Get-Location
```

### Step 2 вЂ” Create nested folders in one command

You need `tiny-agent-project/` with three subdirectories: `src`, `docs`, and `tests`. Instead of creating each one separately, use a single command.

**macOS / Linux**

```bash
mkdir -p tiny-agent-project/src tiny-agent-project/docs tiny-agent-project/tests
```

The `-p` flag creates parent directories as needed and does not complain if a folder already exists.

**Windows PowerShell**

```powershell
New-Item -ItemType Directory -Path tiny-agent-project\src, tiny-agent-project\docs, tiny-agent-project\tests -Force
```

`-Force` is the PowerShell equivalent of `-p` вЂ” it creates the parent and silently skips existing folders.

### Step 3 вЂ” Add starter files

Move into the project and create a few files that make it feel like a real project.

**macOS / Linux**

```bash
cd tiny-agent-project
echo "# Tiny Agent Project" > README.md
echo "# To-do" > docs/plan.md
echo "print('hello world')" > src/main.py
echo "# Tests go here" > tests/test_main.py
```

**Windows PowerShell**

```powershell
cd tiny-agent-project
Set-Content README.md "# Tiny Agent Project"
Set-Content docs\plan.md "# To-do"
Set-Content src\main.py "print('hello world')"
Set-Content tests\test_main.py "# Tests go here"
```

`Set-Content` writes text to a file and creates the file if it does not exist. On macOS/Linux, `echo "text" > file` does the same thing.

### Step 4 вЂ” Verify the full tree

Now see everything you built in one view.

**macOS / Linux**

```bash

```

### Step 5 вЂ” Confirm file contents

Spot-check one of the files to make sure the content was written correctly.

**macOS / Linux**

```bash
cat src/main.py
```

**Windows PowerShell**

```powershell
Get-Content src\main.py
```

You should see `print('hello world')`.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `mkdir: cannot create directory: File exists` | The folder already exists from a previous attempt | Safe to ignore вЂ” `mkdir -p` (macOS/Linux) and `-Force` (Windows) handle this silently |
| `tree: command not found` | `tree` is not installed on your system | Use `find . -type f` (macOS/Linux) or `Get-ChildItem -Recurse -Name` (Windows) instead |
| Files end up in the wrong folder | You forgot to `cd tiny-agent-project` before creating files | Check with `pwd` / `Get-Location`, then move the files with `mv` or `Move-Item` |
| PowerShell `Set-Content` creates a file with BOM encoding | Default PowerShell behavior on older versions | Add `-Encoding utf8` to the command: `Set-Content README.md "# Tiny Agent Project" -Encoding utf8` |
| `echo` creates a file named `"#` on macOS | Quotes were mangled by copy-paste | Re-type the command manually, making sure the double quotes wrap the entire string |
