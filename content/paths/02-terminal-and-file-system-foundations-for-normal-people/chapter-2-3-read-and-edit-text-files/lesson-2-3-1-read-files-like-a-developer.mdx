# Read files like a developer

## Objective

Read text files from the terminal using `cat`, `head`, `tail`, `wc`, and their PowerShell equivalents. By the end you will peek at files, grab the first or last few lines, and count lines — all without opening an editor.

## Walkthrough

### Step 1 — Read an entire file with `cat` / `Get-Content`

Make sure you are inside `tiny-agent-project/` (or wherever your project files live).

**macOS / Linux**

```bash
cd ~/tiny-agent-project
cat INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
cd ~/tiny-agent-project
Get-Content INSTRUCTIONS.md
```

The full contents print to the screen. `cat` is short for *concatenate* — it dumps the file's text straight to your terminal. PowerShell's `Get-Content` does the same thing and also accepts `cat` as an alias.

### Step 2 — Read just the first few lines

Larger files may be hundreds or thousands of lines long. Grab only the top.

**macOS / Linux**

```bash
head -n 3 INSTRUCTIONS.md
```

`head -n 3` prints the first three lines.

**Windows PowerShell**

```powershell
Get-Content INSTRUCTIONS.md | Select-Object -First 3
```

`Select-Object -First 3` takes only the first three lines from the pipeline.

### Step 3 — Read the last few lines

Sometimes the most recent information is at the bottom — for example, in log files.

**macOS / Linux**

```bash
tail -n 2 INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
Get-Content INSTRUCTIONS.md | Select-Object -Last 2
```

### Step 4 — Count lines, words, and characters

How big is a file? Check without reading the whole thing.

**macOS / Linux**

```bash
wc INSTRUCTIONS.md
```

`wc` (word count) prints three numbers: lines, words, and characters.

```bash
wc -l INSTRUCTIONS.md
```

`-l` gives only the line count.

**Windows PowerShell**

```powershell
Get-Content INSTRUCTIONS.md | Measure-Object -Line -Word -Character
```

This prints lines, words, and characters in a table.

### Step 5 — Read a file you did not create

Practice on a file that belongs to the system so you see how these commands work on real-world content.

**macOS / Linux**

```bash
head -n 10 /etc/hosts
```

This shows the first 10 lines of the network hosts file — a real config file that every Mac and Linux machine has.

**Windows PowerShell**

```powershell
Get-Content C:\Windows\System32\drivers\etc\hosts | Select-Object -First 10
```

Same file, same idea. You are reading a real system config safely.

### Step 6 — Combine commands to answer a question

Count how many Python files are in your project.

**macOS / Linux**

```bash
find . -name "*.py" | wc -l
```

**Windows PowerShell**

```powershell
(Get-ChildItem -Recurse -Filter "*.py").Count
```

You should see `2` (main.py and test_main.py) if you are inside `tiny-agent-project/`.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `cat: file: No such file or directory` | File name is misspelled or you are in the wrong folder | Run `ls` / `Get-ChildItem -Name` to see available files, then correct the name |
| Output floods the screen and scrolls too fast | The file is very long | Use `head -n 20 file` (macOS/Linux) or `Get-Content file | Select-Object -First 20` (Windows) to limit output |
| `wc` shows 0 lines | The file has content but no newline at the end | `wc -l` counts newline characters. A file with one line and no trailing newline shows 0. Add a newline: `echo "" >> file` |
| `head: command not found` on Windows | `head` is a Unix command, not available in PowerShell by default | Use `Get-Content file | Select-Object -First N` instead |
| `Get-Content` shows garbled characters | File uses a non-UTF-8 encoding | Add `-Encoding` parameter: `Get-Content file -Encoding utf8` |
