# Make a safe text edit

## Objective
Learn the iron-clad rule of safe editing: **always make a backup**, and then safely modify a text file.

## Walkthrough

Eventually, you have to edit a configuration file or a script. When that time comes, there is a golden rule that separates professionals from beginners: professionals always duplicate the file *before* editing it.

**Step 1: The Backup Copy (`cp`)**
We use the `cp` command (**Copy**) to make a duplicate. 
The syntax is `cp <existing_file> <new_file>`.

Let's say you have a file named `config.txt`. Before opening it, you make a backup copy:
`cp config.txt config.txt.backup`
If your edit completely breaks the file later, you can effortlessly restore the old one and breathe a sigh of relief!

**Step 2: Safe Text Editing**
Since you made your backup, you can now safely add text to your file! 
A safe way to *append* text to a file in the terminal without opening an editor is using `echo` with two arrows `>>`. 

Watch out! Single arrow `>` will completely overwrite your file. Double arrow `>>` will gently add text to the bottom. Let's add a line safely:
`echo "Server=Online" >> config.txt`

Now, if you want a complete text editor, you can always open your file in VS Code or Notepad. But doing small tweaks directly in the terminal makes you incredibly fast.
