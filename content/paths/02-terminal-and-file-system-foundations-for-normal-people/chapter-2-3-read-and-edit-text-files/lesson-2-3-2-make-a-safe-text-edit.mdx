# Make a safe text edit

## Objective

Edit a text file safely using the backup-edit-verify pattern. You will make a backup copy, edit the file with two different methods (command-line append and a real editor), and confirm the change with `diff`.

## Walkthrough

### Step 1 — Start with a file worth editing

<StepMeta purpose="Begin with a known file in a known project so you can focus on the edit workflow, not on file hunting." action="Move into tiny-agent-project and print INSTRUCTIONS.md before changing anything." expectedResult="You can read the current file contents and confirm the file exists in the expected location." whyItMatters="Safe editing starts with understanding the exact starting state you are about to modify." nextStep="Once you know the file looks right, create a backup before making even a small change." askTutor="I'm on Step 1 of Make a safe text edit. Help me confirm that this is the right file to edit and explain what I should notice before changing it." />

Make sure you are in `tiny-agent-project/` and have `INSTRUCTIONS.md` from the previous lessons.

**macOS / Linux**

```bash
cd ~/tiny-agent-project
cat INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
cd ~/tiny-agent-project
Get-Content INSTRUCTIONS.md
```

You should see the heading text from Lesson 2.2.1.

### Step 2 — Make a backup copy before any edit

<StepMeta purpose="Create an immediate recovery point before touching the file." action="Copy INSTRUCTIONS.md to INSTRUCTIONS.md.bak and list the matching markdown files." expectedResult="Both the original file and the .bak backup are visible side by side." whyItMatters="Backups make small edits reversible, which removes pressure and encourages careful experimentation." nextStep="With the backup in place, append a few lines from the command line so you can compare before and after later." askTutor="I'm on Step 2 of Make a safe text edit. Help me verify that I created a real backup copy and not just another reference to the same file." />

This is the professional habit: copy first, edit second.

**macOS / Linux**

```bash
cp INSTRUCTIONS.md INSTRUCTIONS.md.bak
ls *.md*
```

**Windows PowerShell**

```powershell
Copy-Item INSTRUCTIONS.md INSTRUCTIONS.md.bak
Get-ChildItem -Filter "INSTRUCTIONS*"
```

You now have the original and a `.bak` safety net side by side.

### Step 3 — Append a line from the command line

<StepMeta purpose="Practice the safest simple edit: adding text without reopening or rewriting the whole file." action="Append a blank line, a Status heading, and one status sentence to INSTRUCTIONS.md." expectedResult="The file gains three new lines at the end while the earlier content remains intact." whyItMatters="Learning the difference between append and overwrite is one of the most important safety boundaries in shell-based editing." nextStep="After appending, compare the edited file against the backup so you can see only the lines you added." askTutor="I'm on Step 3 of Make a safe text edit. Help me check that I appended to the file rather than overwrote it, and explain why >> or Add-Content is the safe choice here." />

Add a new line to the file without opening an editor.

**macOS / Linux**

```bash
echo "" >> INSTRUCTIONS.md
echo "## Status" >> INSTRUCTIONS.md
echo "Project is active." >> INSTRUCTIONS.md
```

The double arrow `>>` **appends** to the end. A single arrow `>` would erase everything and replace it — never use `>` on a file you want to keep.

**Windows PowerShell**

```powershell
Add-Content INSTRUCTIONS.md ""
Add-Content INSTRUCTIONS.md "## Status"
Add-Content INSTRUCTIONS.md "Project is active."
```

`Add-Content` always appends. The destructive equivalent is `Set-Content`, which overwrites.

### Step 4 — Verify the change with `diff`

<StepMeta purpose="Use a file comparison tool to inspect exactly what changed instead of trusting memory." action="Run diff or Compare-Object between the backup and the edited file." expectedResult="The output highlights only the newly added lines in the edited file." whyItMatters="Diff tools are the safest way to review changes because they expose scope and content precisely." nextStep="Once the append-only change is confirmed, open the file in a real editor for a slightly bigger manual edit." askTutor="I'm on Step 4 of Make a safe text edit. Help me read this diff output and tell me whether it shows only the intended lines being added." />

Compare the edited file to the backup.

**macOS / Linux**

```bash
diff INSTRUCTIONS.md.bak INSTRUCTIONS.md
```

Lines starting with `>` are new additions. You should see the three lines you just added.

**Windows PowerShell**

```powershell
Compare-Object (Get-Content INSTRUCTIONS.md.bak) (Get-Content INSTRUCTIONS.md)
```

Lines marked `=>` exist only in the edited file. This confirms exactly what changed.

### Step 5 — Open the file in a real editor from the terminal

<StepMeta purpose="Switch from command-line appends to a real editor while staying rooted in terminal workflow." action="Open INSTRUCTIONS.md in VS Code or another editor from the terminal, add one small line such as a last-updated note, and save it." expectedResult="The editor opens the target file, the new line is saved, and the file still lives in the same project folder." whyItMatters="Most real changes mix terminal-driven workflow with editor-driven typing, so you need both modes to feel normal and safe." nextStep="After saving the editor change, run the diff again so you can see both the append and the manual edit." askTutor="I'm on Step 5 of Make a safe text edit. Help me choose the safest editor option for my machine and tell me how to verify the save actually happened before I close it." />

For bigger edits, use an actual text editor. VS Code is the most beginner-friendly option.

**macOS / Linux**

```bash
code INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
code INSTRUCTIONS.md
```

This opens the file in VS Code. Make any change you like — for example, add a line that says `Last updated: today`. Save the file (Ctrl+S or Cmd+S).

If VS Code is not installed, you can use:
- **nano** (macOS/Linux): `nano INSTRUCTIONS.md` — edit, then press Ctrl+O to save and Ctrl+X to exit.
- **Notepad** (Windows): `notepad INSTRUCTIONS.md` — edit, then File → Save and close.

### Step 6 — Verify the editor change

<StepMeta purpose="Confirm the editor save changed exactly what you intended and nothing more." action="Run diff or Compare-Object again after the editor save and review the full set of changes against the backup." expectedResult="The comparison now shows both the earlier appended lines and the new editor-made line." whyItMatters="Repeated verification is how you catch accidental edits early before they spread into later commits." nextStep="If the diff looks correct, optionally remove the backup once you are sure you no longer need the safety copy." askTutor="I'm on Step 6 of Make a safe text edit. Help me compare this second diff to the first one and tell me whether any unexpected lines changed during the editor step." />

Run the diff again to see both the append and the editor change.

**macOS / Linux**

```bash
diff INSTRUCTIONS.md.bak INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
Compare-Object (Get-Content INSTRUCTIONS.md.bak) (Get-Content INSTRUCTIONS.md)
```

Every changed line is visible. If you ever need to undo everything, restore from the backup:

**macOS / Linux**

```bash
cp INSTRUCTIONS.md.bak INSTRUCTIONS.md
```

**Windows PowerShell**

```powershell
Copy-Item INSTRUCTIONS.md.bak INSTRUCTIONS.md -Force
```

### Step 7 — Clean up (optional)

<StepMeta purpose="Finish the exercise by deciding whether the backup is still useful or safe to remove." action="Delete INSTRUCTIONS.md.bak only if you are satisfied with the edited file and no longer need the recovery copy." expectedResult="The backup file is removed, leaving the edited main file as the final version." whyItMatters="Cleanup is only safe after verification, and this reinforces the order: backup, edit, verify, then optionally remove the backup." nextStep="After this step, you have completed the full backup-edit-verify loop and are ready to build a larger reference file in the mini project." askTutor="I'm on Step 7 of Make a safe text edit. Help me decide whether it is safe to delete the backup yet or whether I should keep it a little longer." />

If you are happy with the edits and no longer need the backup:

**macOS / Linux**

```bash
rm INSTRUCTIONS.md.bak
```

**Windows PowerShell**

```powershell
Remove-Item INSTRUCTIONS.md.bak
```

Only delete the backup once you are sure the edited version is correct.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| Used `>` instead of `>>` and the file is now one line | Single arrow overwrites the entire file | Restore from backup: `cp INSTRUCTIONS.md.bak INSTRUCTIONS.md` (macOS/Linux) or `Copy-Item INSTRUCTIONS.md.bak INSTRUCTIONS.md -Force` (Windows) |
| `diff` shows no output | The files are identical — your edit did not save | Re-run the append command or re-save in the editor, then diff again |
| `code` command not found | VS Code is not in your system PATH | Open VS Code manually, then install the shell command: open the Command Palette (Ctrl+Shift+P) and run "Shell Command: Install 'code' command in PATH" |
| `nano` is not available on Windows | nano is a Unix editor not included in Windows | Use `notepad INSTRUCTIONS.md` or `code INSTRUCTIONS.md` instead |
| PowerShell `Compare-Object` shows `SideIndicator` columns that are confusing | This is normal output — `<=` means left-only (backup), `=>` means right-only (edited) | Focus on the `InputObject` column for the actual text differences |
