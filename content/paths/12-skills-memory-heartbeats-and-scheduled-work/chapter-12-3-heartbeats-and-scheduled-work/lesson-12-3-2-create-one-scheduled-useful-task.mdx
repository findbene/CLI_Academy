# Create one scheduled useful task

## Objective

In this lesson, you'll learn to create a simple, useful task that can be scheduled to run automatically. More importantly, you'll discover how to build in safety nets by identifying *where* human approval is essential before automated actions take place, especially for tasks that might modify your files. This skill is crucial for building robust and safe automated workflows.

## Walkthrough

Let's set up a scheduled task that helps you manage your files, keeping safety and approval in mind.

1.  **Safety First: Understand Automation (Conceptual Scheduling)**
    Imagine a friendly robot alarm clock for your computer, often called `cron` on Linux/macOS. It simply runs commands for you at specific times. When we automate tasks, especially ones that change things, we must be extra careful.

2.  **Craft a Safe "Cleanup" Listing Task**
    We'll start with a safe task: listing old files in your `Downloads` folder. This is a read-only action, meaning it won't delete or change anything.
    *   Open your terminal.
    *   Run this command:
        ```bash
        find ~/Downloads -maxdepth 1 -type f -mtime +30
        ```
    *   **What this does and why it's safe:**
        *   `find ~/Downloads`: Looks for files within your `Downloads` directory.
        *   `-maxdepth 1`: Tells `find` to only look in the `Downloads` folder itself, not inside any subfolders. This keeps it simple and focused.
        *   `-type f`: Specifies that we're only interested in *files* (not directories).
        *   `-mtime +30`: Filters for files that were last modified more than 30 days ago.
        *   **Crucially:** This command *only displays* the matching files. It does not delete, move, or change them in any way. You can safely run this command as many times as you like.
    *   **Your Task:** This `find` command is our "useful task." It helps identify stale files that *might* be candidates for cleanup.

3.  **Identify the Human Approval Step**
    Now, imagine this command is scheduled to run every week. If we wanted to *actually delete* these old files automatically, that would be risky! This is where human approval rules become vital.
    *   **Where approval belongs:** Before any destructive action (like `rm` or `mv`), a human should review the list generated by the `find` command.
    *   **The Approval Rule:** Your automated process would run the `find` command, collect its output (the list of old files), and *then* send that list to you (or a designated person) for review. Only *after* you give explicit approval would a *separate* command (like `find ... -delete`) be triggered to perform the actual cleanup. This ensures you always have the final say over what gets removed, preventing accidental data loss.
