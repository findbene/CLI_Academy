# What multi-agent means in Claude Code

## Objective

By the end of this lesson you will understand what "multi-agent" actually means inside Claude Code — separate sessions or subagents, each governed by its own `CLAUDE.md` instructions, collaborating on parts of a larger task. You will create a minimal two-agent directory structure, run both agents, and see the handoff in practice.

This is **not** about external frameworks like AutoGen or LangChain. Everything here uses native Claude Code patterns: `CLAUDE.md` contracts, the Task tool for subagent spawning, and file-based handoffs.

## Walkthrough

### Step 1 — Understand the core concept

In Claude Code, "multi-agent" means running more than one Claude session where each session has a distinct `CLAUDE.md` that constrains what it can do. The agents share a filesystem — one agent writes a file, the next agent reads it.

Key pieces:
- **Agent** = a Claude Code session scoped by a `CLAUDE.md` file.
- **Contract** = the `CLAUDE.md` that defines the agent's role, allowed outputs, and stop condition.
- **Handoff** = one agent writes an output file; the next agent's contract names that file as its input.
- **Coordinator** = a top-level `CLAUDE.md` (or a human) that runs agents in order.

### Step 2 — Create a project with two agent contracts

**macOS / Linux:**
```bash
mkdir -p multi-agent-intro && cd multi-agent-intro
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Path multi-agent-intro -Force; Set-Location multi-agent-intro
```

Now create the first agent contract — a **Lister** that inventories Markdown files:

**macOS / Linux:**
```bash
cat > lister-CLAUDE.md << 'EOF'
# Lister Agent Contract

## Role
You list every `.md` file in the current directory and write the list to `file-inventory.md`.

## Outputs
- `file-inventory.md` — one filename per line.

## Boundaries
- Do NOT read file contents.
- Do NOT create any file other than `file-inventory.md`.

## Stop Condition
Stop after writing `file-inventory.md`.
EOF
```

**Windows PowerShell:**
```powershell
@"
# Lister Agent Contract

## Role
You list every ``.md`` file in the current directory and write the list to ``file-inventory.md``.

## Outputs
- ``file-inventory.md`` — one filename per line.

## Boundaries
- Do NOT read file contents.
- Do NOT create any file other than ``file-inventory.md``.

## Stop Condition
Stop after writing ``file-inventory.md``.
"@ | Set-Content lister-CLAUDE.md
```

Next, create the second agent — a **Summarizer** that reads the inventory:

**macOS / Linux:**
```bash
cat > summarizer-CLAUDE.md << 'EOF'
# Summarizer Agent Contract

## Role
You read `file-inventory.md` and write a one-sentence summary of how many files exist and their naming pattern.

## Inputs
- `file-inventory.md` (produced by the Lister agent)

## Outputs
- `summary.md` — a short paragraph.

## Boundaries
- Do NOT modify `file-inventory.md`.
- Do NOT create files other than `summary.md`.
- If `file-inventory.md` does not exist, STOP and output an error message.

## Stop Condition
Stop after writing `summary.md`.
EOF
```

**Windows PowerShell:**
```powershell
@"
# Summarizer Agent Contract

## Role
You read ``file-inventory.md`` and write a one-sentence summary of how many files exist and their naming pattern.

## Inputs
- ``file-inventory.md`` (produced by the Lister agent)

## Outputs
- ``summary.md`` — a short paragraph.

## Boundaries
- Do NOT modify ``file-inventory.md``.
- Do NOT create files other than ``summary.md``.
- If ``file-inventory.md`` does not exist, STOP and output an error message.

## Stop Condition
Stop after writing ``summary.md``.
"@ | Set-Content summarizer-CLAUDE.md
```

### Step 3 — Run the agents in sequence

Open Claude Code in the `multi-agent-intro/` directory.

First, tell Claude Code to act as the Lister:

```
Follow the instructions in lister-CLAUDE.md exactly. Do not deviate from the contract.
```

After it finishes, verify the output:

**macOS / Linux:**
```bash
cat file-inventory.md
```

**Windows PowerShell:**
```powershell
Get-Content file-inventory.md
```

Now run the second agent:

```
Follow the instructions in summarizer-CLAUDE.md exactly. Do not deviate from the contract.
```

Check the final output:

**macOS / Linux:**
```bash
cat summary.md
```

**Windows PowerShell:**
```powershell
Get-Content summary.md
```

You just ran a multi-agent workflow. Two agents, two contracts, one handoff file.

### Step 4 — Understand what makes this "multi-agent"

The pattern has three distinctive properties:

1. **Isolation** — each agent only sees its own contract. The Lister cannot summarize; the Summarizer cannot list.
2. **Explicit handoff** — the file `file-inventory.md` is the only bridge. There is no shared memory or hidden state.
3. **Independent stop conditions** — each agent stops on its own terms, not because the other agent told it to.

If you removed the contracts and told one Claude session to "list files and then summarize them," it would work — but you would lose the ability to audit each step independently or swap one agent for a different one later.

### What can go wrong

| Problem | Cause | Fix |
|---------|-------|-----|
| Summarizer says `file-inventory.md` not found | Lister did not run first or failed silently | Run Lister and verify the file exists before starting Summarizer |
| Lister reads file contents instead of listing names | Contract boundary not strict enough | Add "Do NOT open or read any file" to the Lister contract |
| Both agents write to the same filename | Output names collide in contracts | Give each agent a unique output filename |
| Agent keeps going after producing its file | Missing or vague stop condition | Use "Stop immediately after writing X. Do not iterate." |
| Claude ignores the contract | Prompt phrasing too soft | Start with "Follow the instructions in X-CLAUDE.md exactly" |

> **Cost warning:** If an agent's stop condition is missing or ambiguous, it can keep generating output indefinitely. Every token costs money. Always define explicit stop conditions and test with cheap, small tasks first.
