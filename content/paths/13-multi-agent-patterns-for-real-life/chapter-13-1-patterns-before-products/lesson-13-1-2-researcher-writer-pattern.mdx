# Researcher-writer pattern

## Objective
In this lesson, you'll learn a powerful strategy for tackling complex writing tasks: the "researcher-writer" split. We'll set up a workflow that separates gathering information from drafting your final piece, improving clarity and efficiency. You'll gain practical skills in organizing your project files effectively using the command line.

## Walkthrough

Let's create a structured environment for your project, separating your research notes from your final writing.

1.  **Create Your Project Space:**
    First, we'll make a dedicated folder for your project. This keeps everything tidy and contained.
    ```bash
    mkdir my_new_project
    ```
    *   `mkdir` is a very safe command. It simply *makes* a new, empty *directory* (folder) with the name you provide. It won't touch or change any existing files.

2.  **Enter Your Project:**
    Now, let's move into our new project folder.
    ```bash
    cd my_new_project
    ```
    *   `cd` (change directory) is also a safe command. It only changes your current location in the file system. It doesn't modify any files or folders.

3.  **Define Research and Writing Areas:**
    Inside `my_new_project`, we'll create two distinct folders: one for all your gathered information and another for your final document drafts.
    ```bash
    mkdir research writing
    ```
    *   Again, `mkdir` safely creates these two new, empty folders.

4.  **Simulate the Research Phase:**
    Imagine you're now in "researcher" mode. You're gathering facts, links, and ideas.
    ```bash
    cd research
    echo "Source 1: Interesting article on topic X (link)" > research_notes.md
    echo "Source 2: Key facts for section A" >> research_notes.md
    ```
    *   `echo "..." > research_notes.md`: `echo` simply prints text. The `>` symbol redirects that text into a file named `research_notes.md`. If the file doesn't exist, it's safely created. If it *does* exist, it will be safely *overwritten* with the new text. This is why we use `>` only for the first entry.
    *   `echo "..." >> research_notes.md`: The `>>` symbol also redirects text, but it *appends* it to the end of the file. This is safe for adding more content without deleting what's already there.

5.  **The Handoff Contract:**
    The "handoff contract" is an agreement between your "researcher" self and your "writer" self. In this setup, the contract is simple: the `research/` folder will contain all raw notes, links, and key points, clearly organized, so the "writer" doesn't have to go hunting for information.

6.  **Simulate the Writing Phase:**
    Now, switch to "writer" mode. You'll use the notes from `research/` to craft your final piece.
    ```bash
    cd ../writing
    echo "# Final Document Title" > final_draft.md
    echo "Based on my research, here are the main points..." >> final_draft.md
    ```
    *   `cd ../writing`: `..` means "go up one directory level" (back to `my_new_project`), then `writing` tells `cd` to enter that folder. Safe and effective for navigation.
    *   We use `echo` and `>`/`>>` again to safely create and add content to your `final_draft.md`, mimicking the writing process.

7.  **Review Your Structure:**
    Let's look at what you've built.
    ```bash
    cd ..
    ls -R
    ```
    *   `ls -R` (list recursively) is safe. It only displays the names of files and folders within your project, showing you the organized structure you've created.

You now have a clear, separated workflow for research and writing!
