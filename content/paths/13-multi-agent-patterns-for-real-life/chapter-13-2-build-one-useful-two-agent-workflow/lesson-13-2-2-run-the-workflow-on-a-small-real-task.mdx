# Build a two-agent research-and-write pipeline

## Objective

In Lesson 13.2.1 you created agent contracts with defined roles, boundaries, and stop conditions. Now you will put those contracts to work. You will build a complete two-agent pipeline — a Researcher that gathers findings into a file and a Writer that turns those findings into a polished document — run it end-to-end on a real topic, and verify the handoff.

By the end you will have a working `research-write-pipeline/` directory with contracts, a research output, and a finished document you can read.

## Walkthrough

### Step 1 — Create the pipeline directory

<StepMeta purpose="Create a dedicated workspace for the first full researcher-writer pipeline run." action="Make the research-write-pipeline directory and enter it before creating the two contracts." expectedResult="You have a clean project folder ready for both agent contracts and their handoff/output files." whyItMatters="A separate pipeline directory makes it obvious which files belong to the workflow and simplifies auditing the results later." nextStep="Write the Researcher contract so the pipeline starts from bounded factual output rather than prose." askTutor="Help me verify that this pipeline directory is structured cleanly enough for a simple researcher-writer handoff." />

**macOS / Linux:**
```bash
mkdir -p research-write-pipeline && cd research-write-pipeline
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Path research-write-pipeline -Force; Set-Location research-write-pipeline
```

### Step 2 — Write the Researcher contract

<StepMeta purpose="Define a first agent that produces a narrow, usable findings file on a real beginner topic." action="Create researcher-CLAUDE.md exactly as written so it gathers bullet findings without drifting into recommendations or article prose." expectedResult="The Researcher contract exists and is scoped to produce only research-findings.md in one pass." whyItMatters="A clean writer handoff depends on having a facts-first file instead of a half-finished article from the first agent." nextStep="Add the Writer contract that consumes those findings and turns them into a readable document without inventing anything new." askTutor="Review this Researcher contract and tell me whether it is constrained tightly enough to produce a reliable findings file." />

**macOS / Linux:**
```bash
cat > researcher-CLAUDE.md << 'EOF'
# Researcher Agent Contract

## Role
You are a factual researcher. Gather information on the assigned topic.

## Topic
"What are the most common mistakes beginners make when learning the command line?"

## Outputs
- A file called `research-findings.md` containing:
  - 5–8 bullet points of findings
  - Each bullet includes a brief explanation
  - A confidence note at the bottom: High / Medium / Low

## Boundaries
- Do NOT write prose, conclusions, or recommendations
- Do NOT create any file except `research-findings.md`
- Do NOT continue after writing the file

## Stop Condition
Stop immediately after writing `research-findings.md`. One pass. No iteration.
EOF
```


### Step 3 — Write the Writer contract

<StepMeta purpose="Define the second agent so the writing phase is grounded in the research handoff instead of fresh speculation." action="Create writer-CLAUDE.md with explicit inputs, outputs, and guardrails against inventing facts or editing the findings file." expectedResult="The Writer contract is ready to transform research-findings.md into article-draft.md while respecting the handoff boundary." whyItMatters="Most researcher-writer failures happen when the writer goes beyond the findings, so the boundary text matters as much as the output format." nextStep="Run the Researcher first and confirm the handoff file exists before you let the Writer start." askTutor="Check whether my Writer contract is strict enough to stop it from adding unsupported claims or bypassing the findings file." />

**macOS / Linux:**
```bash
cat > writer-CLAUDE.md << 'EOF'
# Writer Agent Contract

## Role
You are a technical writer. Turn raw findings into a clear, beginner-friendly article.

## Inputs
- `research-findings.md` (produced by the Researcher agent)

## Outputs
- A file called `article-draft.md` containing:
  - A title
  - An introduction (2–3 sentences)
  - A section for each finding, rewritten in plain English
  - A short conclusion

## Boundaries
- Do NOT invent facts that are not in `research-findings.md`
- Do NOT modify `research-findings.md`
- Do NOT create any file except `article-draft.md`
- If `research-findings.md` does not exist or is empty, STOP and report the error

## Stop Condition
Stop immediately after writing `article-draft.md`. Do not iterate.
EOF
```

**Windows PowerShell:**
```powershell
@"

## Role

## Inputs

## Stop Condition
```

### Step 4 — Run the Researcher

<StepMeta purpose="Generate the source-of-truth handoff file that the Writer will depend on next." action="Run the Researcher contract through Claude Code and verify that research-findings.md exists and contains real bullet findings." expectedResult="You get a non-empty findings file that the Writer can consume without ambiguity." whyItMatters="If the handoff file is weak or empty, the rest of the pipeline is invalid before the Writer even starts." nextStep="Start the Writer only after the findings file passes the existence and content check." askTutor="Help me inspect this findings file and tell me whether it is strong enough for the Writer to use safely." />

Open Claude Code in the `research-write-pipeline/` directory. Give it this prompt:

```
Follow the instructions in researcher-CLAUDE.md exactly. Do not deviate from the contract.
```

When it finishes, verify the handoff file exists and has real content:

**macOS / Linux:**
```bash
test -s research-findings.md && echo "Handoff file OK" || echo "ERROR: file missing or empty"
cat research-findings.md
```

**Windows PowerShell:**
```powershell
if (Test-Path research-findings.md) { Write-Host "Handoff file OK"; Get-Content research-findings.md } else { Write-Host "ERROR: file missing or empty" }
```

If the file is missing or empty, re-run the Researcher before continuing. The Writer cannot operate without valid input.

### Step 5 — Run the Writer

<StepMeta purpose="Turn the verified research handoff into a beginner-friendly article while keeping the facts grounded." action="Run the Writer contract exactly, then verify article-draft.md exists and contains the expected titled article structure." expectedResult="The Writer produces a draft article that clearly builds on the findings file rather than on hidden context." whyItMatters="This is where the split either proves its value or reveals drift, because the writer must stay faithful to upstream input." nextStep="Audit the article against the findings side by side and look for unsupported claims or missing translations of key bullets." askTutor="Compare this article draft to the findings file and tell me where the writer may have overreached or underused the source material." />

In the same directory, start a new Claude Code session (or a new prompt in the same session) and say:

```
Follow the instructions in writer-CLAUDE.md exactly. Do not deviate from the contract.
```

Verify the final output:

**macOS / Linux:**
```bash
test -s article-draft.md && echo "Article created" || echo "ERROR: article missing"
cat article-draft.md
```

**Windows PowerShell:**
```powershell
if (Test-Path article-draft.md) { Write-Host "Article created"; Get-Content article-draft.md } else { Write-Host "ERROR: article missing" }
```

### Step 6 — Audit the handoff

<StepMeta purpose="Validate the integrity of the researcher-to-writer handoff before trusting the pipeline output." action="Compare research-findings.md and article-draft.md claim by claim, confirming each article point traces back to the source bullets." expectedResult="You can explain whether the Writer stayed within contract or identify the exact lines where it invented or distorted content." whyItMatters="Handoff audits are what make multi-agent workflows trustworthy rather than merely polished-looking." nextStep="Use the same handoff mindset in the next lesson, where you automate the sequence and insert checks between agents." askTutor="Help me audit this handoff and point out any article sentence that cannot be traced back to a specific finding." />

Check that the Writer did not invent facts:

1. Open `research-findings.md` and `article-draft.md` side-by-side.
2. For every claim in the article, confirm it traces back to a bullet in the findings.
3. If you find a claim with no source, the Writer violated its contract.

This manual audit is how real multi-agent systems are validated. Automated checks are possible (and you will build them in Lesson 13.3.2), but the human eye catches subtle drift that scripts miss.

### What can go wrong

| Problem | Cause | Fix |
|---------|-------|-----|
| `research-findings.md` is empty | Researcher wrote nothing or misunderstood the topic | Check the contract's Topic section is clear and specific |
| Writer invents facts not in the findings | Boundary rule ignored or too vague | Strengthen the contract: "Every claim MUST trace to a bullet in research-findings.md" |
| Writer modifies `research-findings.md` | Contract boundary not enforced | Add "NEVER open research-findings.md for writing" to the contract |
| Researcher writes an article instead of bullet points | Role description ambiguous | Add "Do NOT write prose, paragraphs, or conclusions" explicitly |
| Pipeline takes many minutes and racks up cost | Agents iterating beyond stop condition | Verify each contract has "Stop immediately after writing X. One pass." |

> **Cost warning:** Each agent session consumes API tokens proportional to context size plus output. A two-agent pipeline costs roughly twice what a single session costs. Run on small topics first to validate contracts before scaling to larger tasks.
