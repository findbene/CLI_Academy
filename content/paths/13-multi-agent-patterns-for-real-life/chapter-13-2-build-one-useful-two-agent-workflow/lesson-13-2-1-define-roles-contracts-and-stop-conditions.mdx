# Define roles, contracts, and stop conditions

## Objective

When two Claude Code agents collaborate on a task, each one needs a clear role, a defined boundary, and an explicit stop condition. Without these, agents duplicate effort, contradict each other, or run indefinitely. In this lesson you will create a `CLAUDE.md`-based contract that assigns two agents non-overlapping responsibilities and defines exactly when each one stops.

By the end, you will have a working two-agent contract directory that you can hand to Claude Code to execute.

## Why this matters

Multi-agent systems fail for three common reasons:

1. **Role overlap** — both agents research the same thing and produce conflicting summaries.
2. **Missing stop conditions** — an agent keeps iterating because nobody told it when "done" looks like.
3. **No handoff protocol** — Agent A finishes but Agent B never receives the output.

The contract pattern solves all three.

### Step 2 — Write the Researcher agent contract

<StepMeta purpose="Define the first agent narrowly enough that it produces a reliable handoff instead of a mixed research-plus-analysis blob." action="Create researcher-CLAUDE.md with the exact role, outputs, boundaries, and stop condition shown in the lesson." expectedResult="The Researcher contract writes one findings file and explicitly forbids analysis or extra file creation." whyItMatters="A strong first contract is what keeps downstream analysis grounded in auditable findings instead of vague summaries." nextStep="Add the Analyzer contract next, making sure it only consumes the findings and does not re-do the research." askTutor="Review my Researcher contract and point out any wording that still leaves room for analysis, extra outputs, or endless iteration." />

Create a file called `researcher-CLAUDE.md`. This is the instruction file that defines Agent 1's role, inputs, outputs, and stop condition.

**macOS / Linux:**
```bash
cat > researcher-CLAUDE.md << 'EOF'
# Researcher Agent Contract

## Role
You are a factual researcher. Your job is to gather raw information on the assigned topic.

## Inputs
- A topic string provided by the coordinator
- Access to web search or provided source documents

## Outputs
- A file called `research-findings.md` containing:
  - 5-10 bullet points of factual findings
  - Source attribution for each finding
  - A one-line confidence note ("High confidence" / "Needs verification")

## Boundaries
- Do NOT analyze, interpret, or draw conclusions
- Do NOT modify any file other than `research-findings.md`
- Do NOT continue after writing the findings file

## Stop Condition
Stop immediately after writing `research-findings.md`. Do not iterate.
EOF
```

### Step 3 — Write the Analyzer agent contract

<StepMeta purpose="Create a second role that adds value without overlapping the first agent's work." action="Write analyzer-CLAUDE.md so it consumes the findings file, produces an analysis file, and refuses to gather new information." expectedResult="The Analyzer contract depends on the Researcher output and clearly limits itself to insight extraction and one recommendation." whyItMatters="Distinct responsibilities are what make multi-agent work composable; otherwise you just pay for duplication." nextStep="Write the coordinator file that enforces execution order and protects the handoff boundary." askTutor="Tell me whether the Analyzer contract is truly non-overlapping with the Researcher or whether it still has hidden room to re-research or rewrite facts." />

Create `analyzer-CLAUDE.md` for Agent 2:

**macOS / Linux:**
```bash
cat > analyzer-CLAUDE.md << 'EOF'
# Analyzer Agent Contract

## Role
You are an analyst. Your job is to take the Researcher's findings and extract insights.

## Inputs
- The file `research-findings.md` (produced by the Researcher agent)

## Outputs
- A file called `analysis-report.md` containing:
  - 3-5 key insights derived from the findings
  - One "recommendation" section with a single actionable next step
  - A confidence rating for the overall analysis

## Boundaries
- Do NOT gather new information or search for sources
- Do NOT modify `research-findings.md`
- Do NOT continue after writing the analysis report
- If `research-findings.md` does not exist, STOP and report the error

## Stop Condition
Stop immediately after writing `analysis-report.md`. Do not iterate.
EOF
```

### Step 4 — Write the coordinator CLAUDE.md

<StepMeta purpose="Centralize the workflow order and safety rules so the two contracts behave like one controlled pipeline." action="Create the top-level CLAUDE.md with execution order, topic, and one-run safety constraints for both agents." expectedResult="The project now has a coordinator contract that spells out sequence, verification points, and failure behavior." whyItMatters="Without a coordinator, even good contracts can run in the wrong order or overwrite each other's work." nextStep="Run the two-agent workflow and verify that each expected output file appears with content." askTutor="Review this coordinator file and tell me whether the execution order and safety rules are strong enough to prevent accidental loops or file collisions." />

This is the main `CLAUDE.md` that orchestrates both agents:

**macOS / Linux:**
```bash
cat > CLAUDE.md << 'EOF'
# Two-Agent Research Workflow

## Execution Order
1. Run the Researcher agent using `researcher-CLAUDE.md`
2. Verify `research-findings.md` exists and has content
3. Run the Analyzer agent using `analyzer-CLAUDE.md`
4. Verify `analysis-report.md` exists and has content

## Topic
"What are the three most common mistakes beginners make when setting up Claude Code?"

## Safety Rules
- Each agent runs once. No loops.
- If any agent fails, stop the workflow and report which agent failed and why.
- Never allow an agent to modify another agent's output file.
EOF
```

- `researcher-CLAUDE.md` — Agent 1's contract

- `analysis-report.md` — Agent 2's output

### What can go wrong

| Problem | Cause | Fix |
|---------|-------|-----|
| Researcher keeps iterating | Missing stop condition | Add explicit "Stop immediately after writing" to the contract |
| Analyzer invents new facts | Role boundary not enforced | Add "Do NOT gather new information" to the Boundaries section |
| Both agents produce the same file | Output names collide | Give each agent a unique output filename |
| Analyzer runs before Researcher finishes | No execution order | Define sequential order in the coordinator `CLAUDE.md` |

> [!WARN]
> **Safety note:** Never set agents to run in an unbounded loop. Always define a maximum iteration count or an explicit stop condition. An agent without a stop condition can consume API credits indefinitely and produce unpredictable results.

## Key takeaways

- Every agent needs **Role**, **Inputs**, **Outputs**, **Boundaries**, and a **Stop Condition**.
- The coordinator contract defines execution order and safety rules.
- Claude Code's `CLAUDE.md` file is the natural place to define agent contracts — no external frameworks required.
- Start with two agents. Add more only when you can prove two aren't enough.
