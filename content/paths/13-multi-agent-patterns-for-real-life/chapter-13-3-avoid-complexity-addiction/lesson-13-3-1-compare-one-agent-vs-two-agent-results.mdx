# Compare one-agent vs two-agent results

## Objective
By the end of this lesson, you will have run a simple task using both a single AI agent and a multi-agent (two-agent) setup. You'll capture the results and create a side-by-side comparison sheet. This hands-on experience will help you observe the differences in their output and start to determine if a multi-agent approach genuinely benefits specific workflows.

## Walkthrough

Let's explore the power of single vs. multi-agent systems!

1.  **Prepare your input file:**
    First, let's give our agents something to work with. Create a simple text file with some content.
    ```bash
    echo "The moon is Earth's only natural satellite. It's the fifth-largest satellite in the Solar System. Its surface is covered with craters from asteroid impacts." > moon_facts.txt
    ```
    *   **Safety Check:** The `echo` command only writes the specified text into a new file named `moon_facts.txt` (or overwrites it if it exists). It does not affect other files or your system configuration. It's completely safe.

2.  **Run with a Single Agent:**
    Now, let's have a single agent summarize this file. We'll use a conceptual `run-agent` command for this example.
    ```bash
    run-agent --config single_agent_config.yaml --prompt "Summarize the key points from moon_facts.txt concisely." > single_agent_result.txt
    ```
    *   **Safety Check:** This `run-agent` command (in a real scenario, this would interface with an LLM framework like Autogen or OLLAMA) sends your prompt and the content of `moon_facts.txt` to an AI model. The `> single_agent_result.txt` part safely redirects the AI's textual response from your terminal directly into a new file. No system files are modified, and the process is entirely focused on generating text.

3.  **Run with Two Agents:**
    Next, let's try the same task but with a configuration involving two agents (e.g., one agent to read the file, another to summarize and refine).
    ```bash
    run-agent --config two_agent_config.yaml --prompt "Summarize the key points from moon_facts.txt concisely." > two_agent_result.txt
    ```
    *   **Safety Check:** Just like before, this command orchestrates AI models (in this case, two collaborating ones) to respond to your prompt. The output is safely redirected. It's designed to generate text and observe agent interactions, not to alter your system.

4.  **Create your Comparison Sheet:**
    Finally, combine the results into a markdown file for easy comparison.
    ```bash
    echo "# Single Agent Result" > comparison.md
    cat single_agent_result.txt >> comparison.md
    echo "
# Two Agent Result" >> comparison.md
    cat two_agent_result.txt >> comparison.md
    ```
    *   **Safety Check:** These commands simply write and append text from your result files into a new file named `comparison.md`. They are entirely safe and only manipulate text files you've created.

Now open `comparison.md` in your favorite text editor to review the differences! Consider if the added complexity of two agents genuinely improved the output for this simple task.
