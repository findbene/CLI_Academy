# Run agents in sequence and check handoffs

## Objective

Running agents is only half the job. You also need to verify that each agent finished, that the handoff file is valid, and that the next agent can actually use what the previous one produced. In this lesson you will build a lightweight orchestration script that runs a Researcher and a Writer in sequence, checks the handoff between them, and stops early if something is wrong.

By the end you will have a reusable `run-pipeline.sh` (or `.ps1`) script that coordinates two agents with explicit verification between steps.

## Walkthrough

### Step 1 — Set up the project

**macOS / Linux:**
```bash
mkdir -p sequential-pipeline && cd sequential-pipeline
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Path sequential-pipeline -Force; Set-Location sequential-pipeline
```

Copy or re-create the Researcher and Writer contracts from Lesson 13.2.2 (you can also just copy them):

**macOS / Linux:**
```bash
cat > researcher-CLAUDE.md << 'EOF'
# Researcher Agent Contract
## Role
Gather 5–8 factual bullet points on "common beginner mistakes with Git."
## Outputs
- `research-findings.md` — bullet list with brief explanations.
## Boundaries
- No prose. No conclusions.
- Only create `research-findings.md`.
## Stop Condition
Stop after writing `research-findings.md`.
EOF

cat > writer-CLAUDE.md << 'EOF'
# Writer Agent Contract
## Role
Turn the bullet points in `research-findings.md` into a short, beginner-friendly article.
## Inputs
- `research-findings.md`
## Outputs
- `article-draft.md` — titled article with intro, sections, conclusion.
## Boundaries
- Do not invent facts beyond what is in `research-findings.md`.
- Do not modify `research-findings.md`.
- Only create `article-draft.md`.
- If `research-findings.md` is missing or empty, stop and report error.
## Stop Condition
Stop after writing `article-draft.md`.
EOF
```

### Step 2 — Write the orchestration script (macOS / Linux)

```bash
cat > run-pipeline.sh << 'SCRIPT'
#!/usr/bin/env bash
set -euo pipefail

echo "=== Step 1: Running Researcher ==="
claude --print "Follow the instructions in researcher-CLAUDE.md exactly."

# Verify handoff
if [ ! -s research-findings.md ]; then
  echo "FAIL: research-findings.md is missing or empty. Stopping pipeline."
  exit 1
fi
echo "OK: research-findings.md exists ($(wc -l < research-findings.md) lines)"

echo ""
echo "=== Step 2: Running Writer ==="
claude --print "Follow the instructions in writer-CLAUDE.md exactly."

# Verify output
if [ ! -s article-draft.md ]; then
  echo "FAIL: article-draft.md is missing or empty. Stopping pipeline."
  exit 1
fi
echo "OK: article-draft.md exists ($(wc -l < article-draft.md) lines)"

echo ""
echo "=== Pipeline complete ==="
echo "Files produced:"
ls -la research-findings.md article-draft.md
SCRIPT

chmod +x run-pipeline.sh
```

### Step 3 — Write the orchestration script (Windows PowerShell)

```powershell
@"
`$ErrorActionPreference = "Stop"

Write-Host "=== Step 1: Running Researcher ==="
claude --print "Follow the instructions in researcher-CLAUDE.md exactly."

# Verify handoff
if (-not (Test-Path research-findings.md) -or (Get-Item research-findings.md).Length -eq 0) {
    Write-Host "FAIL: research-findings.md is missing or empty. Stopping pipeline."
    exit 1
}
`$lines = (Get-Content research-findings.md).Count
Write-Host "OK: research-findings.md exists (`$lines lines)"

Write-Host ""
Write-Host "=== Step 2: Running Writer ==="
claude --print "Follow the instructions in writer-CLAUDE.md exactly."

# Verify output
if (-not (Test-Path article-draft.md) -or (Get-Item article-draft.md).Length -eq 0) {
    Write-Host "FAIL: article-draft.md is missing or empty. Stopping pipeline."
    exit 1
}
`$lines = (Get-Content article-draft.md).Count
Write-Host "OK: article-draft.md exists (`$lines lines)"

Write-Host ""
Write-Host "=== Pipeline complete ==="
Get-ChildItem research-findings.md, article-draft.md | Format-Table Name, Length
"@ | Set-Content run-pipeline.ps1
```

### Step 4 — Run the pipeline

**macOS / Linux:**
```bash
./run-pipeline.sh
```

**Windows PowerShell:**
```powershell
.\run-pipeline.ps1
```

Watch for the verification messages between steps. If the Researcher fails, the script exits before the Writer ever starts — exactly the behavior you want.

### Step 5 — Understand what the verification checks do

The key insight is the check between agents:

```bash
if [ ! -s research-findings.md ]; then
  echo "FAIL"; exit 1
fi
```

This does three things:
1. **Confirms the file exists** — the Researcher actually ran and produced output.
2. **Confirms the file is non-empty** — the Researcher didn't create a blank file and stop.
3. **Stops the pipeline early** — the Writer never gets confused by missing input.

Without this check, the Writer would start, see no input file, and either error out with a confusing message or (worse) write an article from nothing.

### What can go wrong

| Problem | Cause | Fix |
|---------|-------|-----|
| Script says `claude: command not found` | Claude CLI is not installed or not on PATH | Install Claude Code CLI and verify with `claude --version` |
| Researcher produces the file but it is empty | Stop condition triggered before output was written | Adjust the contract: output section must come before the stop condition |
| Writer runs even though findings file is bad | Verification check was removed or bypassed | Never skip the handoff check — it is your safety net |
| Pipeline takes 5+ minutes and costs too much | Topic is too broad or agents are iterating | Narrow the topic and confirm stop conditions say "one pass" |
| Script works on Linux but fails on Windows | Bash syntax used in PowerShell | Use the `.ps1` version of the script on Windows |

> **Cost warning:** The orchestration script runs two separate Claude sessions back-to-back. If the Researcher's topic is too broad, it may generate a large output that the Writer then expands further. Keep topics narrow during testing. You can always scale up once the pipeline is proven.
