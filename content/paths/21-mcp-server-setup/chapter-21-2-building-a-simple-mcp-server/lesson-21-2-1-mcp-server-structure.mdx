# MCP Server Structure: What You Need to Build One

## Objective

By the end of this lesson you will have a working `hello-mcp/index.js` MCP server that responds to a `say_hello` tool call. The server will be runnable with Node.js and will return a greeting when Claude invokes the `say_hello` tool. This is the minimum viable MCP server — everything else is an extension of this pattern.

## Walkthrough

### Step 1 — Understand the three required parts

Every MCP server has three required pieces:

**1. Server identity** — name and version, declared at startup. Claude Code uses this to label the server in `/mcp` listings.

**2. Tools array** — a list of tool definitions. Each definition has:
- `name` — the string Claude uses to call the tool
- `description` — plain English explanation of what the tool does (Claude reads this to decide when to use it)
- `inputSchema` — a JSON Schema object describing the parameters Claude must pass

**3. Request handler** — a function that receives the tool name and parameters, executes the logic, and returns a result. This is where your actual code lives.

The official MCP SDK handles the protocol serialization. You write only the tool definitions and the handler logic.

Create the project directory and initialize it.

### Step 3 — Write the server

Create `index.js` in your `hello-mcp` directory.

**macOS / Linux**

```bash
cat > ~/cli-academy/projects/hello-mcp/index.js << 'EOF'
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

// Part 1: Server identity
const server = new Server(
  {
    name: "hello-mcp",
    version: "1.0.0",
  },
  {
    capabilities: {
      tools: {},
    },
  }
);

// Part 2: Tools array
server.setRequestHandler(ListToolsRequestSchema, async () => {
  return {
    tools: [
      {
        name: "say_hello",
        description: "Returns a greeting for a given name. Use this when asked to greet someone.",
        inputSchema: {
          type: "object",
          properties: {
            name: {
              type: "string",
              description: "The name of the person to greet",
            },
          },
          required: ["name"],
        },
      },
    ],
  };
});

// Part 3: Request handler
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  if (request.params.name === "say_hello") {
    const personName = request.params.arguments?.name;
    if (!personName) {
      return {
        content: [
          {
            type: "text",
            text: "Error: name parameter is required",
          },
        ],
        isError: true,
      };
    }
    return {
      content: [
        {
          type: "text",
          text: `Hello, ${personName}! This greeting was generated by your hello-mcp server.`,
        },
      ],
    };
  }

  return {
    content: [
      {
        type: "text",
        text: `Unknown tool: ${request.params.name}`,
      },
    ],
    isError: true,
  };
});

// Start the server on stdin/stdout (the MCP transport Claude Code uses)
const transport = new StdioServerTransport();
await server.connect(transport);
EOF
```

### Step 4 — Add "type": "module" to package.json

The server uses ES module syntax (`import`). Tell Node.js by setting `"type": "module"` in `package.json`.

**macOS / Linux**

```bash
cd ~/cli-academy/projects/hello-mcp
node -e "
const fs = require('fs');
const pkg = JSON.parse(fs.readFileSync('package.json', 'utf8'));
pkg.type = 'module';
fs.writeFileSync('package.json', JSON.stringify(pkg, null, 2));
console.log('Done');
"
```

**Windows PowerShell**

```powershell
Set-Location "$HOME\cli-academy\projects\hello-mcp"
node -e @"
const fs = require('fs');
const pkg = JSON.parse(fs.readFileSync('package.json', 'utf8'));
pkg.type = 'module';
fs.writeFileSync('package.json', JSON.stringify(pkg, null, 2));
console.log('Done');
"@
```

Verify the package.json now includes `"type": "module"`:

**macOS / Linux**

```bash
cat package.json
```

**Windows PowerShell**

```powershell
Get-Content package.json
```

### Step 5 — Test that the server starts without crashing

The server communicates over stdin/stdout, so running it directly will look like it "hangs" (it is waiting for MCP requests). The quick test is to start it and verify it does not immediately crash with an error.

**macOS / Linux**

```bash
cd ~/cli-academy/projects/hello-mcp
timeout 3 node index.js; echo "Exit code: $?"
```

**Windows PowerShell**

```powershell
Set-Location "$HOME\cli-academy\projects\hello-mcp"
$job = Start-Job { node "$HOME\cli-academy\projects\hello-mcp\index.js" }
Start-Sleep -Seconds 2
$state = $job.State
Stop-Job $job
Remove-Job $job
Write-Host "Server state after 2 seconds: $state"
```

On macOS/Linux, `timeout 3` kills the process after 3 seconds — you want the exit code to be `124` (timeout) rather than `1` (error). On Windows, `Running` confirms the server stayed alive.

If Node.js prints an error like `SyntaxError` or `Cannot find module`, read the error message and fix the issue before continuing.

### What can go wrong

| Problem | Cause | Fix |
|---------|-------|-----|
| `Cannot find package '@modelcontextprotocol/sdk'` | npm install did not complete successfully | Re-run `npm install @modelcontextprotocol/sdk` from inside the `hello-mcp` directory |
| `SyntaxError: Cannot use import statement` | `"type": "module"` is missing from package.json | Add `"type": "module"` to package.json as shown in Step 4 |
| Server exits immediately with code 1 | An import path is wrong or the SDK version changed its export structure | Run `node index.js 2>&1` to see the full error; check the SDK's README for the correct import paths |
| `node` command not found | Node.js is not installed or not on PATH | Install Node.js from nodejs.org and restart your terminal |
| The server "hangs" and nothing prints | This is correct behavior — the server is waiting on stdin for MCP requests | Use the timeout test from Step 5 to verify it stays alive without crashing |
