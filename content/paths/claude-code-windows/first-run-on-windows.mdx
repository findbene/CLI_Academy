## What You'll See

A complete walkthrough of your first Claude Code session on Windows. You'll create a project, set up CLAUDE.md, ask Claude Code to build something, and verify it works.

## Prerequisites

You've completed either:

- **WSL2 Setup** (recommended) — You have WSL2, Ubuntu, Node.js, npm, and Claude Code installed
- **PowerShell Alternative** — You have PowerShell, Node.js, npm, and Claude Code installed

You understand CLAUDE.md from the Beginners path.

## The Project: A Simple CLI Tool

We're going to build a simple command-line tool that converts temperatures. This is real enough to be useful, simple enough to complete in 15 minutes.

The tool will:
- Accept a temperature and unit (Celsius/Fahrenheit)
- Convert between the two
- Be easy to use from the command line

## Step 1: Create the Project Directory

**In WSL2:**
```bash
mkdir -p ~/projects/temp-converter
cd ~/projects/temp-converter
```

**In PowerShell:**
```powershell
mkdir C:\Users\YourName\projects\temp-converter
cd C:\Users\YourName\projects\temp-converter
```

From now on, assume you're in this directory in your terminal.

## Step 2: Create CLAUDE.md

Create a file called `CLAUDE.md` in the project directory:

```markdown
# Project: Temperature Converter CLI

A command-line tool that converts between Celsius and Fahrenheit.

## Architecture

- Runtime: Node.js
- Language: JavaScript
- Testing: Jest
- Entry point: index.js or bin/temp-convert

## File Organization

- `bin/` — CLI entry point
- `lib/` — Conversion logic
- `test/` — Jest tests
- `package.json` — Dependencies and scripts

## Coding Standards

- All functions must be documented with JSDoc comments
- All utility functions must have tests
- Use const instead of let/var
- No console.log in library code (only in bin/)

## Before You Code

1. Run `npm test` to verify tests pass
2. Check that Node.js is available: `node --version`
3. If anything is unclear, ask me questions

## Definition of Done

A feature is done when:
- Code passes linting: `npm run lint` (if configured)
- Tests pass: `npm test`
- The CLI works: `node bin/temp-convert 32 F` returns the result
- Documentation is updated
```

## Step 3: Create package.json

Create a file called `package.json`:

```json
{
  "name": "temp-converter",
  "version": "1.0.0",
  "description": "A simple temperature converter CLI",
  "main": "index.js",
  "type": "module",
  "bin": {
    "temp-convert": "bin/temp-convert.js"
  },
  "scripts": {
    "test": "jest",

  },
  "keywords": ["temperature", "converter"],
  "author": "",
  "license": "ISC",
  "devDependencies": {
    "jest": "^29.0.0"
  }
}
```

## Step 4: Install Dependencies

```bash
npm install
```

This creates `node_modules/` and `package-lock.json`. Takes 30 seconds to 1 minute.

/**
 * Convert Celsius to Fahrenheit
 * @param {number} celsius - Temperature in Celsius
 * @returns {number} Temperature in Fahrenheit
 */
export function celsiusToFahrenheit(celsius) {
  return (celsius * 9) / 5 + 32;
}

/**
 * Convert Fahrenheit to Celsius
 * @param {number} fahrenheit - Temperature in Fahrenheit
 * @returns {number} Temperature in Celsius
 */
export function fahrenheitToCelsius(fahrenheit) {
  return ((fahrenheit - 32) * 5) / 9;
}

## Step 7: Ask Claude Code to Create the CLI

Type:

```
Now create the CLI in bin/temp-convert.js that:
- Takes command line arguments: temperature and unit (C or F)
- Calls the appropriate conversion function
- Prints the result in a friendly format

Example usage:
  node bin/temp-convert.js 32 F  → 0°C
  node bin/temp-convert.js 0 C   → 32°F
```

Claude Code creates:

```
Creating bin/temp-convert.js...
✓ Created bin/temp-convert.js

Here's the CLI:

───────────────────────────────────────────────────────────
bin/temp-convert.js
───────────────────────────────────────────────────────────

#!/usr/bin/env node

import { celsiusToFahrenheit, fahrenheitToCelsius } from '../lib/converter.js';

const [temperature, unit] = process.argv.slice(2);

if (!temperature || !unit) {
  console.log('Usage: temp-convert <temperature> <C|F>');
  process.exit(1);
}

const temp = parseFloat(temperature);

if (isNaN(temp)) {
  console.log('Error: Temperature must be a number');
  process.exit(1);
}

if (unit.toUpperCase() === 'C') {
  const result = celsiusToFahrenheit(temp);
  console.log(`${temp}°C = ${result.toFixed(1)}°F`);
} else if (unit.toUpperCase() === 'F') {
  const result = fahrenheitToCelsius(temp);
  console.log(`${temp}°F = ${result.toFixed(1)}°C`);
} else {
  console.log('Error: Unit must be C or F');
  process.exit(1);
}

───────────────────────────────────────────────────────────

✓ Created bin/temp-convert.js

The CLI is ready. Try it:
  node bin/temp-convert.js 32 F
  node bin/temp-convert.js 100 C
```

Perfect! Claude Code is done. Now test it yourself.

## Step 8: Test the CLI

In your terminal (still in the project directory, but you can exit Claude Code first):

```bash
node bin/temp-convert.js 32 F
```

You should see:

```
32°F = 0°C
```

Try another:

```bash
node bin/temp-convert.js 100 C
```

You should see:

```
100°C = 212°F
```

Excellent! The tool works. You asked Claude Code to build it, reviewed the code, and verified it works.
