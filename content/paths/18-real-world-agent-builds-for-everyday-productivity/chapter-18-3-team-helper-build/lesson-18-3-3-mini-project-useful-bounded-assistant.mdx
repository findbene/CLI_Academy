# Mini project: useful bounded assistant

## Objective
Deliver a full design -> build -> pilot -> document cycle for one useful bounded assistant, including:
- running assistant,
- CLAUDE.md contract,
- evidence log,
- stop rules,
- short case study.

## Walkthrough
### Step 1: Define project scope and constraints
Create a bounded mission statement.

```bash
mkdir -p mini-assistant-project
cd mini-assistant-project
cat > project-brief.md <<'EOF'
# Useful Bounded Assistant
- User problem:
- One workflow only:
- Inputs:
- Output artifact:
- Out of scope:
EOF
```

```powershell
New-Item -ItemType Directory -Path mini-assistant-project -Force | Out-Null
Set-Location mini-assistant-project
@"
# Useful Bounded Assistant
- User problem:
- One workflow only:
- Inputs:
- Output artifact:
- Out of scope:
"@ | Set-Content project-brief.md
```

### Step 2: Implement CLAUDE.md, stop rules, and evidence logging

```bash
touch CLAUDE.md stop-rules.md agent-log.md
```

```powershell
New-Item CLAUDE.md, stop-rules.md, agent-log.md -ItemType File -Force | Out-Null
```

Minimum required controls:
- max iterations
- max files processed
- allowed file glob patterns
- explicit forbidden actions (delete/commit/push/install)
- log-every-action requirement

### Step 3: Build and run the assistant
Prompt Claude Code:

```text
Read CLAUDE.md and execute the bounded workflow only. Apply stop-rules.md on every step. Log all actions to agent-log.md.
```

Generate one concrete output artifact, for example:
- `daily-brief-YYYY-MM-DD.md`
- `team-weekly-brief-YYYY-MM-DD.md`
- `intake-triage-YYYY-MM-DD.md`

### Step 4: Pilot with at least one other human
Collect lightweight pilot feedback into `pilot-feedback.md`:
- Was output useful?
- What failed?
- What should change?

### Step 5: Publish a short case study
Create `case-study.md` with:
- Problem
- Approach
- Safety controls used
- Result
- Next iteration plan

Final check:

```bash
ls
```

```powershell
Get-ChildItem
```

You should have all five deliverables in one folder.

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Assistant solves too many problems poorly | Scope not bounded to one workflow | Remove extra objectives from project-brief.md |
| Pilot feedback cannot guide iteration | Missing before/after evidence | Require evidence log entries tied to outcomes |
| Agent attempts risky actions | Forbidden list absent or weak | Explicitly forbid delete, push, install, permission escalation |
| Case study is opinion-only | No measurable result captured | Add concrete output count, quality rating, or time saved |
| Team cannot reuse your assistant | Contract and run steps unclear | Document exact run prompt and required files |
