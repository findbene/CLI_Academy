# Mini project: useful bounded assistant

## Objective
Deliver a full design -> build -> pilot -> document cycle for one useful bounded assistant, including:
- running assistant,
- CLAUDE.md contract,
- evidence log,
- stop rules,
- short case study.

## Walkthrough

### Step 2: Implement CLAUDE.md, stop rules, and evidence logging

<StepMeta purpose="Lay down the core safety and observability files before the assistant is allowed to act." action="Create `CLAUDE.md`, `stop-rules.md`, and `agent-log.md`, then make sure the minimum controls listed in the lesson are reflected in those files." expectedResult="The mini-project now has a contract, explicit stop limits, and an evidence log ready for use." whyItMatters="Build quality improves when safety controls are part of the assistant from the start rather than retrofitted after a messy first run." nextStep="Run the assistant next using the bounded workflow prompt and enforce the stop rules on every step." askTutor="Audit this mini-project control pack and tell me whether the contract, stop rules, and evidence log are enough to keep a first pilot safe." />

```bash
touch CLAUDE.md stop-rules.md agent-log.md
```

```powershell
New-Item CLAUDE.md, stop-rules.md, agent-log.md -ItemType File -Force | Out-Null
```

Minimum required controls:
- max iterations
- max files processed
- allowed file glob patterns
- explicit forbidden actions (delete/commit/push/install)
- log-every-action requirement

### Step 3: Build and run the assistant

<StepMeta purpose="Execute the bounded assistant end to end and produce one concrete artifact under the declared control set." action="Use the provided Claude Code prompt, enforce `stop-rules.md` during execution, log actions to `agent-log.md`, and generate one output artifact such as a daily brief or intake triage file." expectedResult="The project produces one real assistant output plus evidence of how the run happened." whyItMatters="A mini-project only proves capability when the assistant actually runs under the same constraints you claim it follows." nextStep="Pilot the assistant next with at least one other human so you can collect feedback on usefulness and failure points." askTutor="Help me review this assistant run and tell me whether the output artifact and evidence log show that the bounded workflow really executed as intended." />
Prompt Claude Code:

```text
Read CLAUDE.md and execute the bounded workflow only. Apply stop-rules.md on every step. Log all actions to agent-log.md.
```

Generate one concrete output artifact, for example:
- `daily-brief-YYYY-MM-DD.md`
- `team-weekly-brief-YYYY-MM-DD.md`
- `intake-triage-YYYY-MM-DD.md`

### Step 4: Pilot with at least one other human

<StepMeta purpose="Test the assistant with a second human so usefulness and failure modes are evaluated beyond the builder's own perspective." action="Collect pilot feedback into `pilot-feedback.md`, recording what was useful, what failed, and what should change." expectedResult="The mini-project now contains lightweight human feedback from at least one pilot participant." whyItMatters="Assistants often look better to their creators than to actual users, so even a small pilot exposes trust and usability gaps fast." nextStep="Publish the case study next so the project captures problem, approach, safety controls, results, and next iteration plan in one summary artifact." askTutor="Review this pilot-feedback approach and tell me whether it is enough to generate a credible improvement list from a small human pilot." />
Collect lightweight pilot feedback into `pilot-feedback.md`:
- Was output useful?
- What failed?
- What should change?

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Assistant solves too many problems poorly | Scope not bounded to one workflow | Remove extra objectives from project-brief.md |
| Pilot feedback cannot guide iteration | Missing before/after evidence | Require evidence log entries tied to outcomes |
| Agent attempts risky actions | Forbidden list absent or weak | Explicitly forbid delete, push, install, permission escalation |
| Case study is opinion-only | No measurable result captured | Add concrete output count, quality rating, or time saved |
| Team cannot reuse your assistant | Contract and run steps unclear | Document exact run prompt and required files |
