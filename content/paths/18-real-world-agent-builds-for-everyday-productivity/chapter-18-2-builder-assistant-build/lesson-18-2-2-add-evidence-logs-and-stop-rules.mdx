# Add evidence, logs, and stop rules

## Objective
Add a mandatory `agent-log.md` evidence trail and strict stop rules (max iterations, max file count, allowed glob scope) so your assistant cannot run away or silently overreach.

## Walkthrough
### Step 1: Create evidence and rule files

<StepMeta purpose="Add the two files that will hold hard execution limits and durable evidence for every helper run." action="Create `agent-log.md` and `stop-rules.md` so the safety pack exists before you change any contract text." expectedResult="The workspace now contains separate files for evidence logging and stop-rule enforcement." whyItMatters="Operational safeguards are easier to review and reuse when they live in dedicated artifacts instead of hidden lines inside a larger contract." nextStep="Populate the stop-rules file next with explicit, non-negotiable limits for iterations, file counts, paths, and repeated errors." askTutor="Review this safety-pack setup and tell me whether splitting evidence and rules into separate files is enough or whether another control artifact is missing." />

```bash
touch agent-log.md stop-rules.md
```

```powershell
New-Item agent-log.md -ItemType File -Force | Out-Null
New-Item stop-rules.md -ItemType File -Force | Out-Null
```

### Step 2: Define non-negotiable stop rules

<StepMeta purpose="Turn vague safety intentions into explicit machine-checkable limits that the assistant must respect every run." action="Write the five stop rules into `stop-rules.md`, covering max iterations, max files, allowed paths, repeated command errors, and missing approval." expectedResult="You have a concrete stop-rule set that can be referenced before each assistant action." whyItMatters="Assistants overreach when limits are implied rather than enumerated, especially during long or ambiguous runs." nextStep="Define the evidence-log schema next so every action can be recorded against those same rules." askTutor="Audit these stop rules and tell me whether any obvious runaway or overreach case is still unaccounted for." />

```bash
cat > stop-rules.md <<'EOF'
# Stop Rules

1. Max iterations per run: 3
2. Max files processed: 20
3. Allowed paths only: content/drafts/**/*.md, *.md at repo root
4. If command errors twice in a row: stop
5. If user approval is required and not granted: stop
EOF
```

```powershell
@"
# Stop Rules

1. Max iterations per run: 3
2. Max files processed: 20
3. Allowed paths only: content/drafts/**/*.md, *.md at repo root
4. If command errors twice in a row: stop
5. If user approval is required and not granted: stop
"@ | Set-Content stop-rules.md
```

### Step 3: Define evidence log schema

<StepMeta purpose="Create a consistent logging format that ties each assistant run to concrete actions, files, and stop-rule outcomes." action="Add the markdown table header to `agent-log.md` so every future run has a fixed schema for timestamp, run ID, task, action, files, result, and triggers." expectedResult="`agent-log.md` is ready to record comparable evidence across multiple assistant executions." whyItMatters="Structured logs are what turn assistant behavior into something you can inspect, troubleshoot, and improve rather than just remember vaguely." nextStep="Update `CLAUDE.md` next so logging and stop-rule checks become mandatory parts of execution." askTutor="Review this evidence-log schema and tell me whether it captures enough information to explain what the assistant did and why it stopped." />
Add this format to `agent-log.md`:

```md
# Agent Evidence Log

| Timestamp | Run ID | Task | Command/Action | Files Touched | Result | Stop Rule Triggered |
| --- | --- | --- | --- | --- | --- | --- |
```

### Step 4: Update CLAUDE.md to enforce logging and stop rules

<StepMeta purpose="Bind the safety pack into the main assistant contract so the rules are actually applied at runtime." action="Add the logging-and-stop-enforcement section to `CLAUDE.md`, requiring checks before each step and immediate halt on any triggered rule." expectedResult="The assistant contract now formally requires stop-rule checks and action logging during every run." whyItMatters="Safety files are only useful when the assistant is explicitly instructed to consult and obey them." nextStep="Run one helper task next and confirm the assistant writes a real evidence entry using the schema you defined." askTutor="Check this contract update and tell me whether logging and stop-rule enforcement are phrased strongly enough to be operational rather than optional." />
Add this section:

```md
## Logging and stop enforcement
- Every action must be logged to agent-log.md.
- Before each step, check stop-rules.md.
- If any stop rule is reached, halt immediately and report why.
```

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| No evidence after run | Logging not mandatory in contract | Add explicit "log every action" requirement |
| Agent processes too many files | Max file rule missing or ignored | Add `max files: 20` and require pre-check each step |
| Agent loops repeatedly | No iteration cap | Enforce `max iterations: 3` |
| Actions outside expected folders | Missing glob/path allowlist | Restrict to explicit patterns in stop-rules.md |
| Cost overrun on long runs | No hard stop on retries | Stop after two consecutive command errors |
