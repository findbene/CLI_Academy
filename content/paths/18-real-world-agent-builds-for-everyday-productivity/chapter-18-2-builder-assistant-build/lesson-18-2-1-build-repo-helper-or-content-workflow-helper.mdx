# Build: repo helper or content workflow helper

## What you will build
A repo helper or content helper run that produced a saved helper-output.md artifact.

## Why this matters
Real-world agent builds are the proof that everything in the curriculum works in practice, not just in isolated exercises. The everyday productivity scenarios in this path mirror the first agentic projects most developers build when they join an organization using Claude Code.

## Before you start
Build one practical helper assistant that answers real operational questions in your workspace:
- Repo helper: "What changed in the last week?" and "What files have TODOs?"
- Content helper: summarize draft files from a bounded folder.


## Walkthrough
### Step 1: Choose your helper scope and define a contract
Create `CLAUDE.md` rules for one helper mode only.

```md
# Helper Contract

## Mode A: Repo helper
- Answer only:
  1) What changed in the last week?
  2) What files have TODO markers?
- Allowed commands: git log --since="7 days ago" --oneline, grep -R "TODO" .

## Mode B: Content helper
- Read drafts only from content/drafts/.
- Summarize each draft in 3 bullets.

## Shared safety
- Do not edit files.
- Do not commit/push.
- Stop after processing max 20 files.
```

### Step 2: Run the repo helper path

```bash
git log --since="7 days ago" --oneline
grep -R --line-number "TODO" .
```

```powershell
git log --since="7 days ago" --oneline
Get-ChildItem -Recurse -File | Select-String -Pattern "TODO"
```

Then in Claude Code prompt:

```text
Read CLAUDE.md and run Mode A only. Return: (1) one-paragraph summary of last-week changes and (2) a TODO file list with paths.
```

### Step 3: Run the content helper path
Create draft inputs first.

```bash
mkdir -p content/drafts
printf "Draft 1\nThis draft covers onboarding improvements.\n" > content/drafts/draft-1.md
printf "Draft 2\nThis draft covers release notes quality.\n" > content/drafts/draft-2.md
```

```powershell
New-Item -ItemType Directory -Path content/drafts -Force | Out-Null
@"
Draft 1
This draft covers onboarding improvements.
"@ | Set-Content content/drafts/draft-1.md
@"
Draft 2
This draft covers release notes quality.
"@ | Set-Content content/drafts/draft-2.md
```

Prompt Claude Code:

```text
Read CLAUDE.md and run Mode B only. Summarize every file in content/drafts/ into 3 bullets and stop at 20 files max.
```

### Step 4: Record output artifact
Save the assistant response to `helper-output.md`.

```bash
touch helper-output.md
```

```powershell
New-Item helper-output.md -ItemType File -Force | Out-Null
```

Paste your assistant output into that file.

### Step 5: Validate boundaries

```bash
git status --short
```

```powershell
git status --short
```

Your helper should produce useful read-only analysis without changing project code.

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Helper scans entire machine | Missing path bounds | Constrain to repo root or `content/drafts/` explicitly |
| Command fails on Windows (`grep`) | Platform mismatch | Use `Select-String` alternative |
| Too many files processed | No max-file stop rule | Enforce `max 20 files` in contract |
| Output too generic | Prompt asks for summary without structure | Require exact output schema (paragraph + list) |
| Hidden cost creep from repeated broad scans | Re-running recursive scans in large repos | Keep scope narrow and rerun only changed directories |
