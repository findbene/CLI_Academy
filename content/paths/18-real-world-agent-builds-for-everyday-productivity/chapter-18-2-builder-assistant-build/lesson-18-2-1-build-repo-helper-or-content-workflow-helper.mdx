# Build: repo helper or content workflow helper

## Objective
Build one practical helper assistant that answers real operational questions in your workspace:
- Repo helper: "What changed in the last week?" and "What files have TODOs?"
- Content helper: summarize draft files from a bounded folder.

## Walkthrough
### Step 1: Choose your helper scope and define a contract

<StepMeta purpose="Force the helper to solve exactly one bounded operational problem instead of becoming a vague general-purpose assistant." action="Pick either Mode A or Mode B, then write `CLAUDE.md` so it clearly defines the allowed questions, allowed paths, and shared safety rules." expectedResult="You have a helper contract that supports one narrow mode at a time and explicitly forbids edits, commits, and broad scans." whyItMatters="Most helper assistants become expensive or unsafe when they try to answer too many questions across too much of the repo at once." nextStep="Run the repo-helper path next if you chose Mode A so the contract can be tested against actual repo history and TODO markers." askTutor="Review this helper contract and tell me whether the chosen mode is genuinely narrow enough for a safe first version or if I am still blending too many responsibilities." />
Create `CLAUDE.md` rules for one helper mode only.

```md
# Helper Contract

## Mode A: Repo helper
- Answer only:
  1) What changed in the last week?
  2) What files have TODO markers?
- Allowed commands: git log --since="7 days ago" --oneline, grep -R "TODO" .

## Mode B: Content helper
- Read drafts only from content/drafts/.
- Summarize each draft in 3 bullets.

## Shared safety
- Do not edit files.
- Do not commit/push.
- Stop after processing max 20 files.
```

### Step 2: Run the repo helper path

<StepMeta purpose="Test a read-only repo helper that answers concrete maintenance questions from bounded command output." action="Run the last-week git log and TODO search commands, then use the provided Claude Code prompt to produce the paragraph summary and path list." expectedResult="The helper returns a concise summary of recent changes plus a TODO marker file list without editing the repo." whyItMatters="Useful assistants often start as structured read-only summarizers, which are easier to trust and validate than write-capable workflows." nextStep="Run the content-helper path next so you can compare a second bounded helper mode against a different input surface." askTutor="Help me review this repo-helper run and tell me whether the output stays grounded in the command evidence instead of becoming generic filler." />

```bash

grep -R --line-number "TODO" .
```

```powershell
git log --since="7 days ago" --oneline
Get-ChildItem -Recurse -File | Select-String -Pattern "TODO"
```

Then in Claude Code prompt:

```text

```

### Step 3: Run the content helper path

<StepMeta purpose="Practice a second helper pattern that stays tightly scoped to one known content folder and a fixed summary format." action="Create the draft files under `content/drafts/`, then prompt Claude Code to summarize each file into three bullets with the 20-file cap enforced." expectedResult="The helper produces bounded summaries from only the draft files inside the allowed folder." whyItMatters="Path limits and fixed output structure are what keep content helpers efficient instead of drifting into broad editorial or repo-wide behavior." nextStep="Record the helper output next so the run leaves behind an artifact you can review and compare later." askTutor="Check this content-helper setup and tell me whether the scope and output format are constrained enough to keep the run predictable and low-cost." />
Create draft inputs first.

```bash
mkdir -p content/drafts
printf "Draft 1\nThis draft covers onboarding improvements.\n" > content/drafts/draft-1.md
printf "Draft 2\nThis draft covers release notes quality.\n" > content/drafts/draft-2.md
```

```powershell
New-Item -ItemType Directory -Path content/drafts -Force | Out-Null
@"
Draft 1
This draft covers onboarding improvements.
"@ | Set-Content content/drafts/draft-1.md
@"
Draft 2
This draft covers release notes quality.
"@ | Set-Content content/drafts/draft-2.md
```

Prompt Claude Code:

```text
Read CLAUDE.md and run Mode B only. Summarize every file in content/drafts/ into 3 bullets and stop at 20 files max.
```

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Helper scans entire machine | Missing path bounds | Constrain to repo root or `content/drafts/` explicitly |
| Command fails on Windows (`grep`) | Platform mismatch | Use `Select-String` alternative |
| Too many files processed | No max-file stop rule | Enforce `max 20 files` in contract |
| Output too generic | Prompt asks for summary without structure | Require exact output schema (paragraph + list) |
| Hidden cost creep from repeated broad scans | Re-running recursive scans in large repos | Keep scope narrow and rerun only changed directories |
