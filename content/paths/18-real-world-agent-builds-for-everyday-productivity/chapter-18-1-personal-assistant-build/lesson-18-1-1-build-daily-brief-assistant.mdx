# Build: daily brief assistant

## Objective
Build a real CLAUDE.md-driven assistant that creates a morning summary file named `daily-brief-YYYY-MM-DD.md` by reading yesterday's git activity and your `TODO.md`.

## Walkthrough
### Step 1: Create a bounded workspace and starter files

<StepMeta purpose="Start the assistant in a small, isolated repo so its job stays clear and its file access stays bounded." action="Create the `daily-brief-agent` folder, initialize git, and seed a minimal `TODO.md` with just the starter priorities the assistant will read." expectedResult="You have a dedicated workspace with version control and one simple input file for the assistant." whyItMatters="Bounding the workspace is the first safety control because it keeps the assistant from wandering across unrelated repositories or files." nextStep="Define the assistant contract in `CLAUDE.md` next so the workspace has explicit rules before any automation runs." askTutor="Review this starter workspace with me and tell me whether the boundary is tight enough for a first daily brief assistant or whether I am still giving it too much room to roam." />
Use a dedicated folder so your assistant never runs across unrelated repos.

```bash
mkdir -p daily-brief-agent
cd daily-brief-agent
git init
printf "- [ ] Review yesterday's commits\n- [ ] Top 3 priorities today\n" > TODO.md
```

```powershell
New-Item -ItemType Directory -Path daily-brief-agent -Force | Out-Null
Set-Location daily-brief-agent
git init
@"
- [ ] Review yesterday's commits
- [ ] Top 3 priorities today
"@ | Set-Content TODO.md
```

### Step 2: Add the assistant contract in CLAUDE.md

<StepMeta purpose="Write the exact operating contract that constrains the assistant to one output, one task, and a narrow command set." action="Create `CLAUDE.md` with purpose, allowed commands, must-do steps, forbidden actions, and stop conditions exactly as the lesson describes." expectedResult="The assistant has a written contract that limits it to generating a daily brief file and nothing else." whyItMatters="A useful assistant is not just capable; it is predictable, and prediction comes from explicit contracts rather than implied intent." nextStep="Seed one commit next so the brief has real git activity to summarize when you run the assistant." askTutor="Audit this `CLAUDE.md` contract and tell me whether the purpose, allowed commands, and stop conditions are strong enough to prevent side effects." />
This contract keeps the agent focused on one task: produce a daily brief, no side effects.

```bash
cat > CLAUDE.md <<'EOF'
# Daily Brief Assistant Contract

## Purpose
Create one file named daily-brief-YYYY-MM-DD.md from yesterday's git log and TODO.md.

## Allowed commands
- git log --since="yesterday" --oneline
- cat TODO.md
- date

## Must do
1. Read git log for yesterday.
2. Read TODO.md.
3. Create daily-brief-YYYY-MM-DD.md with: Summary, Yesterday's Changes, Today's Priorities, Risks.

## Must not do
- Do not run any write command except creating the brief file.
- Do not stage, commit, push, delete, or rename files.
- Do not install packages.

## Stop conditions
- If TODO.md is missing, stop and report the missing file.
- If git log fails, stop and show the error.
EOF
```

```powershell
@"
# Daily Brief Assistant Contract

## Purpose
Create one file named daily-brief-YYYY-MM-DD.md from yesterday's git log and TODO.md.

## Allowed commands
- git log --since=""yesterday"" --oneline
- Get-Content TODO.md
- Get-Date

## Must do
1. Read git log for yesterday.
2. Read TODO.md.
3. Create daily-brief-YYYY-MM-DD.md with: Summary, Yesterday's Changes, Today's Priorities, Risks.

## Must not do
- Do not run any write command except creating the brief file.
- Do not stage, commit, push, delete, or rename files.
- Do not install packages.

## Stop conditions
- If TODO.md is missing, stop and report the missing file.
- If git log fails, stop and show the error.
"@ | Set-Content CLAUDE.md
```

### What can go wrong

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| No brief file created | Prompt ignored CLAUDE.md contract | Re-run with explicit instruction to follow CLAUDE.md first |
| Agent touched extra files | Contract too vague on write limits | Add stricter "must not do" rules and rerun |
| Empty "Yesterday's Changes" section | No commits in timeframe | Use `--since="2 days ago"` for testing, then revert |
| Too much output or token/cost spike | Prompt asked for unnecessary analysis | Ask only for the four required sections and one file |
| Agent suggests commit/push | Over-permission in instructions | Keep git write actions explicitly forbidden in CLAUDE.md |
