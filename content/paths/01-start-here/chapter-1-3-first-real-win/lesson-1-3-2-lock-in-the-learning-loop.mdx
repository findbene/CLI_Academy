# Lock in the learning loop

## Objective

You will create a `learning-log.md` file, write your first dated entry describing what you built in Path 01, and establish a concrete ritual you will repeat after every future lesson. By the end you will have a running log that proves your progress over time.

## Walkthrough

### Step 1 — Navigate to your notes folder

**macOS / Linux:**
```bash
cd ~/cli-academy/notes
```

**Windows PowerShell:**
```powershell
Set-Location ~/cli-academy/notes
```

### Step 2 — Create learning-log.md with your first entry

Write a real log entry — not a template. Describe what you actually did in Path 01.

**macOS / Linux:**
```bash
cat <<'EOF' > learning-log.md
# CLI Academy Learning Log

## Entry 1 — $(date +%Y-%m-%d)

### What I did today
- Created the cli-academy/ workspace with notes/, projects/, and screenshots/ folders
- Captured my machine baseline (OS, shell, Node, editor)
- Wrote a safety checklist with personal stop rules
- Practiced Ctrl+C, git checkout, and mv to stop/undo/recover
- Built my first project: a README.md written entirely from the terminal

### What surprised me
(Write one thing that was different from what you expected)

### What I want to learn next
(Write one specific thing — for example: "How to use Claude Code to generate a file for me")

### Lesson rating
- Difficulty: easy / medium / hard
- Confidence after finishing: low / medium / high
EOF
```

**Windows PowerShell:**
```powershell
@"
# CLI Academy Learning Log

## Entry 1 — $(Get-Date -Format yyyy-MM-dd)

### What I did today
- Created the cli-academy/ workspace with notes/, projects/, and screenshots/ folders
- Captured my machine baseline (OS, shell, Node, editor)
- Wrote a safety checklist with personal stop rules
- Practiced Ctrl+C, git checkout, and Move-Item to stop/undo/recover
- Built my first project: a README.md written entirely from the terminal

### What surprised me
(Write one thing that was different from what you expected)

### What I want to learn next
(Write one specific thing — for example: "How to use Claude Code to generate a file for me")

### Lesson rating
- Difficulty: easy / medium / hard
- Confidence after finishing: low / medium / high
"@ | Set-Content learning-log.md
```

### Step 3 — Fill in the personal sections

Open the file in your editor:

**Any OS:**
```bash
code learning-log.md
```

Or open it in Notepad (Windows) / TextEdit (macOS) and replace the two placeholder lines:
- **"What surprised me"** — write one honest sentence
- **"What I want to learn next"** — write one specific thing
- **"Lesson rating"** — delete the options you did not pick, leaving one for each

Save the file.

### Step 4 — Read the completed entry

**macOS / Linux:**
```bash
cat learning-log.md
```

**Windows PowerShell:**
```powershell
Get-Content learning-log.md
```

You should see a complete dated entry with your personal notes filled in.

### Step 5 — Understand the ritual

After every CLI Academy lesson going forward, you will:

1. Open `learning-log.md`
2. Add a new `## Entry N — YYYY-MM-DD` section at the bottom
3. Write what you did, what surprised you, and what you want next
4. Rate the lesson difficulty and your confidence

This takes about two minutes and builds a personal record of everything you learn. When you look back after 10 lessons, you will be amazed at how far you have come.

### Step 6 — Commit your log

```bash
cd ~/cli-academy
git add notes/learning-log.md
git commit -m "Start learning log with Path 01 entry"
```

(Same commands on macOS, Linux, and Windows PowerShell.)

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| Placeholder text is still in the file | You did not edit the file in Step 3 | Open it in your editor and replace the parenthetical prompts with real sentences |
| `code: command not found` | VS Code is not on your PATH | Open the file with any text editor — Notepad, TextEdit, nano, vim all work |
| `git commit` says "nothing to commit" | You already committed this file | This is fine — your work is saved |
| Date shows the wrong format | Different locale settings | The date is just a label — any readable date format is acceptable |
| You forgot what you did in earlier lessons | Normal — this is exactly why the log exists | Scroll up through Path 01 lessons and jot down one bullet per lesson |
