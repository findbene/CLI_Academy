# Make a personal safety checklist

## Objective

You will create a `safety-checklist.md` file that lists the rules you follow before saying "yes" to any AI agent command. Then you will practice one safety habit — running `git status` — so the checklist is not just words on a page.

### Step 2 — Create safety-checklist.md

<StepMeta purpose="Write down explicit rules you can use before approving AI-suggested commands." action="Create safety-checklist.md from the lesson template in your terminal." expectedResult="A markdown file is created with before-you-say-yes checks, stop rules, and recovery steps." whyItMatters="A written checklist turns vague caution into a repeatable safety routine you can apply under pressure." nextStep="After writing the file, read it back and make sure the rules sound like ones you will actually follow." askTutor="I'm on Step 2 of Make a personal safety checklist. Explain this file-creation command and tell me how to verify the checklist content actually landed in the file." />

Write the checklist directly from the terminal.

**macOS / Linux:**
```bash
cat <<'EOF' > safety-checklist.md
# My AI Safety Checklist

## Before I say "Yes" to any AI command
- [ ] I can describe in plain English what the command will do
- [ ] The command will not delete files I have not backed up
- [ ] The command will not publish or upload anything without my review
- [ ] The command does not contain passwords, API keys, or secrets in plain text
- [ ] If I am unsure, I will ask the AI to explain the command before I approve it

## Stop rules
- If the terminal is printing nonstop text I did not expect: press Ctrl+C immediately
- If an AI wants to run rm, del, or any "remove" command outside my project folder: say No
- If an AI asks for sudo or Administrator permission and I do not understand why: say No

## Recovery steps
- Undo unsaved file changes: Ctrl+Z in my editor
- Undo an uncommitted file change: git restore filename
- Get help: open the CLI Academy Tutor and describe what happened
EOF
```

**Windows PowerShell:**
```powershell
@"
```

### Step 4 — Practice a real safety habit: git status

<StepMeta purpose="Practice the safest Git command before you ever trust yourself or an AI with a commit." action="Initialize Git in cli-academy and run git status." expectedResult="Git reports the repo state and shows untracked files like notes/." whyItMatters="git status is the fastest way to see what changed before you save or publish anything." nextStep="Once git status works, personalize the checklist with one extra rule that matches your real life." askTutor="I'm on Step 4 of Make a personal safety checklist. Help me understand the git status output I'm seeing and explain what a beginner should pay attention to before moving on." />

In later lessons you will use Git to track your work. The single most important safety command in Git is `git status` — it shows you exactly what has changed before you commit anything.

Let's try it now. First, initialize a Git repository in your workspace.

**macOS / Linux:**
```bash
cd ~/cli-academy
git init
git status
```

**Windows PowerShell:**
```powershell
Set-Location ~/cli-academy
git init
git status
```

You should see output similar to:

```
On branch main
Untracked files:
  (use "git add <file>..." to include in what will be committed)
        notes/
```

This tells you Git is watching your workspace and nothing has been committed yet. Running `git status` before every commit is a safety habit you will use in every future lesson.

> **If `git` is not installed:** You will see `command not found` or `not recognized`. That is okay — install Git from https://git-scm.com and retry. Path 6 covers Git in depth.

<StepMeta purpose="Make the checklist personal enough that you will keep using it after this lesson ends." action="Open safety-checklist.md in an editor and add one real rule you want to follow." expectedResult="The checklist now includes at least one rule that reflects your own habits, limits, or risk tolerance." whyItMatters="Personal rules are easier to remember and more likely to protect you when you are tired, rushed, or unsure." nextStep="After saving the extra rule, re-read the checklist so the final version is the one you leave behind in notes." askTutor="I'm on Step 5 of Make a personal safety checklist. Open `safety-checklist.md` in any text editor (VS Code, Notepad, TextEdit) and add one personal rule at the bottom of the "Before I say \"Yes\"" section. It can be anything, for example:

```
- [ ] I will not run commands after midnight when I am too tired to read them
```

Save the file.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `git: command not found` / `git is not recognized` | Git is not installed | Download it from https://git-scm.com — you can still finish this lesson by skipping Step 4 and returning later |
| `fatal: not a git repository` | You ran `git status` outside a folder that has been initialized | Run `git init` first in `~/cli-academy`, then retry |
| Here-string error in PowerShell | Something follows `@"` on the same line, or `"@` has leading spaces | `@"` must be the last thing on its line, and `"@` must be the first thing on its line |
| File is empty | Heredoc `EOF` line was indented on macOS/Linux | Make sure `EOF` starts at column 1 with no leading spaces |
