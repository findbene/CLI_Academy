# Learn the stop, undo, recover loop

## What you will build
A test-undo.txt file restored to its original content in cli-academy/projects/ plus an undo-recover.md playbook in cli-academy/notes/.

## Why this matters
Professional developers spend the first minutes of any new project setting up a clean workspace. Starting from an organized structure prevents lost files, confusing paths, and the embarrassment of committing secrets. This habit — workspace first, then work — is the foundation of every reliable agentic workflow.

## Before you start
You will practice three emergency skills on your own machine: stopping a runaway command with `Ctrl+C`, undoing a file change with `git checkout`, and recovering a moved file with `mv`. After this lesson, you will have muscle memory for the three most common "oh no" moments in terminal work.


## Walkthrough

### Step 1 — Go to your projects folder

**macOS / Linux:**
```bash
cd ~/cli-academy/projects
```

**Windows PowerShell:**
```powershell
Set-Location ~/cli-academy/projects
```

### Step 2 — Practice STOP: Kill a running command

We will start a harmless long-running command, then kill it.

**macOS / Linux:**
```bash
ping localhost
```

**Windows PowerShell:**
```powershell
ping localhost -t
```

Your terminal will start printing lines nonstop. This is normal — `ping` runs forever until you stop it.

**Now press `Ctrl+C`.**

The command stops immediately. The terminal prints a summary and gives you your prompt back. `Ctrl+C` is your emergency brake. It works on every operating system, in every terminal, for almost every command.

> **Important:** `Ctrl+C` does not delete anything. It simply tells the running program to quit. It is always safe to press.

### Step 3 — Practice UNDO: Revert a file with Git

First, create a test file and commit it so Git has a "known good" copy.

**macOS / Linux:**
```bash
echo "This file is safe." > test-undo.txt
git add test-undo.txt
git commit -m "Add test file for undo practice"
```

**Windows PowerShell:**
```powershell
"This file is safe." | Set-Content test-undo.txt
git add test-undo.txt
git commit -m "Add test file for undo practice"
```

Now simulate a mistake — overwrite the file with bad content:

**macOS / Linux:**
```bash
echo "OOPS I BROKE EVERYTHING" > test-undo.txt
cat test-undo.txt
```

**Windows PowerShell:**
```powershell
"OOPS I BROKE EVERYTHING" | Set-Content test-undo.txt
Get-Content test-undo.txt
```

You will see the broken text. Now restore the last committed version:

```bash
git checkout -- test-undo.txt
```

(This command is the same on macOS, Linux, and Windows PowerShell.)

Read the file again:

**macOS / Linux:**
```bash
cat test-undo.txt
```

**Windows PowerShell:**
```powershell
Get-Content test-undo.txt
```

It should say `This file is safe.` again. The damage is completely reversed.

### Step 4 — Practice RECOVER: Rescue a moved file

Accidentally moving or renaming a file is common. Let's practice finding it again.

**macOS / Linux:**
```bash
mv test-undo.txt ../notes/lost-file.txt
ls test-undo.txt
```

The `ls` command will say "No such file or directory" — the file is gone from this folder. But it is not deleted. Find it:

```bash
ls ../notes/lost-file.txt
```

Move it back:

```bash
mv ../notes/lost-file.txt test-undo.txt
cat test-undo.txt
```

**Windows PowerShell:**
```powershell
Move-Item test-undo.txt ../notes/lost-file.txt
Test-Path test-undo.txt
```

`Test-Path` returns `False` — the file is not here. Find it:

```powershell
Test-Path ../notes/lost-file.txt
```

Returns `True`. Move it back:

```powershell
Move-Item ../notes/lost-file.txt test-undo.txt
Get-Content test-undo.txt
```

Your file is home again.

### Step 5 — Create your undo-recover playbook

Save what you learned as a quick-reference card.

**macOS / Linux:**
```bash
cat <<'EOF' > ../notes/undo-recover.md
# Stop, Undo, Recover Playbook

## STOP — Ctrl+C
Kill any running terminal command instantly. Safe to press anytime.

## UNDO — git checkout -- filename
Restores a file to its last committed state. Only works on files tracked by Git.

## RECOVER — mv / Move-Item
If a file was moved, not deleted. Check nearby folders. Use mv (macOS/Linux) or Move-Item (Windows) to put it back.

## Nuclear option — git restore .
Reverts ALL uncommitted changes in the current folder. Use when everything is broken and you want to go back to your last commit.
EOF
```

**Windows PowerShell:**
```powershell
@"
# Stop, Undo, Recover Playbook

## STOP — Ctrl+C
Kill any running terminal command instantly. Safe to press anytime.

## UNDO — git checkout -- filename
Restores a file to its last committed state. Only works on files tracked by Git.

## RECOVER — mv / Move-Item
If a file was moved, not deleted. Check nearby folders. Use mv (macOS/Linux) or Move-Item (Windows) to put it back.

## Nuclear option — git restore .
Reverts ALL uncommitted changes in the current folder. Use when everything is broken and you want to go back to your last commit.
"@ | Set-Content ../notes/undo-recover.md
```

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `Ctrl+C` did nothing | Your terminal window was not focused when you pressed it | Click inside the terminal window first, then press `Ctrl+C` again |
| `error: pathspec 'test-undo.txt' did not match any file(s)` | The file was never committed with `git add` and `git commit` | Go back to Step 3 and run the `git add` / `git commit` commands first |
| `fatal: not a git repository` | You are in a folder that was not initialized with `git init` | Run `cd ~/cli-academy` then `git init`, then return to `projects/` and retry |
| `mv: cannot stat 'test-undo.txt': No such file or directory` | You already moved the file in a previous attempt | Check `../notes/` for the file — it may already be there as `lost-file.txt` |
| `git checkout` is deprecated warning | Newer Git versions prefer `git restore` | Both work. You can use `git restore test-undo.txt` instead |
