# Create the CLI Academy workspace

## Objective

You will use terminal commands to create a dedicated `cli-academy/` workspace folder with three subfolders. By the end of this lesson you will have built an organized home for every project, note, and screenshot you produce in CLI Academy — entirely from the command line.

## Walkthrough

### Step 1 — Open your terminal

<StepMeta purpose="Start the lesson in the correct shell for your operating system." action="Open Terminal on macOS or Linux, or PowerShell on Windows." expectedResult="You have a terminal window open and ready to accept commands." whyItMatters="Using the right shell keeps the commands in the lesson aligned with what you see on screen." nextStep="Once the terminal is open, move to your home directory before creating anything." askTutor="I'm on Step 1 of Create the CLI Academy workspace. **macOS / Linux:** Open **Terminal** (search "Terminal" in Spotlight or your app launcher).

**Windows:** Press `Win + X` and choose **Windows PowerShell** (or **Terminal**). All Windows commands in this course use PowerShell, not Command Prompt.

### Step 2 — Navigate to your home directory

<StepMeta purpose="Anchor yourself in a safe, known location before creating new folders." action="Run the home-directory command for your OS, then print your current location." expectedResult="The terminal shows a path that points to your user home folder." whyItMatters="Starting in the wrong place is how beginners accidentally create project folders in confusing or protected locations." nextStep="Once your home path looks correct, create the workspace folders in one shot." askTutor="I'm on Step 2 of Create the CLI Academy workspace. Help me verify that the path printed in my terminal is my home directory and safe for this lesson." />

Start from a known, safe location so the workspace lands in the right place.

**macOS / Linux:**
```bash
cd ~
pwd
```

**Windows PowerShell:**
```powershell
Set-Location ~
Get-Location
```

You should see your home path printed — for example `/Users/yourname` or `C:\Users\yourname`.

### Step 3 — Create the workspace and subfolders

<StepMeta purpose="Create the permanent folder structure you will reuse across the rest of CLI Academy." action="Run the single command that creates cli-academy plus notes, projects, and screenshots." expectedResult="The command finishes without errors and the folders now exist on disk." whyItMatters="A clean, predictable folder layout makes later lessons easier to follow and easier to troubleshoot." nextStep="After creating the folders, list the tree so you can confirm each subfolder exists." askTutor="I'm on Step 3 of Create the CLI Academy workspace. Explain exactly what this folder-creation command does and tell me if it's safe to run in my current location." />

One command creates everything at once.

**macOS / Linux:**
```bash
mkdir -p cli-academy/notes cli-academy/projects cli-academy/screenshots
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Path cli-academy\notes, cli-academy\projects, cli-academy\screenshots -Force
```

What each subfolder is for:

- **notes/** — configuration snapshots, checklists, and learning logs
- **projects/** — code and files you build during lessons
- **screenshots/** — saved error messages and terminal output for troubleshooting

### Step 5 — Move into your workspace

<StepMeta purpose="Finish the lesson from inside the workspace you just created." action="Change into cli-academy and print the current path." expectedResult="The printed path ends with cli-academy." whyItMatters="Most later lessons assume you can quickly move into this workspace without guessing where it lives." nextStep="With the workspace confirmed, you are ready to start capturing notes and building small projects inside it." askTutor="I'm on Step 5 of Create the CLI Academy workspace. Help me verify that I really moved into cli-academy and explain what to do if my path still looks wrong." />

Most CLI Academy work starts from this folder, so step inside it now.

**macOS / Linux:**
```bash
cd cli-academy
pwd
```

**Windows PowerShell:**
```powershell
Set-Location cli-academy
Get-Location
```

Confirm the printed path ends with `cli-academy`.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `mkdir: cannot create directory: Permission denied` | You are in a system-protected folder | Run `cd ~` first, then retry the `mkdir` command |
| `New-Item : Access to the path is denied` | Same issue on Windows | Run `Set-Location ~` first, then retry |
| Folder already exists — no error but nothing seems to happen | You ran the command before | This is safe. `mkdir -p` (macOS/Linux) and `-Force` (Windows) silently skip folders that already exist without deleting anything |
| `command not found: mkdir` | You may be in an unusual shell | On Windows, confirm you opened **PowerShell**, not Command Prompt. On macOS/Linux, type `bash` then Enter to start a standard shell |
| Path printed by `pwd` / `Get-Location` is not what you expected | You may have a custom home directory or OneDrive redirect on Windows | Use the full printed path in future commands instead of `~` |
