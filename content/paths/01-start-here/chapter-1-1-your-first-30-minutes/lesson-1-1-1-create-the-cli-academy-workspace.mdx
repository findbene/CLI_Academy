# Create the CLI Academy workspace

## Objective

You will use terminal commands to create a dedicated `cli-academy/` workspace folder with three subfolders. By the end of this lesson you will have built an organized home for every project, note, and screenshot you produce in CLI Academy — entirely from the command line.

## Walkthrough

### Step 1 — Open your terminal

**macOS / Linux:** Open **Terminal** (search "Terminal" in Spotlight or your app launcher).

**Windows:** Press `Win + X` and choose **Windows PowerShell** (or **Terminal**). All Windows commands in this course use PowerShell, not Command Prompt.

### Step 2 — Navigate to your home directory

Start from a known, safe location so the workspace lands in the right place.

**macOS / Linux:**
```bash
cd ~
pwd
```

**Windows PowerShell:**
```powershell
Set-Location ~
Get-Location
```

You should see your home path printed — for example `/Users/yourname` or `C:\Users\yourname`.

### Step 3 — Create the workspace and subfolders

One command creates everything at once.

**macOS / Linux:**
```bash
mkdir -p cli-academy/notes cli-academy/projects cli-academy/screenshots
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Path cli-academy\notes, cli-academy\projects, cli-academy\screenshots -Force
```

What each subfolder is for:

- **notes/** — configuration snapshots, checklists, and learning logs
- **projects/** — code and files you build during lessons
- **screenshots/** — saved error messages and terminal output for troubleshooting

### Step 4 — Verify the folder tree

**macOS / Linux:**
```bash
ls cli-academy
```

Expected output:
```
notes  projects  screenshots
```

**Windows PowerShell:**
```powershell
Get-ChildItem cli-academy -Name
```

Expected output:
```
notes
projects
screenshots
```

### Step 5 — Move into your workspace

Most CLI Academy work starts from this folder, so step inside it now.

**macOS / Linux:**
```bash
cd cli-academy
pwd
```

**Windows PowerShell:**
```powershell
Set-Location cli-academy
Get-Location
```

Confirm the printed path ends with `cli-academy`.

### What can go wrong

| Problem | Cause | Fix |
|---|---|---|
| `mkdir: cannot create directory: Permission denied` | You are in a system-protected folder | Run `cd ~` first, then retry the `mkdir` command |
| `New-Item : Access to the path is denied` | Same issue on Windows | Run `Set-Location ~` first, then retry |
| Folder already exists — no error but nothing seems to happen | You ran the command before | This is safe. `mkdir -p` (macOS/Linux) and `-Force` (Windows) silently skip folders that already exist without deleting anything |
| `command not found: mkdir` | You may be in an unusual shell | On Windows, confirm you opened **PowerShell**, not Command Prompt. On macOS/Linux, type `bash` then Enter to start a standard shell |
| Path printed by `pwd` / `Get-Location` is not what you expected | You may have a custom home directory or OneDrive redirect on Windows | Use the full printed path in future commands instead of `~` |
