# Ask Claude to write your first test

## Objective

Take a simple function you have already written, ask Claude Code to generate an automated test for it, and run that test from the command line. By the end of this lesson you will have one green passing test for a function you understand, using either Jest (JavaScript) or pytest (Python).

## Walkthrough

### Step 1 — Create a small module with a function to test

<StepMeta purpose="Create a tiny known-good function so the lesson stays focused on test-writing rather than bug-fixing." action="Set up the practice folder and write the simple source module with add and multiply functions." expectedResult="You have a minimal source file on disk with behavior simple enough to test and understand completely." whyItMatters="Your first testing lesson works best when the code under test is familiar and obviously correct, because the new skill is the testing workflow itself." nextStep="Initialize the test runner for your language so Claude has a place to put the generated test file." askTutor="I'm on Step 1 of Ask Claude to write your first test. Help me keep this module small enough that I can focus on the mechanics of testing instead of the application logic." />

We need a simple, working function so we can focus entirely on learning to test — not on fixing bugs.

**macOS / Linux**

```bash
mkdir -p ~/projects/testing-practice && cd ~/projects/testing-practice
```

**Windows PowerShell**

```powershell
New-Item -ItemType Directory -Force ~\projects\testing-practice | Set-Location
```

**JavaScript version — `math-utils.js`**

```js
// math-utils.js
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, multiply };
```

**Python version — `math_utils.py`**

```python
# math_utils.py
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b
```

### Step 2 — Initialize the test runner

<StepMeta purpose="Prepare the project so automated tests can actually run from the command line." action="Install or confirm the test runner and make sure the project has the correct test command configured." expectedResult="Your environment is ready to execute Jest or pytest without extra setup after Claude writes the test file." whyItMatters="A generated test file is only useful if the repo already knows how to discover and run it." nextStep="Ask Claude to generate a basic test file for the functions you just created." askTutor="I'm on Step 2 of Ask Claude to write your first test. Help me verify that Jest or pytest is configured correctly before I ask Claude to generate tests." />

**JavaScript — install Jest**

```bash
npm init -y
npm install --save-dev jest
```

Open `package.json` and make sure the `"test"` script says `"jest"`:

```json
{
  "scripts": {
    "test": "jest"
  }
}
```

**Python — confirm pytest is installed**

```bash
pip install pytest
```

**Windows PowerShell (same commands work)**

```powershell
pip install pytest
```

> **Real danger — `npm install` and `pip install` execute code.** Only install packages you recognize and trust. `jest` and `pytest` are the standard test runners for their respective ecosystems and are safe.

### Step 3 — Ask Claude Code to write a test

<StepMeta purpose="Use Claude to generate the repetitive structure of a first test file while you focus on what cases matter." action="Prompt Claude to write tests for the simple math functions using the standard test framework for your language." expectedResult="Claude creates a discoverable test file with several straightforward assertions covering the target functions." whyItMatters="Test generation is a high-leverage use case for Claude because the structure is repetitive but still benefits from clear human intent about what to cover." nextStep="Run the generated tests from the terminal and confirm that the runner discovers them successfully." askTutor="I'm on Step 3 of Ask Claude to write your first test. Help me judge whether Claude's generated tests cover the obvious cases without adding unnecessary complexity." />

Open Claude Code in the project directory and prompt:

```
Write a test file for math-utils.js that tests the add and multiply functions with a few basic cases. Use Jest.
```

Or for Python:

```
Write a test file for math_utils.py that tests the add and multiply functions with a few basic cases. Use pytest.
```

Claude creates a test file. For JavaScript it might produce `math-utils.test.js`:

```js
const { add, multiply } = require("./math-utils");

test("add returns the sum of two positive numbers", () => {
  expect(add(2, 3)).toBe(5);
});

test("add handles zero", () => {
  expect(add(0, 7)).toBe(7);
});

test("add handles negative numbers", () => {
  expect(add(-1, -2)).toBe(-3);
});

test("multiply returns the product of two numbers", () => {
  expect(multiply(3, 4)).toBe(12);
});

test("multiply by zero returns zero", () => {
  expect(multiply(5, 0)).toBe(0);
});
```

For Python it might produce `test_math_utils.py`:

```python
from math_utils import add, multiply

def test_add_positive():
    assert add(2, 3) == 5

def test_add_zero():
    assert add(0, 7) == 7

def test_add_negative():
    assert add(-1, -2) == -3

def test_multiply():
    assert multiply(3, 4) == 12

def test_multiply_by_zero():
    assert multiply(5, 0) == 0
```

Accept the file Claude generates.

### What can go wrong

| Symptom | Cause | Fix |
|---|---|---|
| `npm test` says "no tests found" | Jest cannot find test files — wrong file name | Jest looks for files ending in `.test.js` or `.spec.js`. Rename if needed |
| `pytest` says "no tests ran" | Test functions do not start with `test_` | pytest discovers functions by the `test_` prefix — rename them |
| `Cannot find module './math-utils'` | Test file is in a different folder than the source | Move the test file to the same folder or fix the import path |
| `npm: command not found` | Node.js is not installed | Install Node.js from nodejs.org and restart your terminal |
| Jest fails with "SyntaxError: Unexpected token export" | You used ES module `export` instead of `module.exports` | Either switch to `module.exports` or add `"type": "module"` to `package.json` and configure Jest for ESM |
| Python `ModuleNotFoundError` | pytest cannot find `math_utils.py` | Make sure you run `pytest` from the same directory that contains `math_utils.py` |
