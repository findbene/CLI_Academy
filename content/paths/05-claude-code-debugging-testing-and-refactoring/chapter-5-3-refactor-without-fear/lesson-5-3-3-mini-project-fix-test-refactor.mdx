# Mini project: fix, test, refactor

## Objective

Welcome to your first mini-project! In this lesson, we'll walk through a common, real-world engineering workflow: identifying a bug, writing a test to prove its existence, fixing the bug, and then tidying up your work into a clear, professional story using Git. By the end, you'll have a clean, concise record of your bug fix, ready for review.

## Walkthrough

Let's imagine you've discovered a bug in a simple Python script called `buggy_calculator.py`. Your task is to fix it, making sure you prove the fix with a test.

1.  **Create a safe space:** First, let's create a new branch. This keeps your main codebase safe while you experiment.
    ```bash
    git switch -c fix/subtraction-bug
    ```
    *Why this is safe:* `git switch -c` creates a *new* branch and immediately moves you to it. All your changes will stay isolated on this branch until you explicitly merge them, protecting your main branch from accidental disruption.

2.  **Reproduce the bug (manually):**
    Open the file `buggy_calculator.py`. It has a function `subtract(a, b)` that incorrectly returns `a + b` instead of `a - b`.
    Run the script and try to subtract:
    ```bash
    python buggy_calculator.py
    # Follow the prompts: choose 'subtract', enter '5', then '3'.
    # You should see 'Result: 8' (which is wrong for subtraction).
    ```
    This confirms the bug!

3.  **Write a failing test:** Before fixing, let's write an automated test that *fails* because of this bug. This proves the bug exists and will ensure it stays fixed.
    Create a new file `test_calculator.py`:
    ```bash
    touch test_calculator.py
    # Add the following content to test_calculator.py:
    # from buggy_calculator import subtract
    # def test_subtract_bug():
    #     assert subtract(5, 3) == 2, "Subtraction is not working correctly!"
    ```
    Now, let's run this test to see it fail:
    ```bash
    python -m pytest test_calculator.py
    ```
    *Why this is safe:* `pytest` is a testing framework. Running tests doesn't modify your code; it just checks if your functions behave as expected. Seeing the test fail confirms we've captured the bug!

4.  **Commit the failing test:** It's a good practice to commit the failing test first.
    ```bash
    git add test_calculator.py
    git commit -m "test: Add failing test for subtraction bug"
    ```
    *Why this is safe:* This commit acts as a bookmark. Even if you mess up the next steps, you have a record of "I knew the bug was here, and I proved it with this test."

5.  **Fix the bug:**
    Edit `buggy_calculator.py` and change the `subtract` function:
    ```python
    # Change:
    # def subtract(a, b):
    #     return a + b
    # To:
    def subtract(a, b):
        return a - b
    ```

6.  **Verify the fix:** Run the test again!
    ```bash
    python -m pytest test_calculator.py
    ```
    You should see the test pass now. Great job!

7.  **Commit the fix:**
    ```bash
    git add buggy_calculator.py
    git commit -m "fix: Correct subtraction logic in calculator"
    ```

8.  **Clean up your history (for a clean engineering story):** We now have two commits (`test: Add failing test...` and `fix: Correct subtraction...`). For a single bug fix, it's often cleaner to present this as *one* logical change.
    Let's combine them using an interactive rebase:
    ```bash
    git log --oneline # Observe your two latest commits
    git rebase -i HEAD~2
    ```
    *Why this is safe:* `git rebase -i` lets you rewrite commit history *locally*. As long as you haven't pushed these commits to a shared remote, you're only affecting your own repository. You'll see an editor pop up showing your commits.
    -   Change `pick` on the second commit (`fix: Correct subtraction...`) to `squash` (or `s`).
    -   Save and close the editor.
    -   A new editor will open, letting you combine the commit messages. Delete the individual messages and write a single, clear message like:
        ```
        fix: Correct subtraction logic in calculator

        The subtract function was incorrectly adding two numbers instead of
        subtracting them. This commit fixes the bug and includes a new
        test case to ensure correct functionality.
        ```
    -   Save and close.

9.  **Final check:**
    ```bash
    git log --oneline # You should now see only one concise commit message
    git diff HEAD~1 # This shows the complete, combined change from your single commit.
    ```
    You've successfully completed the workflow! You've identified, tested, fixed, and professionally documented a bug.
