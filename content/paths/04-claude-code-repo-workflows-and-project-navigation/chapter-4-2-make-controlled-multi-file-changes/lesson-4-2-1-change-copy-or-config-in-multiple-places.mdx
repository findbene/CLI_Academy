# Change copy or config in multiple places

## Objective
By the end of this lesson, you will be able to confidently and safely update specific text or configuration values across multiple files in your project. We'll focus on a safe, step-by-step method to ensure no unintended changes or leftover old values.

## Walkthrough

Imagine you need to update an outdated version number or a common configuration setting found in several files. Let's change an example placeholder, `v1.0-alpha`, to `v1.1-beta` across your project.

1.  **Safely Locate All Occurrences:**
    First, we need to find *every* file containing the text `v1.0-alpha` without making any changes. We'll use the `grep` command, which is perfectly safe because it only *reads* files and prints what it finds.

    ```bash
    grep -r "v1.0-alpha" .
    ```

    *   `grep`: The command to search for text patterns.
    *   `-r`: (recursive) Tells `grep` to search through all subdirectories.
    *   `"v1.0-alpha"`: The exact text we're looking for.
    *   `.`: Tells `grep` to start searching in the current directory.

    Review the output carefully. This shows you exactly where the old text appears.

2.  **List Files for Change (Your Change Plan):**
    Based on the `grep` output from the previous step, identify *all* the files you intend to modify. For instance, let's say your `grep` output showed matches in:
    *   `config/settings.txt`
    *   `src/index.js`
    *   `docs/README.md`

    **Before proceeding, you now have a clear list of files that will be affected.** This is your safety checklist!

3.  **Perform the Multi-File Change (With a Safety Net):**
    Now, let's make the change. We'll use `sed` (stream editor), a powerful tool for text manipulation. To ensure safety for beginners, we'll use a feature that creates backup files before making changes.

    ```bash
    sed -i.bak 's/v1.0-alpha/v1.1-beta/g' config/settings.txt src/index.js docs/README.md
    ```

    *   `sed`: The stream editor command.
    *   `-i.bak`: This is your safety net! It tells `sed` to edit files *in-place* but *also* create a backup of each original file with a `.bak` extension (e.g., `config/settings.txt.bak`). If anything goes wrong, you can easily restore the original files.
    *   `'s/v1.0-alpha/v1.1-beta/g'`: This is the substitution command:
        *   `s`: Substitute.
        *   `v1.0-alpha`: The text to find.
        *   `v1.1-beta`: The text to replace it with.
        *   `g`: (global) Replaces *all* occurrences on each line, not just the first.
    *   `config/settings.txt src/index.js docs/README.md`: Explicitly list *all* the files you identified in step 2.

4.  **Verify No Leftovers:**
    It's crucial to confirm that all instances of the old text are gone and that only the new text remains.

    ```bash
    grep -r "v1.0-alpha" .
    ```

    This command should now return *no output*, indicating that `v1.0-alpha` has been successfully replaced everywhere. If it does return output, you've found a leftover that needs to be addressed!

    Optionally, you can also verify the new text is present:
    ```bash
    grep -r "v1.1-beta" .
    ```
    This should show the new text in all the modified files.

5.  **Clean Up (Only if Satisfied):**
    Once you are absolutely certain that your changes are correct and everything works as expected, you can remove the backup files.

    ```bash
    find . -name "*.bak" -delete
    ```

    *   `find .`: Start searching in the current directory.
    *   `-name "*.bak"`: Look for files ending with `.bak`.
    *   `-delete`: Delete them.

    This step should *only* be performed after you've thoroughly verified your changes. If there were any issues, you could use `mv file.txt.bak file.txt` to restore the originals.
