# Update docs as part of the change

## Objective

When you make changes to code, especially in a collaborative project, it's super important to also update any related documentation. This lesson will guide you through the process of recognizing when a code change requires a doc update and how to make that happen in the same commit. This keeps your project clear and easy for everyone to understand, including your future self!

## Walkthrough

1.  **Consider a code change:** Let's imagine you've just added a new command-line option to a script, like `--verbose` to show more output. While this change is in the script itself, anyone using the script will need to know about this new option.
2.  **Identify affected documentation:** Where would a user look for information about script options? Probably the `README.md` file, or perhaps a dedicated `docs/` folder. For this lesson, let's assume it's `README.md`.
3.  **Open and edit the doc file:** You can open `README.md` with your favorite text editor. On the command line, `nano README.md` is a friendly option.
    *   **Safety Note:** Using `nano` to open a text file like `README.md` is very safe. It's a text editor and won't execute any code from the file.
4.  **Make the minimal necessary update:** Find the section describing script usage or options. Add a concise line or bullet point about your new `--verbose` option. For example: `* `--verbose`: Show detailed processing information.` Remember, keep it minimal but sufficient.
5.  **Save the changes:** If using `nano`, press `Ctrl+X`, then `Y` to confirm saving, and then `Enter`.
6.  **Commit both changes together:** Now, when you go to commit your code changes, make sure to stage both the code file *and* `README.md`. For example:
    ```bash
    git add your_script.py README.md
    git commit -m "feat: Add verbose option and update README"
    ```
    Committing them together ensures that the documentation always matches the code it describes. This makes it much easier for others (and your future self!) to understand why a change was made.
