# Make your first tiny edit

## What you will build
README.md has a new Usage section added by Claude Code, committed to Git.

## Why this matters
Claude Code is the primary interface for agentic development. Understanding how it reads context, executes tools, and takes action is the difference between a developer who directs Claude precisely and one who gets unpredictable results. Every advanced agentic pattern builds on this foundation.

## Before you start
Ask Claude Code to make one small, concrete change to your practice project, review the diff it shows you, and then accept or reject it. This is the core editing loop: you stay in control and Claude Code does the typing.


## Walkthrough

### Step 1 — Start a session in your practice repo

**macOS / Linux**
```bash
cd ~/projects/cli-practice
claude
```

**Windows PowerShell**
```powershell
Set-Location "$HOME\projects\cli-practice"
claude
```

### Step 2 — Request a small, specific edit

At the Claude Code prompt, type:

```
Add a one-paragraph "Usage" section to README.md that tells someone how to run this project with npm start.
```

Claude Code reads your current `README.md`, writes a new version, and shows you a **diff** — lines prefixed with `+` are additions and lines prefixed with `-` are removals.

The diff will look something like:

```diff
 # cli-practice

 A small starter project for learning Claude Code.
+
+## Usage
+
+Make sure you have Node.js 18+ installed, then run:
+
+```
+npm start
+```
```

### Step 3 — Review the diff carefully

Before accepting, check three things:

1. **Scope** — Did it only change `README.md`? Nothing else should appear in the diff.
2. **Accuracy** — Does the new text match what the project actually does?
3. **Tone** — Is the wording clear and helpful for someone new?

If something looks wrong you can type:

```
Undo that change.
```

Claude Code reverts the file to its previous state. No harm done.

### Step 4 — Accept the change

If the diff looks good, Claude Code has already written it to disk. Confirm by reading the file outside the session:

**macOS / Linux**
```bash
cat README.md
```

**Windows PowerShell**
```powershell
Get-Content README.md
```

You should see the new Usage section in the file.

### Step 5 — Commit the edit with Git

Leave the Claude Code session with `/exit`, then commit your change:

```bash
git add README.md
git commit -m "docs: add usage section to README"
```

You just completed the ask → review → accept → commit loop for the first time.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| Claude Code changed more files than expected | The prompt was too broad (e.g., "improve the project") | Ask Claude Code to undo, then re-prompt with a specific file and change |
| The diff has factually wrong instructions | Claude Code guessed at a run command | Correct it: "The start command is `npm start`, not `node app.js`. Please fix the README." |
| You accepted a change you did not want | The file was written to disk before you caught the problem | Run `git checkout -- README.md` to restore the last committed version |
| `git commit` says "nothing to commit" | You forgot to `git add` the file | Run `git add README.md` first |
