# Open your first practice repo

## What you will build
A practice repo with starter files and a successful Claude Code launch.

## Why this matters
Claude Code is the primary interface for agentic development. Understanding how it reads context, executes tools, and takes action is the difference between a developer who directs Claude precisely and one who gets unpredictable results. Every advanced agentic pattern builds on this foundation.

## Before you start
Create a small practice project with real files, then launch Claude Code inside it so you can see the welcome screen and the interactive session prompt. By the end you will have a working repo that Claude Code can read and talk about.


## Walkthrough

### Step 1 — Create a project folder

Pick a location where you keep coding projects and create a new folder:

**macOS / Linux**
```bash
mkdir -p ~/projects/cli-practice && cd ~/projects/cli-practice
```

**Windows PowerShell**
```powershell
New-Item -ItemType Directory -Force -Path "$HOME\projects\cli-practice" | Out-Null
Set-Location "$HOME\projects\cli-practice"
```

### Step 2 — Initialize a Git repo

Claude Code works best inside a Git repository because it uses the file history for context.

```bash
git init
```

This is the same command on every platform. You should see `Initialized empty Git repository`.

### Step 3 — Add starter files

Create three files that give Claude Code something to read.

**macOS / Linux**
```bash
cat > index.js << 'EOF'
// A tiny greeter
function greet(name) {
  return `Hello, ${name}! Welcome to CLI Academy.`;
}

console.log(greet("Learner"));
EOF

cat > README.md << 'EOF'
# cli-practice

A small starter project for learning Claude Code.
EOF

cat > package.json << 'EOF'
{
  "name": "cli-practice",
  "version": "1.0.0",
  "description": "Starter project for CLI Academy Path 03",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}
EOF
```

**Windows PowerShell**
```powershell
@'
// A tiny greeter
function greet(name) {
  return `Hello, ${name}! Welcome to CLI Academy.`;
}

console.log(greet("Learner"));
'@ | Set-Content index.js

@'
# cli-practice

A small starter project for learning Claude Code.
'@ | Set-Content README.md

@'
{
  "name": "cli-practice",
  "version": "1.0.0",
  "description": "Starter project for CLI Academy Path 03",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}
'@ | Set-Content package.json
```

### Step 4 — Make an initial commit

```bash
git add -A
git commit -m "Initial commit: starter greeter project"
```

Having at least one commit lets Claude Code see your project history.

### Step 5 — Launch Claude Code

From inside the `cli-practice` folder, start a session:

```bash
claude
```

Claude Code prints a welcome banner and drops you into an interactive prompt (you will see a `>` or similar cursor). This is where you type natural-language requests — not shell commands. You are now inside a Claude Code session.

Type `/help` to see the list of built-in slash commands, then type `/exit` to leave when you are done exploring.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| `claude: command not found` | Claude Code is not installed or the terminal was not restarted after install | Follow Lesson 3.1.1 to install, then close and reopen your terminal |
| `Not a git repository` warning on launch | You ran `claude` before running `git init` | Run `git init` and make an initial commit first |
| Claude Code starts but says the project is empty | Files were created in a different folder | Run `ls` (macOS/Linux) or `Get-ChildItem` (PowerShell) to confirm `index.js`, `README.md`, and `package.json` are in the current directory |
| `git commit` says "nothing to commit" | Files were not staged with `git add` | Run `git add -A` first, then `git commit` |
