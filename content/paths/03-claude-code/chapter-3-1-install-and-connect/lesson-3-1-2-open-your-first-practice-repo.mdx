# Open your first practice repo

## Objective

Create a small practice project with real files, then launch Claude Code inside it so you can see the welcome screen and the interactive session prompt. By the end you will have a working repo that Claude Code can read and talk about.

## Walkthrough

### Step 1 — Create a project folder

<StepMeta purpose="Create a clean workspace folder dedicated to your first Claude Code practice repo." action="Make the cli-practice folder under your projects area and move into it." expectedResult="A new cli-practice directory exists and your prompt is now inside it." whyItMatters="Giving Claude Code a small isolated project makes the first session easier to understand and easier to troubleshoot." nextStep="Once the folder exists, initialize it as a Git repository so Claude Code can use repo context." askTutor="I'm on Step 1 of Open your first practice repo. Help me confirm that I created cli-practice in a sensible location and that my shell is now inside the repo folder." />

Pick a location where you keep coding projects and create a new folder:

**macOS / Linux**
```bash
mkdir -p ~/projects/cli-practice && cd ~/projects/cli-practice
```

**Windows PowerShell**
```powershell
New-Item -ItemType Directory -Force -Path "$HOME\projects\cli-practice" | Out-Null
Set-Location "$HOME\projects\cli-practice"
```

### Step 2 — Initialize a Git repo

<StepMeta purpose="Turn the plain folder into a repository Claude Code can reason about more effectively." action="Run git init inside cli-practice and confirm Git recognizes the new repository." expectedResult="Git reports that it initialized an empty repository in the current folder." whyItMatters="Claude Code gets more useful context from Git-aware projects, and later lessons depend on commit history being available." nextStep="After Git is initialized, create a few starter files so the repo contains something real to inspect." askTutor="I'm on Step 2 of Open your first practice repo. Help me verify that git init succeeded in the correct folder and explain why Claude Code prefers working inside repos." />

Claude Code works best inside a Git repository because it uses the file history for context.

```bash
git init
```

This is the same command on every platform. You should see `Initialized empty Git repository`.

### Step 3 — Add starter files

<StepMeta purpose="Give the new repo enough real content for Claude Code to summarize and edit meaningfully." action="Create index.js, README.md, and package.json with the provided starter content." expectedResult="All three files exist at the repo root with the expected starter code and metadata." whyItMatters="A repo without meaningful files gives Claude Code nothing useful to explain, edit, or run in later lessons." nextStep="Once the files exist, make an initial commit so the repo has a baseline history." askTutor="I'm on Step 3 of Open your first practice repo. Help me verify that these file-creation commands produced the right files in the repo root before I commit anything." />

Create three files that give Claude Code something to read.

**macOS / Linux**
```bash
cat > index.js << 'EOF'
// A tiny greeter
function greet(name) {
  return `Hello, ${name}! Welcome to CLI Academy.`;
}

console.log(greet("Learner"));
EOF

cat > README.md << 'EOF'
# cli-practice

A small starter project for learning Claude Code.
EOF

cat > package.json << 'EOF'
{
  "name": "cli-practice",
  "version": "1.0.0",
  "description": "Starter project for CLI Academy Path 03",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}
EOF
```

**Windows PowerShell**
```powershell
@'
// A tiny greeter
function greet(name) {
  return `Hello, ${name}! Welcome to CLI Academy.`;
}

console.log(greet("Learner"));
'@ | Set-Content index.js

@'
# cli-practice

A small starter project for learning Claude Code.
'@ | Set-Content README.md

@'
{
  "name": "cli-practice",
  "version": "1.0.0",
  "description": "Starter project for CLI Academy Path 03",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}
'@ | Set-Content package.json
```

### Step 4 — Make an initial commit

<StepMeta purpose="Create a clean baseline snapshot before Claude Code starts suggesting edits." action="Stage all files and create the initial Git commit with the provided message." expectedResult="Git records the first commit for the practice project and the working tree becomes clean." whyItMatters="A baseline commit makes diffs, undo, and later change reviews much easier to understand." nextStep="After the repo has a first commit, launch Claude Code from the project root and inspect the welcome session." askTutor="I'm on Step 4 of Open your first practice repo. Help me check that I am staging the expected files and explain what this initial commit gives me before I use Claude Code." />

```bash
git add -A
git commit -m "Initial commit: starter greeter project"
```

Having at least one commit lets Claude Code see your project history.

### Step 5 — Launch Claude Code

<StepMeta purpose="Start the first real Claude Code session inside a repo that now has both files and history." action="Run claude from the cli-practice folder, inspect the welcome prompt, try /help, then exit." expectedResult="Claude Code opens an interactive session and responds to /help before you exit cleanly." whyItMatters="This is the first proof that installation, authentication, terminal setup, and repo context are all working together." nextStep="After this step, you are ready to ask Claude Code for understanding before you let it make any edits." askTutor="I'm on Step 5 of Open your first practice repo. Help me confirm that this welcome screen means Claude Code started correctly in the repo I just prepared." />

From inside the `cli-practice` folder, start a session:

```bash
claude
```

Claude Code prints a welcome banner and drops you into an interactive prompt (you will see a `>` or similar cursor). This is where you type natural-language requests — not shell commands. You are now inside a Claude Code session.

Type `/help` to see the list of built-in slash commands, then type `/exit` to leave when you are done exploring.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| `claude: command not found` | Claude Code is not installed or the terminal was not restarted after install | Follow Lesson 3.1.1 to install, then close and reopen your terminal |
| `Not a git repository` warning on launch | You ran `claude` before running `git init` | Run `git init` and make an initial commit first |
| Claude Code starts but says the project is empty | Files were created in a different folder | Run `ls` (macOS/Linux) or `Get-ChildItem` (PowerShell) to confirm `index.js`, `README.md`, and `package.json` are in the current directory |
| `git commit` says "nothing to commit" | Files were not staged with `git add` | Run `git add -A` first, then `git commit` |
