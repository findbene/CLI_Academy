# Run the project and inspect output

## Objective

Run your practice project from the terminal, read the output, and then use Claude Code to explain anything unexpected. By the end you will know how to execute code, spot errors, and get instant help understanding what went wrong.

## Walkthrough

### Step 1 — Run the project

Make sure you are in the `cli-practice` folder and run the start script:

**macOS / Linux**
```bash
cd ~/projects/cli-practice
npm start
```

**Windows PowerShell**
```powershell
Set-Location "$HOME\projects\cli-practice"
npm start
```

Because `package.json` maps `start` to `node index.js`, you should see:

```
Hello, Learner! Welcome to CLI Academy.
```

If you see that line, your code is working. Skip to Step 3.

### Step 2 — If something went wrong, ask Claude Code

If the output contains an error (for example a `SyntaxError` or `ReferenceError`), do not try to guess the fix. Open Claude Code and paste the error:

```bash
claude
```

At the prompt, type:

```
I ran npm start and got this error:

<paste the full error message here>

What is wrong and how do I fix it?
```

Claude Code reads your `index.js`, compares it against the error, and explains the cause in plain language. It may offer to fix the file for you — review the diff before accepting, just like in Lesson 3.2.2.

After fixing, exit Claude Code with `/exit` and re-run `npm start` to confirm the error is gone.

### Step 3 — Intentionally break the project, then fix it

This step teaches you what errors look like so you are not caught off guard later.

Open `index.js` in any editor and change `console.log` to `console.logg` (extra `g`). Save the file and run:

```bash
npm start
```

You will see an error like:

```
TypeError: console.logg is not a function
```

Now launch Claude Code:

```bash
claude
```

Type:

```
I changed console.log to console.logg by accident. Fix it.
```

Claude Code shows a one-line diff restoring `console.log`. Accept the fix and run `npm start` again to see the greeting.

### Step 4 — Commit the working state

```bash
git add -A
git commit -m "fix: restore console.log after intentional break exercise"
```

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| `npm start` says `Missing script: "start"` | `package.json` does not have a `start` entry, or you are in the wrong folder | Run `cat package.json` (macOS/Linux) or `Get-Content package.json` (PowerShell) to check. Re-create the file from Lesson 3.1.2 if needed |
| `node: command not found` | Node.js is not installed or not on PATH | Install Node.js LTS and restart your terminal |
| `npm start` prints nothing and exits | `index.js` is empty or missing | Verify the file exists with `ls` / `Get-ChildItem` and re-create it from Lesson 3.1.2 |
| Claude Code fixes the file but the error persists | You are running `npm start` from a different folder than where the file was fixed | Make sure your terminal is in `cli-practice` before running |
