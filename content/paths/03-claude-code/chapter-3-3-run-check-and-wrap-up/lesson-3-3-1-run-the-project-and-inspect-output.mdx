# Run the project and inspect output

## Objective

Run your practice project from the terminal, read the output, and then use Claude Code to explain anything unexpected. By the end you will know how to execute code, spot errors, and get instant help understanding what went wrong.

## Walkthrough

### Step 1 â€” Run the project

<StepMeta purpose="Test the project in its normal execution path before inventing any troubleshooting work." action="Move into cli-practice and run npm start." expectedResult="The project prints the greeting message, or it fails with a clear error you can inspect." whyItMatters="Running the project is the only reliable way to know whether the current code actually works instead of merely looking plausible." nextStep="If you see an error instead of the greeting, open Claude Code and ask it to diagnose the failure using the real output." askTutor="I'm on Step 1 of Run the project and inspect output. Make sure you are in the `cli-practice` folder and run the start script:

**macOS / Linux**
```bash
cd ~/projects/cli-practice
npm start
```

**Windows PowerShell**
```powershell
Set-Location "$HOME\projects\cli-practice"
npm start
```

Because `package.json` maps `start` to `node index.js`, you should see:

```
Hello, Learner! Welcome to CLI Academy.
```

If you see that line, your code is working. Skip to Step 3.

### Step 2 â€” If something went wrong, ask Claude Code

<StepMeta purpose="Use the actual runtime error as evidence instead of guessing at the fix." action="Launch Claude Code, paste the full error message, and ask what is wrong and how to fix it." expectedResult="Claude Code explains the likely cause in plain language and may propose a targeted fix." whyItMatters="Error-driven troubleshooting keeps the assistant grounded in what actually happened instead of generic advice." nextStep="After you understand the troubleshooting flow, intentionally break the project once so you can practice recognizing and fixing a predictable error." askTutor="I'm on Step 2 of Run the project and inspect output. Help me turn this exact runtime error into the safest fix path and explain why that fix is the right first move." />

If the output contains an error (for example a `SyntaxError` or `ReferenceError`), do not try to guess the fix. Open Claude Code and paste the error:

```bash
claude
```

At the prompt, type:

```
I ran npm start and got this error:

<paste the full error message here>

What is wrong and how do I fix it?
```

Claude Code reads your `index.js`, compares it against the error, and explains the cause in plain language. It may offer to fix the file for you â€” review the diff before accepting, just like in Lesson 3.2.2.

After fixing, exit Claude Code with `/exit` and re-run `npm start` to confirm the error is gone.

### Step 3 â€” Intentionally break the project, then fix it

<StepMeta purpose="Build confidence by seeing a common runtime mistake and then recovering from it deliberately." action="Change console.log to console.logg, run npm start to trigger the error, then ask Claude Code to restore the correct line." expectedResult="You see the TypeError, Claude Code shows a one-line fix, and the project works again after the repair." whyItMatters="Practicing on a controlled mistake makes real future errors less intimidating and reinforces the debug-review-fix loop." nextStep="Once the project runs cleanly again, save the repaired working state with a Git commit." askTutor="I'm on Step 3 of Run the project and inspect output. Help me connect this deliberate console.logg mistake to the error message and confirm that Claude Code's one-line fix really restores the original behavior." />

This step teaches you what errors look like so you are not caught off guard later.

Open `index.js` in any editor and change `console.log` to `console.logg` (extra `g`). Save the file and run:

```bash
npm start
```

You will see an error like:

```
TypeError: console.logg is not a function
```

Now launch Claude Code:

```bash
claude
```

Type:

```
I changed console.log to console.logg by accident. Fix it.
```

Claude Code shows a one-line diff restoring `console.log`. Accept the fix and run `npm start` again to see the greeting.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| `npm start` says `Missing script: "start"` | `package.json` does not have a `start` entry, or you are in the wrong folder | Run `cat package.json` (macOS/Linux) or `Get-Content package.json` (PowerShell) to check. Re-create the file from Lesson 3.1.2 if needed |
| `node: command not found` | Node.js is not installed or not on PATH | Install Node.js LTS and restart your terminal |
| `npm start` prints nothing and exits | `index.js` is empty or missing | Verify the file exists with `ls` / `Get-ChildItem` and re-create it from Lesson 3.1.2 |
| Claude Code fixes the file but the error persists | You are running `npm start` from a different folder than where the file was fixed | Make sure your terminal is in `cli-practice` before running |
