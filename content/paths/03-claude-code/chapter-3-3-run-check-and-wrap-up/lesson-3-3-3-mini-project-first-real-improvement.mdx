# Mini project: first real improvement

## Objective
Fantastic job getting your development environment ready! Now, let's make your very first real improvement in your `cli-practice` repository. You'll use the full "ask-plan-edit-run-verify" loop to implement and verify a small, safe change. This is a core skill, and we'll tackle it together, step-by-step.

## Walkthrough
We'll follow a proven development loop: **Ask**, **Plan**, **Edit**, **Run**, and **Verify**. This ensures every change is intentional and tested.

1.  **Ask: What improvement should we make?**
    *   Think of a very small, safe change for your `cli-practice` repo. Examples: adding a greeting line to `README.md`, creating a new `notes.txt` file with a simple message, or updating a 'todo' list.
    *   *Choose something clear and simple that you can describe in one sentence.*

2.  **Plan: How will we make it?**
    *   Mentally (or jot down) the specific steps required to implement your chosen improvement.
    *   *Example plan:* If adding a line to `README.md`: 1. Navigate to `cli-practice`. 2. Open `README.md` with an editor. 3. Add text. 4. Save and exit.

3.  **Edit: Make the change.**
    *   First, navigate to your `cli-practice` directory: `cd cli-practice`.
    *   Use `nano` to open or create your file. For example: `nano README.md` or `nano my_notes.txt`.
        *   **Safety check:** `nano` only affects the file you specify. You can always exit without saving (`Ctrl+X`, then `N`).
    *   Type your improvement (e.g., "Hello, CLI Academy!").
    *   Save: `Ctrl+O` (write out), then `Enter`.
    *   Exit `nano`: `Ctrl+X`.

4.  **Run: See your change.**
    *   Display the file's contents to confirm your edit.
    *   Use `cat [filename]`. For example: `cat README.md` or `cat my_notes.txt`.
        *   **Safety check:** `cat` simply prints content to your terminal; it does not modify files.

5.  **Verify: Does it look correct?**
    *   Compare the output from `cat` with your planned change.
    *   *Is your improvement exactly as you intended?* If not, repeat from step 3 (Edit) to refine it until it's perfect.
