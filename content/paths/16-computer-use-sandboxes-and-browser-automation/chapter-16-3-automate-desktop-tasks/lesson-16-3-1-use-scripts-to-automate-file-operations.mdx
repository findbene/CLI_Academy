# Use scripts to automate file operations

## Objective

By the end of this lesson you will have three working scripts: one that organizes a messy downloads folder by file type, one that batch-renames files with a consistent pattern, and one that cleans up old log files. Each script runs on your local machine with no browser or API required. You will write both a Bash version (macOS / Linux) and a PowerShell version (Windows).

## Walkthrough

File operations — sorting, renaming, deleting — are the bread and butter of automation. A well-written script does in seconds what would take you 30 minutes by hand, and it does it the same way every time.

### Step 1 — Organize files by extension

<StepMeta purpose="Show how a simple local script can replace repetitive file-manager clicking for a common housekeeping task." action="Create the organize script for your platform, generate the dummy test files, run the script against the test folder, and inspect the new extension-based subfolders." expectedResult="Files are sorted into `pdf`, `txt`, `jpg`, `csv`, or `no-ext` folders and the moves are reported clearly in the terminal." whyItMatters="Local file automation is often a better fit than browser or computer-use automation because it is fast, testable, and does not depend on screen state." nextStep="Build the batch-rename script next so you practice a second file operation pattern with safer guards against repeated runs." askTutor="Review this organize script and tell me whether any part of it could move the wrong files or behave unexpectedly on a real folder." />

This script scans a folder and moves each file into a subfolder named after its extension (e.g., `.pdf` files go into `pdf/`, `.jpg` files go into `jpg/`).

**macOS / Linux (`organize.sh`):**

```bash
mkdir -p ~/cli-academy-16/file-ops && cd ~/cli-academy-16/file-ops

cat > organize.sh << 'SCRIPTEOF'
#!/usr/bin/env bash
set -euo pipefail

TARGET_DIR="${1:-.}"   # default to current directory if no argument

echo "Organizing files in: $TARGET_DIR"

for file in "$TARGET_DIR"/*; do
  [ -f "$file" ] || continue          # skip directories
  ext="${file##*.}"                     # extract extension
  [ "$ext" = "$file" ] && ext="no-ext" # files without an extension
  ext=$(echo "$ext" | tr '[:upper:]' '[:lower:]')  # normalize to lowercase

  mkdir -p "$TARGET_DIR/$ext"
  mv "$file" "$TARGET_DIR/$ext/"
  echo "  Moved: $(basename "$file") -> $ext/"
done

echo "Done."
SCRIPTEOF
chmod +x organize.sh
```

**Windows (PowerShell — `organize.ps1`):**

```powershell
New-Item -ItemType Directory -Force -Path "$HOME\cli-academy-16\file-ops" | Out-Null
Set-Location "$HOME\cli-academy-16\file-ops"

@'
param(
    [string]$TargetDir = "."
)

Write-Host "Organizing files in: $TargetDir"

Get-ChildItem -Path $TargetDir -File | ForEach-Object {
    $ext = if ($_.Extension) { $_.Extension.TrimStart('.').ToLower() } else { "no-ext" }
    $destDir = Join-Path $TargetDir $ext

    if (-not (Test-Path $destDir)) {
        New-Item -ItemType Directory -Path $destDir | Out-Null
    }

    Move-Item $_.FullName -Destination $destDir
    Write-Host "  Moved: $($_.Name) -> $ext/"
}

Write-Host "Done."
'@ | Set-Content organize.ps1
```

### Step 2 — Batch rename files

<StepMeta purpose="Practice a second common automation pattern that changes many filenames consistently without manual repetition." action="Create the rename script, test it on a small folder, and confirm the date prefix is added exactly once per file." expectedResult="The test files are renamed with a date prefix and any already-prefixed files are skipped rather than doubled." whyItMatters="Rename automation is useful only when it is idempotent enough to rerun safely, so the skip guard is part of the design, not an extra nicety." nextStep="Use a dry-run-first cleanup script next so you can automate deletion tasks without turning one mistake into data loss." askTutor="Check this batch-rename script for me and tell me whether the prefix guard is strong enough to prevent repeated accidental renaming." />

This script adds a date prefix to every file in a folder — useful for photos, invoices, or any set of files you want sorted chronologically.

**macOS / Linux (`batch-rename.sh`):**

```bash
cat > batch-rename.sh << 'SCRIPTEOF'
#!/usr/bin/env bash
set -euo pipefail

TARGET_DIR="${1:-.}"
PREFIX=$(date +%Y-%m-%d)

echo "Renaming files in: $TARGET_DIR with prefix $PREFIX"

for file in "$TARGET_DIR"/*; do
  [ -f "$file" ] || continue
  base=$(basename "$file")
  # Skip if already prefixed
  if [[ "$base" == "$PREFIX"* ]]; then
    echo "  Skipped (already prefixed): $base"
    continue
  fi
  mv "$file" "$TARGET_DIR/${PREFIX}_${base}"
  echo "  Renamed: $base -> ${PREFIX}_${base}"
done

echo "Done."
SCRIPTEOF
chmod +x batch-rename.sh
```

**Windows (PowerShell — `batch-rename.ps1`):**

```powershell
@'
param(
    [string]$TargetDir = "."
)

$prefix = Get-Date -Format "yyyy-MM-dd"
Write-Host "Renaming files in: $TargetDir with prefix $prefix"

Get-ChildItem -Path $TargetDir -File | ForEach-Object {
    if ($_.Name.StartsWith($prefix)) {
        Write-Host "  Skipped (already prefixed): $($_.Name)"
        return
    }
    $newName = "${prefix}_$($_.Name)"
    Rename-Item $_.FullName -NewName $newName
    Write-Host "  Renamed: $($_.Name) -> $newName"
}

Write-Host "Done."
'@ | Set-Content batch-rename.ps1
```

### Step 3 — Clean up old log files

<StepMeta purpose="Teach the safer pattern for deletion automation: preview first, then require an explicit force step for real removal." action="Create the cleanup script, run it in dry-run mode, and inspect the printed candidate files before considering a real delete run." expectedResult="The script lists files older than the configured threshold without deleting anything by default." whyItMatters="Deletion is the highest-risk local file automation in this lesson, so the dry-run default is the main safety feature you are practicing here." nextStep="Use the next lesson to schedule scripts like these so they become recurring automation instead of one-off terminal commands." askTutor="Review this cleanup script and tell me whether the dry-run and age-threshold design are safe enough before I ever let it delete real files." />

This script deletes files older than a specified number of days. Always perform a dry run first.

**macOS / Linux (`clean-logs.sh`):**

```bash
cat > clean-logs.sh << 'SCRIPTEOF'
#!/usr/bin/env bash
set -euo pipefail

LOG_DIR="${1:-.}"
MAX_AGE_DAYS="${2:-30}"
DRY_RUN="${3:-true}"

echo "Cleaning files older than $MAX_AGE_DAYS days in: $LOG_DIR"
echo "Dry run: $DRY_RUN"

find "$LOG_DIR" -type f -mtime +"$MAX_AGE_DAYS" | while read -r file; do
  if [ "$DRY_RUN" = "true" ]; then
    echo "  [DRY RUN] Would delete: $file"
  else
    rm "$file"
    echo "  Deleted: $file"
  fi
done

echo "Done."
SCRIPTEOF
chmod +x clean-logs.sh
```

**Windows (PowerShell — `clean-logs.ps1`):**

```powershell
@'
param(
    [string]$LogDir = ".",
    [int]$MaxAgeDays = 30,
    [switch]$Force
)

$cutoff = (Get-Date).AddDays(-$MaxAgeDays)
Write-Host "Cleaning files older than $MaxAgeDays days in: $LogDir"
Write-Host "Dry run: $(-not $Force)"

Get-ChildItem -Path $LogDir -File | Where-Object { $_.LastWriteTime -lt $cutoff } | ForEach-Object {
    if (-not $Force) {
        Write-Host "  [DRY RUN] Would delete: $($_.Name)"
    } else {
        Remove-Item $_.FullName
        Write-Host "  Deleted: $($_.Name)"
    }
}

Write-Host "Done."
'@ | Set-Content clean-logs.ps1
```

**Safety note:** The default behavior is a dry run — it only prints what *would* be deleted. Pass `false` (Bash) or `-Force` (PowerShell) to actually delete. Always review the dry-run output before running for real.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| Script moves files you did not intend to move | Ran the organize script on your real Downloads folder instead of a test folder | Always test on a dummy directory first; back up before running on real data |
| `mv: cannot move — Permission denied` | Files are owned by another user or are read-only | Check permissions with `ls -la`; use `chmod` or run as the correct user |
| Rename script creates double-prefixed names | Ran the script twice on the same folder | The `already prefixed` check prevents this — but verify your prefix format matches |
| Clean script deletes files you wanted to keep | `MAX_AGE_DAYS` set too low, or ran without dry-run review | Always do a dry run first; set a conservative age threshold |
| PowerShell script won't run — `execution policy` error | Windows blocks unsigned scripts by default | Run `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` once |
| `find: unknown predicate` on macOS | GNU vs BSD `find` syntax difference | macOS uses BSD `find` — the `-mtime +N` syntax works on both, but more complex predicates may differ |
