# Mini project: automate a daily workflow

## Objective

By the end of this mini project you will have built, scheduled, and tested a complete daily automation workflow that combines file operations, monitoring, and scheduling into a single pipeline. You will commit the project to a Git repository so you can iterate on it over time.

## Walkthrough

This project ties together everything from Path 16: you will pick a real daily task, write scripts to automate it, schedule the scripts to run unattended, add monitoring to verify they succeeded, and package the whole thing into a project folder.

### Step 1 — Choose your daily workflow

Pick one of these real workflows (or propose your own):

| Workflow | What it does |
|---|---|
| **Downloads organizer** | Sort today's downloads by file type, rename with date prefix, delete anything older than 30 days |
| **Dev environment health check** | Verify Node, Python, and Git are available, check disk space, log the results |
| **Website uptime report** | Check 5 URLs, log results, generate a one-line summary at the end |

For this walkthrough we will build the **Dev environment health check**, but the structure works for any choice.

### Step 2 — Set up the project folder

**macOS / Linux:**

```bash
mkdir -p ~/cli-academy-16/daily-workflow && cd ~/cli-academy-16/daily-workflow
git init
```

**Windows (PowerShell):**

```powershell
New-Item -ItemType Directory -Force -Path "$HOME\cli-academy-16\daily-workflow" | Out-Null
Set-Location "$HOME\cli-academy-16\daily-workflow"
git init
```

### Step 3 — Write the main automation script

**macOS / Linux (`daily-check.sh`):**

```bash
cat > daily-check.sh << 'SCRIPTEOF'
#!/usr/bin/env bash
set -uo pipefail

LOG_DIR="$HOME/cli-academy-16/daily-workflow/logs"
mkdir -p "$LOG_DIR"
LOG_FILE="$LOG_DIR/$(date '+%Y-%m-%d').log"

log() { echo "[$(date '+%H:%M:%S')] $1" | tee -a "$LOG_FILE"; }

log "=== Daily health check started ==="

# Check 1: Node.js
if command -v node &>/dev/null; then
  log "✓ Node.js $(node -v)"
else
  log "✗ Node.js not found"
fi

# Check 2: Python
if command -v python3 &>/dev/null; then
  log "✓ Python $(python3 --version 2>&1)"
else
  log "✗ Python 3 not found"
fi

# Check 3: Git
if command -v git &>/dev/null; then
  log "✓ Git $(git --version)"
else
  log "✗ Git not found"
fi

# Check 4: Disk space
disk_usage=$(df -h / | awk 'NR==2 {print $5}')
log "Disk usage on /: $disk_usage"
usage_pct=${disk_usage%%%}
if [ "$usage_pct" -gt 90 ]; then
  log "⚠ WARNING: Disk usage above 90%!"
fi

# Check 5: Internet connectivity
if curl -s --max-time 5 https://1.1.1.1 > /dev/null 2>&1; then
  log "✓ Internet connectivity OK"
else
  log "✗ Internet connectivity FAILED"
fi

log "=== Daily health check complete ==="
SCRIPTEOF
chmod +x daily-check.sh
```

**Windows (PowerShell — `daily-check.ps1`):**

```powershell
@'
$logDir = "$HOME\cli-academy-16\daily-workflow\logs"
if (-not (Test-Path $logDir)) { New-Item -ItemType Directory -Path $logDir | Out-Null }
$logFile = Join-Path $logDir "$(Get-Date -Format 'yyyy-MM-dd').log"

function Log($msg) {
    $line = "[$(Get-Date -Format 'HH:mm:ss')] $msg"
    Write-Host $line
    Add-Content -Path $logFile -Value $line
}

Log "=== Daily health check started ==="

# Check 1: Node.js
try {
    $nodeVersion = & node -v 2>&1
    Log "[OK] Node.js $nodeVersion"
} catch {
    Log "[FAIL] Node.js not found"
}

# Check 2: Python
try {
    $pyVersion = & python --version 2>&1
    Log "[OK] $pyVersion"
} catch {
    Log "[FAIL] Python not found"
}

# Check 3: Git
try {
    $gitVersion = & git --version 2>&1
    Log "[OK] $gitVersion"
} catch {
    Log "[FAIL] Git not found"
}

# Check 4: Disk space
$drive = Get-PSDrive C
$usedPct = [math]::Round(($drive.Used / ($drive.Used + $drive.Free)) * 100)
Log "Disk usage on C: ${usedPct}%"
if ($usedPct -gt 90) {
    Log "[WARNING] Disk usage above 90%!"
}

# Check 5: Internet connectivity
try {
    $null = Invoke-WebRequest -Uri "https://1.1.1.1" -TimeoutSec 5 -UseBasicParsing -ErrorAction Stop
    Log "[OK] Internet connectivity OK"
} catch {
    Log "[FAIL] Internet connectivity FAILED"
}

Log "=== Daily health check complete ==="
'@ | Set-Content daily-check.ps1
```

### Step 4 — Run the script manually and review the log

**macOS / Linux:**

```bash
bash daily-check.sh
cat logs/$(date '+%Y-%m-%d').log
```

**Windows (PowerShell):**

```powershell
.\daily-check.ps1
Get-Content "logs\$(Get-Date -Format 'yyyy-MM-dd').log"
```

You should see five checks, each with a pass or fail indicator, plus a disk usage percentage.

### Step 5 — Schedule the script

**macOS / Linux (cron):**

```bash
crontab -e
# Add:
# 0 7 * * * /bin/bash $HOME/cli-academy-16/daily-workflow/daily-check.sh
```

**Windows (PowerShell — Task Scheduler):**

```powershell
$action = New-ScheduledTaskAction `
    -Execute "powershell.exe" `
    -Argument "-NoProfile -ExecutionPolicy Bypass -File `"$HOME\cli-academy-16\daily-workflow\daily-check.ps1`""

$trigger = New-ScheduledTaskTrigger -Daily -At 7:00AM

Register-ScheduledTask `
    -TaskName "CLI Academy Daily Health Check" `
    -Action $action `
    -Trigger $trigger `
    -Description "Runs the dev environment health check every morning at 7 AM"
```

### Step 6 — Add a summary report

Create a script that reads the latest log and prints a one-line summary.

**macOS / Linux (`summary.sh`):**

```bash
cat > summary.sh << 'SCRIPTEOF'
#!/usr/bin/env bash
LOG_FILE="logs/$(date '+%Y-%m-%d').log"

if [ ! -f "$LOG_FILE" ]; then
  echo "No log for today. Run daily-check.sh first."
  exit 1
fi

passes=$(grep -c '✓' "$LOG_FILE" || true)
fails=$(grep -c '✗' "$LOG_FILE" || true)
warnings=$(grep -c '⚠' "$LOG_FILE" || true)

echo "Summary for $(date '+%Y-%m-%d'): $passes passed, $fails failed, $warnings warnings"
SCRIPTEOF
chmod +x summary.sh
bash summary.sh
```

**Windows (PowerShell — `summary.ps1`):**

```powershell
@'
$logFile = "logs\$(Get-Date -Format 'yyyy-MM-dd').log"

if (-not (Test-Path $logFile)) {
    Write-Host "No log for today. Run daily-check.ps1 first."
    exit 1
}

$content = Get-Content $logFile
$passes = ($content | Select-String "\[OK\]").Count
$fails = ($content | Select-String "\[FAIL\]").Count
$warnings = ($content | Select-String "\[WARNING\]").Count

Write-Host "Summary for $(Get-Date -Format 'yyyy-MM-dd'): $passes passed, $fails failed, $warnings warnings"
'@ | Set-Content summary.ps1
.\summary.ps1
```

### Step 7 — Commit the project

```bash
cat > .gitignore << 'EOF'
logs/
node_modules/
EOF

git add -A
git commit -m "feat: daily workflow automation with health checks, scheduling, and summary"
```

Your project folder should look like this:

```
daily-workflow/
├── .gitignore
├── daily-check.sh      # or daily-check.ps1
├── summary.sh          # or summary.ps1
└── logs/
    └── 2026-04-09.log  # (gitignored — one per day)
```

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| Script reports "Node.js not found" but you can run `node -v` interactively | cron/Task Scheduler uses a stripped-down PATH that does not include the directory where Node is installed | Use the full path to `node` in the script (e.g., `/usr/local/bin/node`), or add `export PATH="/usr/local/bin:$PATH"` at the top of the script |
| Log file is created but empty | The script crashed before writing anything | Run the script manually to see the error; check file permissions on the logs directory |
| Disk usage check fails on macOS | `df` output format differs slightly from Linux | Adjust the `awk` column index — test with `df -h / | head -2` to see the layout |
| Scheduled task runs but you never notice failures | No alerting mechanism — the log sits unread | Extend the summary script to send a notification (email, Slack webhook, desktop notification) if any check fails |
| Git commit includes the logs folder | Missing or incorrect `.gitignore` | Add `logs/` to `.gitignore` and run `git rm -r --cached logs/` if already tracked |
| The workflow runs indefinitely on a cloud VM | No cost controls and the VM stays on 24/7 | Set the VM to auto-shutdown after the task, or use a serverless function instead |
