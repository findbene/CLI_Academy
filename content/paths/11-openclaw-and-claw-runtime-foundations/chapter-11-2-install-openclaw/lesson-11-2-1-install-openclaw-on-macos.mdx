# Install OpenClaw on macOS

## What you will build

OpenClaw running natively on your Mac, accessible at `http://localhost:3301`, configured with your API key, and set to start automatically on login via a LaunchAgent.

## Why this matters

macOS is the smoothest platform for a first OpenClaw install. No WSL2, no virtual machine overhead. Apple Silicon Macs (M1 through M4) have enough power to run OpenClaw as an always-on local gateway with idle power draw under 10 W. A Mac Mini makes an excellent dedicated home server. This lesson gets you to a working gateway you can use in the rest of this path.

## Before you start

You need:
- macOS 13 (Ventura) or later
- A Pro or Team Claude API key from Anthropic
- Homebrew installed (recommended), or git and Node.js as an alternative
- About 20 minutes

## Walkthrough

### Step 1 — Install via Homebrew (recommended)

```bash
brew tap openclaw-project/openclaw
brew install openclaw
```

Homebrew places the binary at `/opt/homebrew/bin/openclaw` (Apple Silicon) or `/usr/local/bin/openclaw` (Intel), and creates `~/.openclaw/` with a sample `.env`.

Verify the install:
```bash
openclaw --version
```

**Alternative — install via git clone:**
```bash
cd ~
git clone https://github.com/openclaw-project/openclaw.git
cd openclaw
npm install
```

### Step 2 — Configure your environment

Copy the example configuration:
```bash
# Homebrew path
cp ~/.openclaw/.env.example ~/.openclaw/.env
nano ~/.openclaw/.env

# Git clone path
cp .env.example .env
nano .env
```

Set these required values:
```env
ANTHROPIC_API_KEY=sk-ant-...
OPENCLAW_PORT=3301
OPENCLAW_WEB_UI=true
OPENCLAW_SECRET=replace_with_random_string
OWNER_DISPLAY_NAME=YourName
OWNER_TIMEZONE=America/New_York
```

Generate a secure secret for `OPENCLAW_SECRET`:
```bash
openssl rand -hex 32
```

Copy the output into your `.env` file.

### Step 3 — First launch

```bash
# Homebrew
openclaw start

# Git clone
npm run start
```

Open `http://localhost:3301` in your browser. Complete the first-run wizard to set your admin password.

Confirm the gateway is running:
```bash
curl -s http://localhost:3301/health
```

You should see a JSON response with `"status": "ok"`.

### Step 4 — Auto-start on login (LaunchAgent)

A LaunchAgent is macOS's built-in mechanism for running a process automatically when you log in. This is lighter than Docker and integrates properly with macOS service management.

Create the plist file:
```bash
nano ~/Library/LaunchAgents/dev.openclaw.gateway.plist
```

Paste this content (adjust the binary path for Intel if needed):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>dev.openclaw.gateway</string>

  <key>ProgramArguments</key>
  <array>
    <string>/opt/homebrew/bin/openclaw</string>
    <string>start</string>
  </array>

  <key>EnvironmentVariables</key>
  <dict>
    <key>PATH</key>
    <string>/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin</string>
  </dict>

  <key>RunAtLoad</key>
  <true/>

  <key>KeepAlive</key>
  <true/>

  <key>StandardOutPath</key>
  <string>/tmp/openclaw.log</string>

  <key>StandardErrorPath</key>
  <string>/tmp/openclaw-error.log</string>
</dict>
</plist>
```

Set permissions and load:
```bash
chmod 644 ~/Library/LaunchAgents/dev.openclaw.gateway.plist
launchctl load ~/Library/LaunchAgents/dev.openclaw.gateway.plist
```

Verify it is running:
```bash
launchctl list | grep openclaw
```

You should see a PID number next to `dev.openclaw.gateway`.

### Common issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| `openclaw: command not found` after Homebrew | Shell not sourcing Homebrew | Run `eval "$(brew shellenv)"` or open a new terminal |
| LaunchAgent not auto-starting | Wrong binary path in plist | Confirm path with `which openclaw`, update plist |
| Port 3301 blocked | macOS Firewall | System Settings → Network → Firewall → Allow OpenClaw |
| `OPENCLAW_SECRET` error on start | Secret too short | Use `openssl rand -hex 32` to generate a valid secret |
