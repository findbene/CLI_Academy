# Use GitHub Codespaces for cloud development

## Objective

By the end of this lesson you will be able to launch a GitHub Codespace from any repository, use the built-in terminal, install and run Claude Code inside it, and understand the free-tier limits so you do not accidentally incur charges.

## Walkthrough

GitHub Codespaces gives you a full Linux development environment running in Microsoft's cloud. You access it through your browser or VS Code — no local setup required. It is one of the fastest ways to get a working CLI environment when you are on a locked-down computer or want a throwaway sandbox.

### Step 1 — Create a Codespace from a repository

1. Go to any GitHub repository (your own or a public one).
2. Click the green **Code** button.
3. Switch to the **Codespaces** tab.
4. Click **Create codespace on main** (or whichever branch you need).

GitHub provisions a virtual machine, clones the repo, and opens VS Code in your browser within about 60 seconds.

### Step 2 — Explore the Codespace terminal

At the bottom of the VS Code browser UI, open the **Terminal** panel (`` Ctrl+` `` or **View → Terminal**). This is a real Linux bash shell:

```bash
uname -a
# Linux codespaces-xxxxxx 6.x.x ... x86_64 GNU/Linux

whoami
# codespace

pwd
# /workspaces/your-repo-name
```

Your repo files are already cloned into `/workspaces/`. Git is configured with your GitHub identity.

### Step 3 — Install Claude Code in the Codespace

The Codespace has Node.js preinstalled:

```bash
node -v
# v20.x.x or later

npm install -g @anthropic-ai/claude-code
claude --version
```

Then authenticate:

```bash
claude
# Follow the browser-based OAuth flow to link your Anthropic account
```

After authentication, Claude Code works exactly as it does locally — but the compute runs on GitHub's cloud.

### Step 4 — Understand the free-tier limits

| Resource | Free-tier allowance (per month) |
|---|---|
| Core hours (2-core machine) | 60 hours |
| Core hours (4-core machine) | 30 hours |
| Storage | 15 GB |

**Key safety rules:**

- **Stop your Codespace when you are done.** A running Codespace burns core hours even if you are not typing. Click the blue bar at the bottom-left → **Stop Codespace**, or run from any terminal:

```bash
# From your LOCAL terminal (not inside the Codespace)
gh codespace stop
```

- **Delete Codespaces you no longer need.** Idle Codespaces still consume storage.
- **Set a spending limit.** Go to **Settings → Billing → Spending limits → Codespaces** and set it to $0 to prevent any charges.

### Step 5 — Connect to a Codespace from your local terminal

If you prefer your local editor, install the GitHub CLI:

**macOS / Linux:**

```bash
# macOS
brew install gh

# Ubuntu / Debian
sudo apt install gh
```

**Windows (PowerShell):**

```powershell
winget install GitHub.cli
```

Then SSH into your running Codespace:

```bash
gh codespace ssh
```

This sets up an SSH tunnel — you get a remote shell with full access to the Codespace filesystem.

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| Codespace creation fails | You have hit the free-tier limit or your account lacks Codespaces access | Check **Settings → Billing → Codespaces** for usage; free personal accounts on GitHub Free have Codespaces access |
| `npm install -g` fails with `EACCES` | Permission issue in the Codespace (rare) | Use `sudo npm install -g @anthropic-ai/claude-code` |
| Codespace is running but you forgot about it | Core hours draining silently | Set auto-stop timeout: **Settings → Codespaces → Default idle timeout** to 30 minutes |
| Claude authentication flow does not open a browser | Codespace is running headless | Copy the URL printed in the terminal and paste it into your local browser |
| Files disappear after stopping | You deleted the Codespace instead of stopping it | Stopping preserves files; deleting wipes them — commit and push before deleting |
| Unexpected charges on your GitHub bill | Ran over free-tier hours or storage | Set spending limit to $0 and delete unused Codespaces |
