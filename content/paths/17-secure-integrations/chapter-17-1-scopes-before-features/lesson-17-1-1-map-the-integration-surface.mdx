# Map the integration surface

## What you will build
A .integration-audit.md file with at least three services mapped, access types recorded, and Write/Delete/Admin rows flagged as Dangerous.

## Why this matters
Integrating Claude with external APIs and services is where most production security incidents happen. A misconfigured webhook, an exposed key, or an over-permissioned OAuth scope can expose entire systems. This path teaches the integration patterns that hold up under a security review.

## Before you start
Before you connect anything, you need to know *what* you are connecting. An **integration surface** is the complete set of data, actions, and credentials that an external service can reach the moment you grant it access. Mapping the surface before writing a single line of code lets you spot dangerous zones early, pick the right credential type, and make a deliberate decision about every permission you grant.

In this lesson you will create a real `.integration-audit.md` file that catalogs the services you plan to integrate with. By the end you will have a reusable template any teammate can fill out for any new integration.


## Walkthrough

### Step 1 — Understand what belongs on the map

Every integration has four things worth recording:

| Field | What to capture |
|---|---|
| **Service** | Name of the external system (GitHub, Slack, Google Sheets…) |
| **Data touched** | What data the integration reads or writes |
| **Access type** | Read-only / Write / Delete / Admin |
| **Credential type** | OAuth token, API key, Personal Access Token, service account |

A **dangerous surface** is any row where access type is Write, Delete, or Admin — or where the data touched is sensitive (PII, billing data, auth secrets, private code).

### Step 2 — Create your integration audit file

**macOS / Linux:**
```bash
mkdir -p ~/projects/integration-safety
cd ~/projects/integration-safety

cat > .integration-audit.md << 'EOF'
# Integration Surface Audit

Last updated: FILL_IN_DATE

## Service map

| Service        | Data touched                   | Access type   | Credential type         | Dangerous?   |
|----------------|--------------------------------|---------------|-------------------------|--------------|
| GitHub         | Repos, PRs, issues, code       | Read / Write  | Personal Access Token   | ⚠️ Write     |
| Slack          | Messages, channels, users      | Read / Post   | OAuth Bot Token         | ⚠️ Post      |
| Google Sheets  | Spreadsheet rows               | Read / Write  | OAuth Service Account   | ⚠️ Write     |
| npm registry   | Published packages             | Write         | Automation Token        | 🔴 Publish   |

## Notes
- Replace each row with your actual integrations.
- Flag every Write / Delete / Admin row for scope review in lesson 17.1.2.
- Never add OAuth client secrets or raw API keys to this file — record credential *type* only.
EOF

cat .integration-audit.md
```

**Windows PowerShell:**
```powershell
New-Item -ItemType Directory -Force -Path "$HOME\projects\integration-safety" | Out-Null
Set-Location "$HOME\projects\integration-safety"

$auditContent = @'
# Integration Surface Audit

Last updated: FILL_IN_DATE

## Service map

| Service        | Data touched                   | Access type   | Credential type         | Dangerous?   |
|----------------|--------------------------------|---------------|-------------------------|--------------|
| GitHub         | Repos, PRs, issues, code       | Read / Write  | Personal Access Token   | ⚠️ Write     |
| Slack          | Messages, channels, users      | Read / Post   | OAuth Bot Token         | ⚠️ Post      |
| Google Sheets  | Spreadsheet rows               | Read / Write  | OAuth Service Account   | ⚠️ Write     |
| npm registry   | Published packages             | Write         | Automation Token        | 🔴 Publish   |

## Notes
- Replace each row with your actual integrations.
- Flag every Write / Delete / Admin row for scope review in lesson 17.1.2.
- Never add OAuth client secrets or raw API keys to this file — record credential *type* only.
'@

$auditContent | Out-File -FilePath ".integration-audit.md" -Encoding utf8
Get-Content .integration-audit.md
```

### Step 3 — Customize the map for your own integrations

Replace the example rows with the real services you plan to use. Map **at least three services**. For each Write-access row, add a one-sentence note in the Notes section explaining *why* write access is necessary — if you cannot justify it in one sentence, you probably do not need it yet.

### Step 4 — Verify the file is complete

**macOS / Linux:**
```bash
# Confirm the file exists and has content
wc -l .integration-audit.md       # expect more than 10 lines
grep -i "dangerous" .integration-audit.md   # should print the table header
```

**Windows PowerShell:**
```powershell
# Confirm the file exists and has content
(Get-Content .integration-audit.md).Count   # expect more than 10
Select-String -Path .integration-audit.md -Pattern "Dangerous"
```

### What can go wrong

| Symptom | Likely cause | Fix |
|---|---|---|
| File is empty after `cat` / `Get-Content` | Here-doc delimiter `'EOF'` had trailing whitespace | Delete the file, re-run `cat > .integration-audit.md << 'EOF'` with no trailing spaces |
| PowerShell shows `?` characters instead of emoji | File saved in Latin-1 encoding | Add `-Encoding utf8` to `Out-File` |
| Audit table only has one row | Rushed the map | Add all real services before moving to 17.1.2 |
| Every row lists Admin access | Defaulting to broadest permission | Challenge each row: what is the *minimum* access this workflow actually needs? |
| Secrets appear in the file | Pasted token values directly | Remove them immediately, rotate the exposed token, then record only the credential *type* |

> **Safety warning — incomplete maps compound over time.** An integration audit with gaps is worse than no audit because it creates false confidence. If you skip rows now, you will discover missing scopes only after you have already granted broad access — and broad tokens are difficult to shrink without breaking live workflows.
