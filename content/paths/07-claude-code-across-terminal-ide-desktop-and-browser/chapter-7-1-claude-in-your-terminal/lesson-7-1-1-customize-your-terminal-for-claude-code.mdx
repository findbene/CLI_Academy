# Customize your terminal for Claude Code

## What you will build
By the end of this lesson you will have a terminal prompt that shows your current Git branch and Node.

## Why this matters
Real-world agentic work spans surfaces — you start in the terminal, switch to the IDE, check GitHub, and come back to the terminal. Learners who can only use Claude in one surface leave efficiency on the table. Cross-surface fluency is what makes Claude Code genuinely useful for full development cycles.

## Before you start
By the end of this lesson you will have a terminal prompt that shows your current Git branch and Node.js version, custom tab colors or profiles for different tasks, and a short alias so you can launch Claude Code by typing `cc` instead of the full command. These small changes remove friction every time you start a session.


## Walkthrough

### Step 1 — Show the Git branch in your prompt

When you work with Claude Code inside a repository, knowing which branch you are on prevents mistakes. A prompt that displays the branch saves you from running `git branch` constantly.

**macOS / Linux (Bash):**

Open your shell config file:

```bash
nano ~/.bashrc   # or ~/.zshrc if you use Zsh
```

Add these lines at the bottom:

```bash
# Git branch in prompt
parse_git_branch() {
  git branch 2>/dev/null | grep '\*' | sed 's/* //'
}
export PS1="\[\e[32m\]\u@\h \[\e[34m\]\w \[\e[33m\]\$(parse_git_branch)\[\e[0m\]\n\$ "
```

Save and reload:

```bash
source ~/.bashrc
```

Your prompt now looks like:

```
yourname@machine ~/projects/my-app main
$
```

**Windows PowerShell:**

Open your PowerShell profile:

```powershell
notepad $PROFILE
```

If the file does not exist, PowerShell will ask to create it — say Yes. Add these lines:

```powershell
function prompt {
    $branch = git rev-parse --abbrev-ref HEAD 2>$null
    $loc = (Get-Location).Path.Replace($HOME, '~')
    if ($branch) {
        Write-Host "$loc " -NoNewline -ForegroundColor Cyan
        Write-Host "[$branch]" -NoNewline -ForegroundColor Yellow
    } else {
        Write-Host "$loc" -NoNewline -ForegroundColor Cyan
    }
    return "`n> "
}
```

Save the file and reload:

```powershell
. $PROFILE
```

Your prompt now shows the folder path and current Git branch in color.

> **Why this matters:** Claude Code often tells you "switch to branch X" or "commit on main." A branch-aware prompt means you always know your context at a glance.

### Step 2 — Show the Node.js version in your prompt

Many Claude Code projects depend on a specific Node version. Displaying it avoids silent version-mismatch bugs.

**macOS / Linux (Bash):**

Edit the same `~/.bashrc` (or `~/.zshrc`) and replace the PS1 line with:

```bash
export PS1="\[\e[32m\]\u@\h \[\e[34m\]\w \[\e[33m\]\$(parse_git_branch) \[\e[35m\]node-\$(node -v 2>/dev/null)\[\e[0m\]\n\$ "
```

Reload:

```bash
source ~/.bashrc
```

**Windows PowerShell:**

Update the `prompt` function in `$PROFILE`:

```powershell
function prompt {
    $branch = git rev-parse --abbrev-ref HEAD 2>$null
    $nodeVer = node -v 2>$null
    $loc = (Get-Location).Path.Replace($HOME, '~')
    Write-Host "$loc " -NoNewline -ForegroundColor Cyan
    if ($branch) { Write-Host "[$branch] " -NoNewline -ForegroundColor Yellow }
    if ($nodeVer) { Write-Host "node-$nodeVer" -NoNewline -ForegroundColor Magenta }
    return "`n> "
}
```

Reload:

```powershell
. $PROFILE
```

### Step 3 — Set up terminal profiles with distinct colors

Color-coding terminals helps you separate Claude Code sessions from regular shell work.

**Windows Terminal (built-in on Windows 11):**

Open Windows Terminal Settings (Ctrl + ,), click **Add a new profile**, then:

1. Name: `Claude Code`
2. Command line: `pwsh.exe` (or `powershell.exe`)
3. Starting directory: your projects folder, e.g. `C:\Users\you\projects`
4. Under **Appearance**, set the color scheme to a dark variant like **One Half Dark** or **Tango Dark** and set the tab color to a unique accent (e.g., `#7B61FF`).
5. Click **Save**.

Now when you open a new tab you can pick "Claude Code" and instantly recognize it.

**macOS (iTerm2):**

1. Open **iTerm2 → Preferences → Profiles**.
2. Create a new profile called `Claude Code`.
3. Under **Colors**, pick a preset like **Solarized Dark** or set a session tab color.
4. Under **General → Command**, leave it as Login shell.
5. Under **General → Working directory**, set your projects folder.

**Linux (GNOME Terminal):**

1. Open **Preferences → Profiles**.
2. Add a profile called `Claude Code`.
3. Pick a dark color palette and set a custom background tint so it stands out.

### Step 4 — Create a short alias for Claude Code

Typing `claude` every time is fine, but `cc` is faster.

**macOS / Linux (Bash/Zsh):**

```bash
echo 'alias cc="claude"' >> ~/.bashrc   # or ~/.zshrc
source ~/.bashrc
```

Verify:

```bash
cc --version
```

**Windows PowerShell:**

```powershell
Add-Content $PROFILE 'Set-Alias cc claude'
. $PROFILE
```

Verify:

```powershell
cc --version
```

Both should print the Claude Code version, confirming the alias works.

> **Safety note:** Make sure `cc` does not collide with another command on your system. Run `Get-Command cc` (PowerShell) or `which cc` (Bash) before creating the alias. On some Linux systems, `cc` is a symlink to the C compiler. If that is the case, use `cla` or `cl` instead.

### Step 5 — Persist everything and validate

All changes above edit dotfiles (`~/.bashrc`, `$PROFILE`). They only take effect in new terminal sessions or after a reload. Run through this checklist:

1. Open a **new** terminal tab or window.
2. `cd` into a Git repository.
3. Confirm the prompt shows the branch name and Node version.
4. Type `cc --version` and confirm it prints a version number.

If the prompt looks plain (no colors, no branch), the most common cause is that you edited the wrong config file. See the troubleshooting table below.

### What can go wrong

| Symptom | Cause | Fix |
|---|---|---|
| Prompt shows `parse_git_branch: command not found` | The function was not loaded; you edited `~/.bashrc` but your default shell reads `~/.zshrc` | Check your shell with `echo $SHELL` and edit the matching file |
| `node -v` shows nothing in the prompt | Node.js is not installed or not on PATH | Install Node via `nvm install --lts` (macOS/Linux) or download from nodejs.org (Windows) |
| `cc : The term 'cc' is not recognized` in PowerShell | Profile was not reloaded | Run `. $PROFILE` or open a new tab |
| `cc` runs the C compiler instead of Claude | `cc` is already bound to `/usr/bin/cc` on your system | Use a different alias like `cla`; update with `alias cla="claude"` |
| Colors do not appear | Terminal emulator does not support ANSI escape codes | Use Windows Terminal, iTerm2, or any modern terminal instead of legacy `cmd.exe` |
| `$PROFILE` path does not exist | PowerShell has never created a profile file | Run `New-Item -Path $PROFILE -ItemType File -Force` to create it |
