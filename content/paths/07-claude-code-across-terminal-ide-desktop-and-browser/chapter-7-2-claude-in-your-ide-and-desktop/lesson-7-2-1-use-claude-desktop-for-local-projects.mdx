# Use Claude Desktop for local projects

## Objective

By the end of this lesson you will have Claude Desktop installed, opened a local project folder inside it, and used its drag-and-drop feature to share files with Claude. You will also understand exactly what Claude Desktop can and cannot do compared to the Claude Code CLI.

## Walkthrough

### Step 1 — Install Claude Desktop

<StepMeta purpose="Get the graphical Claude Desktop app installed so you can compare it with the CLI from real experience." action="Download the official installer for your operating system, complete the installation, launch the app, and sign in." expectedResult="Claude Desktop opens successfully on your machine and you are authenticated with your Anthropic account." whyItMatters="You cannot evaluate Desktop versus CLI fairly until the app is actually available and working on your own system." nextStep="Open a local project by sharing a few files so Claude Desktop has concrete material to discuss." askTutor="I'm on Step 1 of Use Claude Desktop for local projects. Help me choose the correct installer for my OS and confirm the app installed properly." />

Download the app from the official source.

**macOS:**

1. Go to [claude.ai/download](https://claude.ai/download) in your browser.
2. Download the `.dmg` file for macOS.
3. Open the `.dmg` and drag **Claude** into your Applications folder.
4. Launch Claude from Applications. Sign in with your Anthropic account.

**Windows:**

1. Go to [claude.ai/download](https://claude.ai/download) in your browser.
2. Download the `.exe` installer for Windows.
3. Run the installer and follow the prompts.
4. Launch Claude from the Start menu. Sign in with your Anthropic account.

**Linux:**

Claude Desktop is available as a `.deb` package for Debian/Ubuntu-based distributions:

```bash
# Download the latest .deb from claude.ai/download
sudo dpkg -i claude-desktop_*.deb
```

Launch it from your application menu or run `claude-desktop` in a terminal.

> **Note:** You need an active Anthropic account (free or paid) to use Claude Desktop. The app uses the same account as claude.ai.

### Step 2 — Open a local project folder

<StepMeta purpose="Learn the Desktop file-sharing workflow that replaces direct filesystem access." action="Use drag and drop or the attachment button to bring a few project files into a Claude Desktop conversation." expectedResult="Claude Desktop displays the attached files in the chat and can reference their contents in follow-up answers." whyItMatters="This manual sharing model is the core difference from the CLI, so it is important to feel exactly how context enters a Desktop conversation." nextStep="Ask Claude to analyze the shared files so you see how useful the attached context becomes." askTutor="I'm on Step 2 of Use Claude Desktop for local projects. Help me decide which files are best to attach first and explain what Claude can infer from them." />

Claude Desktop can work with files you share with it. There are two ways to bring files in:

**Method A — Drag and drop:**

1. Open your system file manager (Finder on macOS, File Explorer on Windows).
2. Navigate to a project folder.
3. Select one or more files (e.g., `README.md`, `package.json`, `src/index.ts`).
4. Drag them into the Claude Desktop chat window.
5. Claude reads the file contents and you can ask questions about them.

**Method B — Use the attachment button:**

1. In the Claude Desktop chat, click the **paperclip icon** (📎) or the **+** button.
2. Select **Add files** from the menu.
3. Browse to your project folder and select the files you want to share.

Either method copies the file contents into the conversation. Claude can now analyze, explain, or suggest changes to those files.

### Step 3 — Ask Claude Desktop to analyze your project files

<StepMeta purpose="Verify that Claude Desktop can reason over the files you shared even though it cannot browse your disk directly." action="Prompt Claude to summarize the project, inspect dependencies, or point out bugs based on the attached files." expectedResult="Claude gives answers grounded in the file contents you explicitly provided in the chat." whyItMatters="This proves that Desktop is strong for explanation and planning work once you have supplied the relevant context." nextStep="Generate a new file in chat so you can experience the manual save step that separates Desktop from the CLI." askTutor="I'm on Step 3 of Use Claude Desktop for local projects. Help me write a prompt that will prove Claude is actually using the attached files instead of answering generically." />

Once you have dragged in a few files, try these prompts:

```
What does this project do, based on the files I shared?
```

```
Are there any potential bugs in this code?
```

```
Summarize the dependencies listed in package.json.
```

Claude Desktop reads the content you provided and responds. It cannot browse your file system on its own — it only knows about the files you explicitly share.

### Step 4 — Try creating a new file with Claude Desktop

<StepMeta purpose="Experience the Desktop generate-then-save workflow instead of assuming it edits files automatically." action="Ask Claude to generate file contents, then copy the result out of the chat and save it into a real file yourself." expectedResult="You end up with the generated content in a new file on disk, but only after a manual copy-paste step." whyItMatters="Understanding this manual handoff helps you choose Desktop for planning and drafting rather than for direct disk edits." nextStep="Compare Desktop and CLI capabilities directly so you know what work each surface is actually good at." askTutor="I'm on Step 4 of Use Claude Desktop for local projects. Help me save Claude's generated text into a real file without mangling formatting or line endings." />

Ask Claude to generate something:

```
Write a .gitignore file for a Node.js project.
```

Claude produces the content in the chat. To save it:

1. Select the generated text in the chat.
2. Copy it (Ctrl+C / Cmd+C).
3. Open your editor or terminal and paste it into a new file.

**macOS / Linux:**

```bash
# Paste the clipboard content into a file
pbpaste > .gitignore   # macOS
xclip -selection clipboard -o > .gitignore   # Linux with xclip
```

**Windows PowerShell:**

```powershell
Get-Clipboard | Out-File -Encoding utf8 .gitignore
```

This copy-paste step is a key difference from Claude Code in the terminal, which writes files directly.

### Step 5 — Understand what Claude Desktop can and cannot do

<StepMeta purpose="Build an honest mental model of Desktop strengths and limits so you stop expecting CLI behavior from it." action="Read the comparison table and decide which tasks belong in Claude Desktop versus Claude Code CLI." expectedResult="You can clearly explain at least one scenario where Desktop is the better tool and one where the CLI is better." whyItMatters="Tool choice gets easier once you stop treating every Claude surface as interchangeable and instead match the surface to the task." nextStep="Try the visual-file workflow next by sharing an image or PDF that the CLI would handle less naturally." askTutor="I'm on Step 5 of Use Claude Desktop for local projects. Help me compare Claude Desktop and Claude Code in a way that is practical, not just theoretical." />

Here is the honest comparison:

| Capability | Claude Code (CLI) | Claude Desktop |
|---|---|---|
| Read files in your project | Yes — directly from the file system | Only files you drag in or attach |
| Write/edit files | Yes — edits files on disk | No — generates text you must copy-paste |
| Run terminal commands | Yes — executes shell commands | No |
| Access Git history | Yes — runs `git log`, `git diff`, etc. | No |
| Multi-file refactoring | Yes — can modify many files in one pass | No — one conversation, manual saves |
| Conversation memory | Resets each session (unless using CLAUDE.md) | Remembers within the project/conversation |
| Visual UI | Text-only terminal | Graphical window with rich formatting |
| File preview (images, PDFs) | Limited | Yes — renders images and formatted content |
| Speed of iteration | Fastest for code changes | Best for reading, planning, and discussion |

**When to use Claude Desktop:**

- You want to discuss architecture or plan before coding.
- You need to share an image, screenshot, or PDF for analysis.
- You prefer a graphical conversation with formatted code blocks.
- You are on a machine where you cannot install the Claude Code CLI.

**When to use Claude Code CLI:**

- You want Claude to read, edit, and run code directly.
- You are in the middle of a development session and need fast iteration.
- You need Git integration or shell access.

### Step 6 — Try image and PDF sharing

<StepMeta purpose="Use one Desktop-native strength that feels different from the terminal experience." action="Drag in a screenshot or PDF and ask Claude to explain the error, content, or design shown in the visual file." expectedResult="Claude analyzes the uploaded image or PDF directly in the graphical interface and responds with a grounded explanation." whyItMatters="This is one of the clearest reasons to reach for Desktop first: it handles visual context naturally while the CLI stays focused on text and shell work." nextStep="Move on to MCP so Claude Desktop can go beyond manual attachments and access approved local folders directly." askTutor="I'm on Step 6 of Use Claude Desktop for local projects. Claude Desktop handles visual files that the CLI cannot. Try this:

1. Take a screenshot of an error message or a UI design.
2. Drag the image into the Claude Desktop chat.
3. Ask: `What error is shown in this screenshot?` or `How could this UI be improved?`

Claude analyzes the image and responds. This is something the CLI version cannot do natively — it works only with text files.

### What can go wrong

| Symptom | Cause | Fix |
|---|---|---|
| Claude Desktop does not launch | Installation was incomplete or OS requirements not met | Re-download from claude.ai/download; check system requirements (macOS 12+, Windows 10+) |
| "Sign in" loop — keeps asking for credentials | Browser cookie issue or network firewall | Clear browser cookies for claude.ai, check that your network allows HTTPS to anthropic.com |
| Drag and drop does nothing | You dragged a folder instead of files | Open the folder and drag individual files; Claude Desktop accepts files, not folders |
| File content appears truncated | The file is very large and exceeds the context window | Split large files into smaller chunks or share only the relevant sections |
| Claude says "I can't access your file system" | You asked it to read a file by path without sharing it | Drag the file in or use the attachment button; Claude Desktop cannot browse your disk |
| Generated code has wrong line endings | Claude may produce Unix line endings on Windows | Open the file in VS Code and change line endings (bottom-right status bar) to CRLF if needed |
