# Connect Claude Desktop to local files with MCP

## Objective

By the end of this lesson you will configure a Model Context Protocol (MCP) filesystem server in Claude Desktop, verify the connection, and use it to read and write local files without manual drag-and-drop. This bridges the gap between Claude Desktop's graphical interface and Claude Code's direct file access.

## Walkthrough

### Step 1 — Understand what MCP does

<StepMeta purpose="Build the right mental model before editing config so MCP feels like a controlled tool bridge, not magic." action="Read what MCP enables, focus on the filesystem server tools, and note that access is limited to paths you explicitly allow." expectedResult="You can explain that Claude Desktop talks to a local server which exposes specific file tools for approved folders only." whyItMatters="Good security decisions start with understanding the trust boundary, especially before you grant Desktop any local file access." nextStep="Verify Node.js and the filesystem server package are available so Claude Desktop can launch the server." askTutor="I'm on Step 1 of Connect Claude Desktop to local files with MCP. Help me understand what MCP changes compared with ordinary drag and drop file sharing." />

In Lesson 7.2.1 you saw that Claude Desktop cannot browse your file system on its own — you must drag files in. MCP changes that.

**Model Context Protocol (MCP)** is a standard that lets Claude Desktop connect to local servers that provide tools — like reading files, listing directories, or even running commands. You install a small server on your machine, point Claude Desktop at it, and Claude gains new abilities.

The **filesystem MCP server** gives Claude Desktop these tools:

- `read_file` — read the contents of a file by path
- `write_file` — write content to a file
- `list_directory` — list files and folders in a path
- `search_files` — search for files matching a pattern

> **Safety note:** The filesystem MCP server only allows access to directories you explicitly configure. It cannot read outside those paths. You choose exactly which folders Claude can see.

### Step 2 — Install the filesystem MCP server

<StepMeta purpose="Confirm the local runtime dependency that Claude Desktop will rely on to start the filesystem server." action="Check that Node.js is installed, then run the npx help command for the filesystem MCP server package." expectedResult="Node prints a supported version and the npx command returns usage information for the filesystem server." whyItMatters="If Node or the package launch path is broken, Claude Desktop cannot start the MCP server later even if the config file looks correct." nextStep="Find the Claude Desktop config file location for your operating system so you know exactly where to wire the server in." askTutor="I'm on Step 2 of Connect Claude Desktop to local files with MCP. Help me verify whether Node is installed correctly and what the npx test result should look like." />

The official filesystem server is an npm package. You need Node.js installed.

Confirm Node.js is available:

**macOS / Linux:**

```bash
node --version
```

**Windows PowerShell:**

```powershell
node --version
```

You should see `v18.x.x` or newer. If not, install Node.js from [nodejs.org](https://nodejs.org).

You do not need to install the server globally — Claude Desktop runs it automatically using `npx`. But you can verify the package exists:

```bash
npx @modelcontextprotocol/server-filesystem --help
```

This should print usage information. If it downloads the package first, that is expected.

### Step 3 — Locate the Claude Desktop config file

<StepMeta purpose="Identify the exact configuration file Claude Desktop reads for MCP servers on your operating system." action="Check the OS-specific Claude Desktop config path and note whether the file already exists or needs to be created." expectedResult="You know the precise path to claude_desktop_config.json and whether you are editing an existing file or making a new one." whyItMatters="Using the wrong config location is a common failure mode, so confirming the path now saves confusing restart attempts later." nextStep="Edit that config file and add a filesystem server entry pointing at the folder you want Claude to access." askTutor="I'm on Step 3 of Connect Claude Desktop to local files with MCP. Help me verify the correct config path on my OS and what to do if the file is missing." />

Claude Desktop stores its MCP configuration in a JSON file. The location depends on your operating system.

**macOS:**

```bash
ls ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

**Windows PowerShell:**

```powershell
Test-Path "$env:APPDATA\Claude\claude_desktop_config.json"
```

**Linux:**

```bash
ls ~/.config/Claude/claude_desktop_config.json
```

If the file does not exist yet, you will create it in the next step.

### Step 4 — Add the filesystem MCP server to the config

<StepMeta purpose="Authorize Claude Desktop to start the filesystem MCP server with access only to the folders you choose." action="Open or create claude_desktop_config.json, paste the filesystem server JSON, and replace the sample path with your real absolute folder path." expectedResult="The config file contains valid JSON with a filesystem server entry that points at your chosen project directory." whyItMatters="This is the step that defines both functionality and scope: the server command must launch correctly and the path list must match the folders you actually want to share." nextStep="Restart Claude Desktop and confirm the tools indicator appears, proving the server launched successfully." askTutor="I'm on Step 4 of Connect Claude Desktop to local files with MCP. Help me write the JSON correctly for my machine and avoid path escaping mistakes." />

Open the config file in a text editor. If it does not exist, create it.

**macOS / Linux:**

```bash
# macOS path (adjust for Linux: ~/.config/Claude/)
mkdir -p ~/Library/Application\ Support/Claude
nano ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

**Windows PowerShell:**

```powershell
# Create the directory if needed
New-Item -ItemType Directory -Force -Path "$env:APPDATA\Claude"
notepad "$env:APPDATA\Claude\claude_desktop_config.json"
```

Paste this JSON (replace the paths with your actual project folder):

**macOS / Linux:**

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/projects"
      ]
    }
  }
}
```

**Windows:**

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\yourname\\projects"
      ]
    }
  }
}
```

> **Important:** Use the actual absolute path to your projects folder. On Windows, use double backslashes (`\\`) in JSON. You can list multiple folders by adding more path arguments after the package name.

Save and close the file.

### Step 5 — Restart Claude Desktop and verify the connection

<StepMeta purpose="Prove that Claude Desktop actually launched the MCP server instead of merely storing the config." action="Fully quit and relaunch Claude Desktop, then look for the tools indicator and inspect the available filesystem tools." expectedResult="Claude Desktop shows the hammer or tools indicator and lists filesystem tools such as read_file, write_file, list_directory, and search_files." whyItMatters="A valid-looking config file is not enough; you need evidence that Desktop started the server and attached the tools successfully." nextStep="Use one of the new tools to read a real local file through MCP." askTutor="I'm on Step 5 of Connect Claude Desktop to local files with MCP. Help me troubleshoot why the tools indicator is missing after I restarted Desktop." />

Completely quit Claude Desktop (not just close the window), then relaunch it:

- **macOS:** Right-click the dock icon → Quit, then reopen from Applications.
- **Windows:** Right-click the system tray icon → Exit, then reopen from the Start menu.
- **Linux:** Close the window and relaunch from your app menu.

Open a new conversation in Claude Desktop. You should see a small **hammer icon** (🔨) or **tools indicator** near the input box. Click it to see the available MCP tools. You should see tools like `read_file`, `write_file`, `list_directory`, and `search_files`.

If the tools indicator does not appear, check the Claude Desktop logs:

**macOS:**

```bash
cat ~/Library/Logs/Claude/mcp*.log
```

**Windows PowerShell:**

```powershell
Get-Content "$env:APPDATA\Claude\logs\mcp*.log" -ErrorAction SilentlyContinue
```

The log will tell you if the server failed to start (usually a Node.js path issue or JSON syntax error).

### Step 6 — Use MCP to read a local file

<StepMeta purpose="Verify that MCP turned Desktop into a tool-using client that can inspect approved local files on demand." action="Ask Claude to list a folder and then read a specific file such as README.md using the filesystem tools." expectedResult="Claude uses list_directory and read_file to return real data from your local disk without any drag and drop step." whyItMatters="Reading a real file is the first concrete proof that MCP is bridging Claude Desktop to your local project in a controlled way." nextStep="Use the complementary write tool to create a small verification file through the same MCP connection." askTutor="I'm on Step 6 of Connect Claude Desktop to local files with MCP. Help me tell whether Claude really used the filesystem tool or just answered from earlier context." />

In the Claude Desktop chat, try:

```
List the files in my projects folder.
```

Claude should use the `list_directory` tool and show you a file listing. Then:

```
Read the README.md file in my-app.
```

Claude uses `read_file` and displays the contents. No drag-and-drop needed — Claude is reading directly from your disk through the MCP server.

### Step 7 — Use MCP to write a local file

<StepMeta purpose="Confirm that the MCP connection is not read-only and can safely create files inside approved folders." action="Ask Claude to create mcp-test.txt through write_file, then verify the file contents from your terminal and remove it." expectedResult="The test file is created with the expected text, terminal verification succeeds, and cleanup removes it afterward." whyItMatters="Write access is what turns MCP from a passive reader into a practical workflow bridge for drafts, notes, and controlled file edits." nextStep="Optionally expand the config with more approved folders if you want Desktop to reach additional work areas." askTutor="I'm on Step 7 of Connect Claude Desktop to local files with MCP. Help me verify the file was really written by MCP and explain how to keep this access safe." />

Ask Claude to create a file:

```
Create a file called mcp-test.txt in my projects folder with the content "MCP connection verified."
```

Claude uses `write_file` to write the file. Verify it worked from your terminal:

**macOS / Linux:**

```bash
cat ~/projects/mcp-test.txt
```

**Windows PowerShell:**

```powershell
Get-Content "$HOME\projects\mcp-test.txt"
```

You should see `MCP connection verified.` printed. Claude Desktop just wrote a file to your disk without you copy-pasting anything.

Clean up:

**macOS / Linux:**

```bash
rm ~/projects/mcp-test.txt
```

**Windows PowerShell:**

```powershell
Remove-Item "$HOME\projects\mcp-test.txt"
```

### Step 8 — Add access to more folders (optional)

<StepMeta purpose="Learn how to expand MCP access deliberately instead of granting broad filesystem visibility all at once." action="Add additional folder paths to the args array only for directories you truly want Desktop to access, then restart Claude Desktop." expectedResult="The config supports multiple approved directories and Desktop can use the filesystem tools within those extra paths after a restart." whyItMatters="Scoped expansion keeps the security model understandable: more power is fine, but only when you add access intentionally and minimally." nextStep="After basic MCP works, compare this Desktop plus MCP workflow with the browser and CLI surfaces you have already tried." askTutor="I'm on Step 8 of Connect Claude Desktop to local files with MCP. Help me decide which folders are reasonable to expose and which ones I should keep out of scope." />

To give Claude access to additional directories, add more paths to the args array:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/projects",
        "/Users/yourname/Documents/notes"
      ]
    }
  }
}
```

Restart Claude Desktop after editing the config.

> **Safety note:** Only add folders you are comfortable sharing with Claude. Do not add your entire home directory or system paths. Start with specific project folders and expand only if needed.

### What can go wrong

| Symptom | Cause | Fix |
|---|---|---|
| No tools indicator (hammer icon) after restart | Config file has a JSON syntax error | Validate JSON at jsonlint.com or run `cat claude_desktop_config.json | python3 -m json.tool` |
| `Error: Cannot find module` in MCP logs | Node.js is not on PATH when Claude Desktop launches | Install Node.js globally; on macOS add Node to `/etc/paths` or use `nvm` with shell init |
| `read_file` returns "permission denied" | The MCP server does not have access to that path | Add the folder to the args list in the config and restart Claude Desktop |
| `write_file` fails silently | Target directory does not exist | Create the directory first, or ask Claude to create it |
| Tools appear but Claude never uses them | Claude chose not to use tools for that request | Be explicit: "Use the read_file tool to read README.md" |
| Config file path is wrong on Windows | Used forward slashes or single backslashes | Use double backslashes (`\\`) in JSON on Windows, e.g., `C:\\Users\\you\\projects` |
| Claude Desktop hangs on startup | MCP server process crashed at launch | Check MCP logs; try running `npx -y @modelcontextprotocol/server-filesystem /your/path` manually in a terminal to see the error |

<VerificationBlock>
Confirm the MCP connection is active:

1. Open Claude Desktop and start a new conversation.
2. Click the tools indicator (hammer icon near the input box).
3. You should see `read_file`, `write_file`, `list_directory`, and `search_files` in the tools list.

Then in your terminal, verify the config file exists:

**macOS:**

```bash
cat ~/Library/Application\ Support/Claude/claude_desktop_config.json | python3 -m json.tool
```

**Windows PowerShell:**

```powershell
Get-Content "$env:APPDATA\Claude\claude_desktop_config.json" | ConvertFrom-Json | ConvertTo-Json -Depth 5
```

The output should show valid JSON with a `filesystem` entry under `mcpServers`. If both checks pass, your MCP connection is working.
</VerificationBlock>
