# Mini-project: automate a team ritual

## Objective

In this mini-project you will pick one recurring team ritual, automate it with Claude, document the process, save it as a reusable template, and commit the entire package to version control. By the end you will have a working, documented automation that anyone on the team can run.

## Walkthrough

### Step 1 — Pick a team ritual to automate

<StepMeta purpose="Choose a recurring workflow that is worth automating and safe to hand to a Claude-assisted process." action="Evaluate candidate rituals against the weekly, formatting-heavy, and low-risk criteria, then commit to one concrete ritual for the mini-project." expectedResult="You select one ritual, such as weekly changelog generation, that is realistic to automate with Claude." whyItMatters="Picking the right ritual determines whether the project is useful and safe; high-frequency, low-risk formatting work is the best place to start." nextStep="Create a dedicated automation directory for that ritual so the scripts, prompts, and outputs stay together." askTutor="I'm on Step 1 of Mini-project: automate a team ritual. Help me judge whether this ritual is the right kind of repeated, low-risk work to automate with Claude." />

Choose a ritual that:
- Happens at least weekly
- Involves gathering information and formatting it
- Currently takes 15+ minutes of manual effort
- Does not involve sensitive decisions (no deploys, no access changes)

Good candidates:
- Weekly standup summary from git history
- Sprint retrospective notes from a shared doc
- Changelog generation from merged PRs
- Weekly metrics snapshot from a dashboard

For this walkthrough, we will automate **weekly changelog generation from git**.

### Step 3 — Write the data-gathering script

<StepMeta purpose="Automate the collection of raw evidence so Claude receives consistent inputs every time the ritual runs." action="Create the Bash and PowerShell gather scripts that export commits, diff stats, and merged branches into the output directory." expectedResult="You end with at least one data-gathering script that produces the raw files needed for changelog generation." whyItMatters="Reliable automation starts with deterministic inputs, and the gather script is what turns a repeatable ritual into something teammates can run consistently." nextStep="Write the Claude session brief that tells Claude how to turn those raw files into a changelog without inventing details." askTutor="I'm on Step 3 of Mini-project: automate a team ritual. Help me review this gather script and check whether it collects the right raw inputs for a trustworthy changelog." />

Create a script that collects the raw inputs for Claude.

**macOS / Linux**

```bash
cat > gather-changelog-data.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

REPO_DIR="${1:-.}"
OUTPUT_DIR="$(pwd)/output"
mkdir -p "$OUTPUT_DIR"

cd "$REPO_DIR"

echo "Gathering commit log..."
git log --since="7 days ago" --no-merges --format="%h %s (%an)" > "$OUTPUT_DIR/commits.txt"

echo "Gathering diff stats..."
git diff --stat "HEAD~$(git rev-list --count --since='7 days ago' HEAD)" HEAD > "$OUTPUT_DIR/diffstat.txt" 2>/dev/null || echo "No diff available" > "$OUTPUT_DIR/diffstat.txt"

echo "Gathering merged branches..."
git branch --merged main --no-color | grep -v "main" | sed 's/^[* ]*//' > "$OUTPUT_DIR/merged-branches.txt" 2>/dev/null || echo "None" > "$OUTPUT_DIR/merged-branches.txt"

echo "Data gathered in $OUTPUT_DIR/"
ls -la "$OUTPUT_DIR/"
SCRIPT
chmod +x gather-changelog-data.sh
```

**Windows PowerShell**

```powershell
@'
param(
    [string]$RepoDir = "."
)

$OutputDir = Join-Path (Get-Location) "output"
New-Item -ItemType Directory -Force -Path $OutputDir | Out-Null

Push-Location $RepoDir

Write-Host "Gathering commit log..."
git log --since="7 days ago" --no-merges --format="%h %s (%an)" | Out-File "$OutputDir\commits.txt"

Write-Host "Gathering diff stats..."
try {
    $count = git rev-list --count --since="7 days ago" HEAD
    git diff --stat "HEAD~$count" HEAD | Out-File "$OutputDir\diffstat.txt"
} catch {
    "No diff available" | Out-File "$OutputDir\diffstat.txt"
}

Write-Host "Gathering merged branches..."
try {
    git branch --merged main --no-color | Where-Object { $_ -notmatch "main" } | ForEach-Object { $_.Trim("* ").Trim() } | Out-File "$OutputDir\merged-branches.txt"
} catch {
    "None" | Out-File "$OutputDir\merged-branches.txt"
}

Pop-Location

Write-Host "Data gathered in $OutputDir/"
Get-ChildItem $OutputDir
'@ | Out-File "gather-changelog-data.ps1"
```

### Step 4 — Write the Claude session brief for this ritual

<StepMeta purpose="Define the exact transformation Claude should perform on the gathered data." action="Create changelog-prompt.md with the context, file list, grouping rules, formatting expectations, and output path for the weekly changelog." expectedResult="You have a reusable session brief that tells Claude how to generate the changelog from the raw output files." whyItMatters="The session brief is the contract between your data-gathering step and Claude's summarization step, which keeps the automation consistent across runs." nextStep="Run the gather script, inspect the collected files, and then feed the session brief to Claude for the full end-to-end flow." askTutor="I'm on Step 4 of Mini-project: automate a team ritual. Help me tighten this session brief so Claude groups the changelog correctly without guessing beyond the raw files." />

**macOS / Linux**

```bash
cat > changelog-prompt.md << 'EOF'
# Session Brief: Weekly Changelog

## Context
- Project: [repo name]
- Goal: Generate a user-friendly changelog from this week's git activity
- Files to read: output/commits.txt, output/diffstat.txt, output/merged-branches.txt

## Instructions
1. Read all three files in the output/ directory.
2. Group commits by theme (features, fixes, docs, chores).
3. Write each entry as a single clear sentence a non-developer can understand.
4. Skip merge commits and trivial chores (typo fixes, formatting).
5. If a commit message is unclear, keep it brief rather than guessing.

## Format
```markdown
# Changelog — Week of [date]

## New Features
- [description]

## Bug Fixes
- [description]

## Documentation
- [description]

## Other
- [description]
```

## Output
Save to: output/changelog-YYYY-MM-DD.md
EOF
```

```powershell

### Step 6 — Verify the changelog against the raw data

<StepMeta purpose="Make sure the generated changelog is grounded in actual commits and categorized honestly." action="Compare the changelog entries to output/commits.txt and the other raw files, checking for invented features, misgrouped fixes, or missing items." expectedResult="You confirm that each changelog entry traces back to real raw data and that the grouping is accurate." whyItMatters="Stakeholder-facing ritual automation only works when the generated output stays trustworthy, and changelogs are especially sensitive to invented claims." nextStep="Document the entire process in a README so another teammate can run the automation without reverse-engineering it." askTutor="I'm on Step 6 of Mini-project: automate a team ritual. Help me audit this generated changelog so I can catch any invented or miscategorized entries before sharing it." />

After Claude generates the changelog, check every entry:

**macOS / Linux**

```bash
# Show the generated changelog
cat output/changelog-*.md

# Compare against raw commits
diff <(grep -oP '(?<=^[a-f0-9]+ ).*' output/commits.txt | sort) <(echo "manual comparison")
```

**Windows PowerShell**

```powershell
# Show the generated changelog
Get-Content output\changelog-*.md

# Show raw commits for comparison
Get-Content output\commits.txt
```

Confirm:
1. Every changelog entry maps to at least one real commit.
2. No invented features or fixes appear.
3. The grouping (Features, Fixes, Docs) is accurate.

> **Real danger:** Sharing a changelog with invented entries — features that were not built, bugs that were not fixed — misleads stakeholders and erodes trust. Always verify before distribution.

### What can go wrong

| Symptom | Cause | Fix |
|---|---|---|
| Gather script finds zero commits | No commits in the last 7 days, or wrong repo path | Check the repo path argument and widen the date range |
| Changelog groups entries incorrectly | Commit messages do not follow a convention (feat:, fix:, etc.) | Add a note in the README that conventional commits improve changelog quality |
| Claude invents entries not in commits.txt | Vague commit messages triggered hallucination | Add to the prompt: "Only reference work explicitly described in commits.txt" |
| Script fails on Windows with permission error | PowerShell execution policy blocks scripts | Run `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned` |
| Output directory grows unbounded over weeks | Old changelogs accumulate | Add a cleanup step to the script or `.gitignore` the entire `output/` directory (already done in Step 8) |
| Teammate cannot reproduce the results | Different git config or branch state | Document required branch (e.g., "Run from the main branch") in the README |
