## What You'll See

A roadmap for your next learning steps. You'll understand which path to take based on your operating system and goals.

## Why This Matters

Claude Code works on all platforms, but each has unique setup steps, quirks, and best practices. This lesson points you to the right learning path so you can avoid frustration and work efficiently on your platform.

## The Paths

You've completed the **Claude Code Beginners** path. You now understand:

- What Claude Code is and how it differs from Claude.ai
- How to install and authenticate it
- How to write CLAUDE.md to guide Claude Code
- How to ask Claude Code for help
- How to understand context windows
- How to work safely with secrets
- How to read Claude Code's output

That's the foundation. Now comes platform-specific knowledge.

## Choose Your Platform

### If You're on Windows

Take the **Claude Code for Windows** path.

You'll learn:

- **Windows Environment Options** — WSL2 vs Git Bash vs PowerShell. Which is best and why.
- **WSL2 Setup** — Complete walkthrough. WSL2 gives you the best experience.
- **PowerShell Alternative** — If you can't or don't want WSL2, how to use Claude Code in PowerShell.
- **Path and Permissions** — Windows-specific quirks: backslashes, execution policy, PATH settings.
- **Windows File System Quirks** — `/mnt/c` vs `C:\`, line endings, Git settings that prevent pain.
- **First Run on Windows** — End-to-end walkthrough of your first Claude Code session on Windows.

**Recommendation:** Start with WSL2 Setup. It's the most powerful option and worth 30 minutes of setup.

### If You're on macOS

Take the **Claude Code for macOS** path.

You'll learn:

- **Homebrew and Node** — Install Node.js properly using Homebrew and nvm (Node Version Manager).
- **Zsh Configuration** — Set up your shell so Claude Code, npm, and other tools work smoothly.
- **Apple Silicon Notes** — If you have an M1/M2/M3 Mac, what to know about Rosetta and native arm64 support.
- **First Run on macOS** — End-to-end walkthrough of your first Claude Code session on macOS.
- **Common macOS Gotchas** — Gatekeeper, System Integrity Protection (SIP), permission dialogs, sudo issues.

**Recommendation:** Start with Homebrew and Node. Good tooling is the foundation.

### If You're on Linux

Great! You probably already have Node.js and a shell. You might not need a whole path. But keep the Windows and macOS paths in mind—they have good troubleshooting patterns you can adapt.

Key things to verify:
- Node.js 18+: `node --version`
- npm works: `npm --version`
- npm can install globally: `npm install -g` without sudo

If all three work, you're ready to go deeper.

## After Platform Setup

Once you've completed your platform path, you have two options:

## FAQ: Should I Learn X?

**Q: Should I learn advanced MCP stuff right now?**

A: No. Build a project first. Learn what you need when you need it.

**Q: Should I memorize all the Claude Code flags and options?**

A: No. Just use `claude --help` when you need something. You'll pick up patterns over time.

**Q: What if I hit a problem?**

A: Ask Claude Code. It's probably encountered the problem before. If it can't solve it, come back to the lessons or ask the community.

**Q: How long until I'm "good" at using Claude Code?**

A: After using it on 2-3 real projects, you'll be comfortable. After 10+ projects, you'll be fluent. Expertise comes from practice, not lessons.
