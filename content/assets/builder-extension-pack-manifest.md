# Builder Extension Pack Manifest

Use this manifest in Week 5 when the spine project starts becoming a real system.

## Required pieces

- One `CLAUDE.md` file with project rules
- One reusable skill
- One memory rule or memory review note
- One safe integration with minimum viable scopes
- One review checklist before approval

## Suggested folder layout

```text
builder-extension-pack/
├── CLAUDE.md
├── skills/
├── memory/
├── integrations/
└── review/
```

## Acceptance criteria

- Every file has a clear owner and purpose.
- Nothing requires hidden context to understand.
- A reviewer can see where instructions live, where memory lives, and how to disable or revoke integrations.
- The pack improves one real workflow instead of trying to be a universal framework.
