# Week 1 Local Starter Assistant Field Manual

Use this during the first 10 hours of CLI Academy.

## Outcome

By the end of Week 1 you should have:

- a `cli-academy/` workspace
- a safe terminal baseline
- Claude Code installed and authenticated
- one practice repo opened
- one verified first improvement

## Stop Gates

Do not move to the next stage until the current one is true.

1. Workspace gate
   - `cli-academy/notes`
   - `cli-academy/projects`
   - `cli-academy/screenshots`
2. Terminal gate
   - you can print your current path
   - you can list files
   - you know how to get back to your home directory
3. Claude Code gate
   - CLI starts successfully
   - auth is complete
   - you can open a repo and ask for an explanation before editing
4. First win gate
   - one small change completed
   - output verified
   - a short change note written in plain English

## Common Failure Points

- Wrong shell on Windows
- PATH not refreshed after install
- Working in a protected or confusing folder
- Asking Claude to edit before asking it to explain
- Accepting changes before checking the diff

## Evidence To Keep

- one screenshot of the workspace tree
- one screenshot of Claude Code running
- one text note explaining your first improvement
