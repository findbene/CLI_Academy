# Builder Extension Pack Blueprint

Use this blueprint to turn one-off success into a reusable builder system.

## Pack Contents

- one project `CLAUDE.md`
- one narrow skill
- one useful hook or slash command concept
- one prompt template
- one context packet

## Design Rules

- narrow scope beats ambitious scope
- explicit boundaries beat implied intent
- read-only first
- keep one human review point

## Suggested Build Order

1. Define the workflow in plain English.
2. Write the minimum `CLAUDE.md`.
3. Add one skill that triggers reliably.
4. Add one hook only if it removes repeated manual work.
5. Test the pack on one small real task.

## Packaging Checklist

- purpose is clear
- triggers are specific
- no secret values included
- instructions explain stop conditions
- one example run exists
