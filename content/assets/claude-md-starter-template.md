# CLAUDE.md Starter Template

Copy this into `CLAUDE.md` at the root of a new project, then replace the placeholders with the real details.

```markdown
# Project: [Project name]

Short description of what this project does and who it serves.

## Mission

- Main goal:
- What success looks like:
- What should not be changed casually:

## Architecture

- Frontend:
- Backend:
- Database:
- Deployment:

## File Organization

- `/src/` - Main source code
- `/src/components/` - UI components
- `/src/lib/` - Shared helpers
- `/tests/` - Tests

## Coding Standards

- Prefer clear, readable changes over clever changes
- Keep comments short and only explain non-obvious reasoning
- Do not commit secrets, tokens, or private env files
- Match the project's existing style before introducing new patterns

## Before You Change Code

1. Read the relevant files first
2. Check for existing decisions or constraints
3. Run the project's verification commands before calling work done

## Definition of Done

- Lint passes
- Type checks pass
- Relevant tests pass
- Build succeeds
- User-facing behavior is explained clearly

## Safety Rules

- Ask before destructive operations
- Do not rewrite unrelated files
- Do not add dependencies without a reason
```

## First Edits to Make

- Replace the project name.
- Replace the architecture placeholders.
- Replace the verification commands with the real commands for your repo.
- Add any business or safety rules that matter to your team.
